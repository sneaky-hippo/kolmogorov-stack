# kolm SOTA — Spec / Map / Plan
*v7.7. CSO + CTO + CEO synthesis. 2026-05-11.*

## 1. The product in one sentence

**kolm compiles a task into a working AI you own** — one signed file (`.kolm`) that carries the model, the adapter, the verifier, the eval suite, and a cryptographic receipt of the gate that let it ship. Run it on a laptop, a phone, or inside a VPC. Sovereign by construction.

## 2. The strategic frame (CSO)

### 2.1 Who we are NOT
- We are not a memory company (that's Mem0, Zep, Letta).
- We are not an inference gateway (OpenRouter, Anthropic, Together).
- We are not a fine-tuning service (Predibase, OpenPipe).
- We are not a RAG vendor (LangChain, LlamaIndex).
- We are not a benchmark leaderboard (LMSYS, HELM).

### 2.2 Who we are
The **spec layer of AI**: a portable, signed, reproducible artifact format + the compiler that emits it. Closest mental model: Docker for AI workflows. The customer owns the artifact; we run the compiler and the registry.

### 2.3 Why now (the wedge)
- 2026: AI cost is hitting board agendas — Q1 OpenAI usage in regulated industries is being pulled back. Buyers want sovereignty without giving up frontier quality.
- Frontier models (Claude / GPT / Gemini) hit cost + latency walls in narrow tasks where a distilled 7B on hardware you own does the job at 100× cheaper.
- Compounding: every captured run can become a training signal. Skeleton LoRA → living model — a moat that grows monthly.

### 2.4 The market we're targeting (in priority order)
1. **AI-native SaaS founders** building coding agents, classifiers, summarizers. They use Claude Code / Cursor and want to localize the hot path.
2. **Healthcare CIOs** with PHI workloads who can't send to cloud LLMs.
3. **Fintech / hedge fund quants** with MNPI redaction, low-latency compliance.
4. **Defense / gov contractors** with airgap requirements.
5. **Legal IT** with attorney-client privilege.
6. **Edge / robotics** with no-network embedded deployment.

### 2.5 Competitive moat (5 layers, weakest to strongest)
1. **Brand**: "the private AI compiler" — buy the name, lose the noise.
2. **Format**: `.kolm` is the artifact people port between vendors. Whoever owns the format owns the standard.
3. **Receipts**: signed cryptographic proof of (source, input → output) at compile-time + run-time. Auditable.
4. **K-score**: the gate. One ruler. Same formula at compile / install / serve / tune-promote.
5. **Compounding evolution**: skeleton LoRA → tune → promote → still the same `.kolm`. The artifact gets smarter on YOUR data without rebuild.

## 5. Mobile-app decision

**Do we need a mobile app?**

**No, not yet — but mobile-responsive dashboard is P1.**

### Reasoning
1. The product is a developer + ops tool. Compilation happens in CI/CD or on a workstation. Mobile is **operator-monitor**, not **operator-author**.
2. The natural-language assistant we shipped today already does 80% of what a mobile app would do (status, builds, usage, tune trigger) over plain HTTP. Wrap the dashboard in a responsive grid and a homescreen icon (PWA) and you're done.
3. The 20% mobile users actually need is **alerts** (job done, K-score dropped, billing event). That's email + webhook + Slack, not an app.
4. iOS app shop overhead is 8-12 weeks of effort that returns ~$0 in MRR against developer ICP.

### Action
- T+30: dashboard.html + signup/signin become mobile-first (current is desktop-first).
- T+45: PWA manifest + add-to-homescreen.
- T+60: Slack notifier (/v1/account/notify-channel { slack_webhook }).
- T+180: revisit; if mobile traffic exceeds 15% of authed sessions, scope an app.

- Hosted training cluster (we ship compile-bridge for customer-owned GPUs).
- Frontend SDK for browsers (use the existing `/v1` API with fetch).
- A leaderboard people pay to be on (the leaderboard is free, K-score is the gate).
- Plugins/extensions ecosystem (the CLI + MCP + hooks ARE the ecosystem).
