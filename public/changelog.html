v5.5 2026-05-08 Hotfix - mobile pass

Kill horizontal scroll on iPhone. Fix brand-dot duplication on canonical pages. Hide nav under 920px so the header stops overflowing.

Real-device test on a 5G iPhone Safari surfaced six issues that desktop emulators didn't. Fixed all of them in brand-refresh.css with !important overrides that win over the page-level inline <style> blocks (brand-refresh loads after).

  • Brand dot rendered twice on canonical pages. Removed .site-header from the ::before pseudo-dot selector — those pages already render an inline <span class="brand-dot">.
  • Nav showing at 390px when styles.css said display:none at <920px. The brand-refresh layer was forcing display:flex !important across all widths. Wrapped the rule in @media (min-width:921px).
  • "get a key" CTA wrapped to three lines. Tightened to padding:6px 12px and white-space:nowrap below 720px.
  • Hero lede paragraph cut off mid-word. The page was scrolling sideways. Added html, body { max-width:100%; overflow-x:hidden } as the defensive net plus min-width:0 on every grid child that could push the viewport.
  • Terminal blocks horizontal-scrolling on /serve, /compile, /k-score. Forced overflow-wrap:anywhere; word-break:break-word; white-space:pre-wrap on every pre, .terminal, .codebox, code at <720px so they wrap instead of widening the page.
  • /k-score figure.plate cap overlapping the SVG diagram. Below 480px, dropped aspect-ratio:16/9 and made the cap position:static so it sits below the image as a normal block instead of absolute-overlapping it.
