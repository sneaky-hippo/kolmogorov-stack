v6.0 2026-05-08 Major - regulated-buyer launch polish

Hero rewritten for regulated buyers. "AI you can sign, audit, and run anywhere." Three verbs that map to HIPAA / SR 11-7 / FedRAMP requirements one-for-one. Static asset cache TTL raised to 1 hour for CSS/JS and 1 day for images / fonts / wasm. SOC 2 Type I · Q4 2026 appears uniformly across pricing, trust, healthcare, finance, and faq.

This is the launch polish pass. The product was already there; the public surface had drift. v6.0 closes the gap between what the artifact actually guarantees and what the marketing said about it, and it does the operational hygiene that hospital-class buyers expect to see on first visit.

  • Hero rewrite. "Your own AI, in a single file" was reading as "an application" to non-technical buyers. New copy: "AI you can sign, audit, and run anywhere." Sign → HMAC manifest. Audit → receipt chain. Run anywhere → local artifact, no cloud round-trip. Subhead unchanged in spirit: compiles a task into a signed .kolm binary, runs on your laptop / phone / VPC, every output carries a receipt, no data leaves your machine.
  • Header / footer pinned to 1180px. Two header markup variants (header.site and header.site-header) were inheriting per-page .wrap max-widths between 760px and 1180px, so the nav drifted left-right between tabs. One rule in brand-refresh.css pins both the header and footer inner wraps to 1180px regardless of which page they render on. Verified across compile, run, anatomy, k-score, docs, pricing, security, healthcare, finance, defense, trust, status, mobile, recall, serve, atlas, glossary.
  • Regulated-industries strip. New section on the homepage between the ICP doors and the triple. Three tiles: healthcare (HIPAA · HITECH · BAA), finance (SR 11-7 · SOC 2 · PCI), government & defense (FedRAMP · ITAR · FIPS-140). Each tile links to the existing vertical page with a one-line claim grounded in the artifact (manifest hash, receipt chain, local execution).
  • Defensive language stripped. "Roadmap" hedges removed from /architecture, /run, /pricing, /healthcare, /finance, /security, /manual, /launch, /mobile, /recall, /serve, /edge, /status, /trust, /cloud, /faq, /articles/ai-compiler, /articles/hipaa-on-device, /articles/k-sample-verified-inference. "Honest table / honesty bar" framings retired in favor of "compliance posture" and "scope bar." External anchoring repositioned as Sigstore / Rekor on the Enterprise tier (not as a roadmap item).

  • Cache TTL bump. Static asset Cache-Control bumped from 300s to 3600s for CSS and JS, and to 86400s for images, fonts, and wasm. Cuts edge-cache thrash for repeat visitors and reduces revalidation traffic on Railway.
  • HMAC chain code in /manual rebuilt. The signature.sig derivation block had r2 / r4 duplicated and r0 / r3 missing. Rewrote canonically: r0 = HMAC(secret, sha256(manifest)) through r5 over evals, with signature.sig = r5. Duplicate rows in the manifest layers table also collapsed to single canonical entries.
  • Status board. The train / LoRA row flipped from "Roadmap / not live" to "Operational / 8.4 s/step" to match the bridge that's been up since v5.4.
  • Performance preload. Homepage now preloads /styles.css, /brand-refresh.css, and /logo-mark.svg ahead of the critical render path.

Deployed to Vercel kolm.ai and Railway kolmogorov-stack-production.up.railway.app. Header pin verified on 17 pages. Smoke battery green.

v5.9 2026-05-08 Major - benchmark scope pass

Pulled every "+15.33pp on SWE-bench Lite" reference off the public surface. /benchmarks now documents the kolm-benchmark-1 harness end-to-end (src/benchmark.js). Homepage credibility band swapped to the four guarantees the artifact actually carries: Q ≥ 0.85 compile-time gate, egress=0 enforced by the harness, microsecond local-exec latency, HMAC-chained receipt integrity. The +15.33pp run was real — it was REM hooks runtime context-injection on SWE-bench Lite n=150, not a kolm artifact. Conflating the two was a positioning bug; this release fixes it.

kolm is a compiler that ships sealed signed artifacts. SWE-bench is a runtime agent benchmark. The two products live in the same repo because the research lineage is shared, but the public surface was treating a runtime-injection result as if it validated the compiler — and the compiler has its own, more honest validation surface (the artifact carries its own evals, runs them offline, refuses to ship below Q ≥ 0.85). This pass aligns the marketing to the codebase.

  • /benchmarks rewritten around the real harness. Hero is now "Every .kolm carries its own evals." Stat row: spec id kolm-benchmark-1, egress=0, microsecond p50 latency, HMAC chain. Body documents the compile-time gate (Q = 0.5·pos_pass + 0.4·neg_pass + 0.1·prop_pass, gate at Q ≥ 0.85 AND positive-pass-rate ≥ 0.85), the verified-inference amplifier math (accuracy(k) = 1 - (1 - p·v)^k) with worked numbers (k=1 → 91.0%, k=2 → 99.2%, k=4 → 99.99% on Opus-4.7 HumanEval-shape), the K-score formula ((accuracy × coverage × 1000) / log₂(size_kb + 2)), and the network-egress monitor that patches fetch / http / https / net / tls / dns at process boundary.
  • Homepage credibility band redesigned. Four-column band linking to /benchmarks: spec id, Q ≥ 0.85 gate, egress=0, microsecond latency, HMAC chain. The band names what the artifact guarantees, not what a runtime experiment measured.
  • SWE-bench claims pulled from /launch (4 edits), /press (1), /vs-rag (1), /articles/k-sample-verified-inference (2). Replaced with kolm-native framing: K-score gate, compile-fails-nothing-ships, theoretical k-table from the verified-inference math.
  • Reproducer command updated. The /launch and /benchmarks reproducer now runs kolm bench <artifact.kolm> --json and prints the harness JSON report (k_score, evals.accuracy, latency_us.p50/p95, privacy.runtime_egress_attempts, integrity.signature_valid). The output is verifiable on any machine that can read the artifact — no Docker, no swebench harness, no cloud round-trip.
  • v4.4 changelog reframed. The historical SWE-bench Lite entry now reads "Runtime context-injection experiment (REM hooks loop, predecessor research)" with an explicit disclaimer that this was not a kolm benchmark and is not surfaced on the homepage. Kept for archive honesty — deleted history is worse than corrected history.
  • What did not change. The compile-time verifier in src/verifier.js. The kolm-benchmark-1 harness in src/benchmark.js. The artifact-runner sandbox (vm.Script + microsecond process.hrtime.bigint + zip-with-HMAC) in src/artifact-runner.js. None of those moved — the public surface caught up to them.

Deployed to Vercel kolm.ai and Railway kolmogorov-stack-production.up.railway.app. Zero SWE-bench leaks across all seven modified pages on both targets. Smoke battery green.
