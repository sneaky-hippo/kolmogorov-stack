kolm is a compiler that ships sealed signed artifacts. SWE-bench is a runtime agent benchmark. The two products live in the same repo because the research lineage is shared, but the public surface was treating a runtime-injection result as if it validated the compiler, and the compiler has its own, more honest validation surface (the artifact carries its own evals, runs them offline, refuses to ship below Q ≥ 0.85). This pass aligns the marketing to the codebase.

  • /benchmarks rewritten around the real harness. Hero is now "Every .kolm carries its own evals." Stat row: spec id kolm-benchmark-1, egress=0, microsecond p50 latency, HMAC chain. Body documents the compile-time gate (Q = 0.5·pos_pass + 0.4·neg_pass + 0.1·prop_pass, gate at Q ≥ 0.85 AND positive-pass-rate ≥ 0.85), the verified-inference amplifier math (accuracy(k) = 1 - (1 - p·v)^k) with worked numbers (k=1 → 91.0%, k=2 → 99.2%, k=4 → 99.99% on Opus-4.7 HumanEval-shape), the K-score formula ((accuracy × coverage × 1000) / log₂(size_kb + 2)), and the network-egress monitor that patches fetch / http / https / net / tls / dns at process boundary.
  • Homepage credibility band redesigned. Four-column band linking to /benchmarks: spec id, Q ≥ 0.85 gate, egress=0, microsecond latency, HMAC chain. The band names what the artifact guarantees, not what a runtime experiment measured.
