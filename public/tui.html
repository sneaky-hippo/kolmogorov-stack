kolm tui

Alt-screen multi-pane shell for the .kolm value loop. Live captures via SSE, artifact inspect & verify, one-key distill & replay. Designed to live inside a tmux pane over SSH+Tailscale, on a phone, on a remote box. Wave 222

Down · up · top · bottom · switch active pane.

Source

1  2

1 = live captures (SSE from /v1/capture/stream). 2 = your local artifacts.

Filter & command

/  :

/ types a filter (Enter commits, Esc cancels). : opens vim-style command mode for :open, :inspect, :run, :verify, :quit.

Actions

Enter  r

Enter opens the highlighted row. r refreshes both panes (re-lists artifacts, re-opens the SSE stream).

Value loop

d  R  v

d distills the selected capture namespace (POST /v1/distill/from-captures). R replays the capture against the loaded artifact (POST /v1/replay). v verifies the loaded artifact receipt (POST /v1/receipts/verify).

Help & exit

?  q

? toggles the help overlay. q quits (cleanly exits the alt screen, restores cursor + scrollback).

Install & run

# grab the CLI (the rule path · zero heavy deps)
curl -fsSL https://kolm.ai/install.sh | sh

# in any project where you ran `kolm proxy on`
kolm tui

# over SSH + tmux + Tailscale (the @sudoingX 5-foundations stack)
tmux new -d -s kolm 'ssh box "tmux new -As kolm \"kolm tui\""'

Requires a TTY. In non-interactive contexts (CI, piped stdout) kolm tui exits cleanly with a hint pointing you at kolm list / kolm inspect / kolm tail captures.

What this is for

  • You're in a remote tmux pane over Tailscale, on a phone via Termius, or on a sleep-then-resume laptop. The TUI survives disconnects (the tmux session holds it; kolm tui itself is just a process).
  • You want to watch captures arrive live (the W213 SSE stream) without leaving the terminal for a browser tab.
  • You want one-key actions on what you see: d distill, R replay, v verify — the close-the-loop verbs from the W213-W216 value loop.
  • You don't want yet another Electron app installed and don't want a third-party TUI framework dragging dozens of transitive deps.

What it intentionally is not

  • Not a full terminal-emulator-grade renderer · we redraw the whole frame per keypress because the data volumes (≤500 captures, ≤200 artifacts) make diff-rendering premature optimization.
  • Not a replacement for /captures when you want a mouse + drawer-style UX in a browser. Both consume the same SSE.
  • Not an embedded editor · vim-style command mode is for navigation parity, not for replacing your shell or editor.
