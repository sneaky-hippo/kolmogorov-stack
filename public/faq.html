Runtime

Where does a .kolm artifact run-

The Node CLI inspects, evaluates, runs, and serves .kolm artifacts over MCP on macOS, Linux, and Windows today. The same signed bytes execute on iOS and Android through native adapters; each adapter verifies the manifest and receipt before the model is loaded.

Does it actually run offline-

Yes. After the artifact is present, recipe-mode and LoRA-backed artifacts execute locally with no network call. Verify with kolm inspect and kolm eval on a disconnected machine to confirm the runtime path declared in the manifest.

How do I update an artifact-

Re-run kolm compile with new examples or a new corpus, push the new artifact to your distribution channel (MDM, App Store, internal mirror, signed sneakernet). The manifest carries a version field; clients can pin or auto-upgrade.

The artifact joins the local MCP bus and is discoverable by Claude, Codex, Cursor, and Zed by name. Matching agent calls hit the local cache instead of the frontier. Latency drops from cloud round-trip to local. /serve.

What if my user's device is too old-

Each release publishes a benchmark matrix on named hardware: p50 / p95 latency, memory, bundle size, battery draw, and thermal posture. The artifact picks the best fit at runtime — native LoRA on capable devices, recipe-only on tighter ones — with a documented fallback chain. Hardware-tier targeting expands across the device matrix on each release.

Compliance

HIPAA Business Associate Agreement-

BAA reviewed and countersigned on Enterprise. The customer-hosted runtime keeps PHI inside the boundary the auditor already trusts. Tenancy, retention, networking, and responsibility split are documented before contract. Healthcare overview.

SOC 2-

Type I attestation tracking Q4 2026, Type II Q2 2027. Controls library, evidence checklist, and audit calendar available under NDA for Enterprise diligence.

EU AI Act-

Designed with high-risk Annex III obligations in mind. Per-artifact manifests, verifier gates, and receipts support technical documentation; the actual risk tier and conformity path are customer- and use-case-specific.

FedRAMP, CMMC, ITAR-

No FedRAMP authorization, CMMC certification, or ITAR compliance attestation is available today. The defense page describes an architecture direction for offline artifact provenance; regulated deployments require customer-led classification, controls mapping, and legal review. Defense & intel overview.

Is the privacy claim audited by anyone other than you-

No third-party audit report is published yet. The receipt chain is cryptographically checkable today for the HMAC-bound fields it covers: manifest hash, layer hashes, eval metadata, and receipt body. That is useful evidence, not a replacement for a security audit.

Open source
