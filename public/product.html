The four verbs

1 · Capture

Point your traffic at the kolm proxy. Every prompt + response lands in a durable corpus you own, with a receipt header on each row.

Live capture dashboard →

2 · Compile

One command turns a namespace of captured pairs into a signed .kolm artifact. Frontier-tier presets pick the base model for your hardware.

kolm compile →

3 · Ship

Run the artifact on your hardware. Build llama.cpp from source for the exact target you need: cuda, rocm, vulkan, metal, cpu.

kolm runtime →

4 · Audit

Every call carries a receipt. Drift detection + supersession + signed eval holdouts mean a frozen artifact stays honest.

Use cases

kolm.ai is shape-agnostic — anywhere you call a model, kolm can compile a specialized version. Common starting points:

  • Healthcare — PHI-safe distillation, HIPAA Safe Harbor receipts.
  • Health insurance — EDI 837/835, denial letters, prior-auth.
  • Finance — PCI-aware extraction, KYC, fraud-narrative classification.
  • All use cases →

Why the artifact matters

A .kolm file is the portable, signed, frozen-eval-gated unit of a compiled AI. It carries the spec, the seeds, the eval-split, the recipe, the external + adversarial holdouts, the tenant shadow corpus, the auditor attestation, and the supersession chain — in one file you can run, ship, or hand to procurement.
