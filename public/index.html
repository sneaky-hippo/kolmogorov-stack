01position

The layer above device runtimes.

Core ML, LiteRT, ONNX Runtime, and ExecuTorch execute models. kolm packages the task, policy, recall, verifier, and receipt chain so regulated teams can decide what is allowed to ship before it reaches a phone, browser, server, or edge box.

03why teams buy it

Turn legal review into a build artifact.

The technical wedge is simple: if the sensitive data cannot leave the boundary, the AI feature has to become portable, inspectable, and verifiable.

Compile in the boundary.

Use approved examples, private policy, and regulated context without sending runtime user data to a cloud model.

Ship a signed artifact.

The artifact carries a manifest, verifier, policy, hashes, and a release score that can be checked after deployment.

Run where the data lives.

Phones, browsers, desktops, servers, and edge boxes can execute the same task contract without a runtime data exhaust.

04compile

Cache the expensive intelligence once.

The compiler turns a task into a releaseable unit: examples become tests, private context becomes recall, policy becomes a runtime guard, and the approved behavior becomes the cache.

patient-intake.task approved task contract 
task: classify incoming patient messages
labels: urgent | billing | schedule | follow-up
examples: ./labels.jsonl
corpus: ./clinical-policy/
constraints:
  runtime_egress: 0
  k_score_min: 0.90
  receipt: required
$ kolm compile "classify patient messages" \
  --examples ./labels.jsonl \
  --data ./clinical-policy \
  --base-model qwen2.5-coder-7b-instruct-q4_0

ingest   1,284 policy chunks indexed
verify   412 examples accepted
package  manifest, verifier, recall, policy
sign     hmac-sha256 receipt chain

ok wrote patient-intake.kolm
ok K-score 0.92 / release gate 0.90
05primitives

One artifact. Four jobs.

The product should feel like one command, not a stack of ML chores. The internal pieces stay visible enough for engineering, security, and legal to trust the build.

$ kolm compile

Freeze the task.

Capture the task, examples, base model, policy, and corpus namespace in a signed manifest.

$ kolm compile --data

Attach context.

Package searchable context with the artifact so local runs can cite the approved source material.

$ kolm eval

Gate the release.

Re-run embedded tests, recompute K-score, and reject artifacts that fall under the threshold.

$ kolm run

Execute locally.

Run the artifact in the product boundary and emit a receipt that can be checked later.

07proof
