One command. Then kolm compile.

No GPU, no Docker, no setup. The CLI handles the cloud compile, the local cache, and the MCP serve. The artifact you produce runs unchanged on every supported runtime.

Drop the artifact in your app
// node, web, edge — same artifact import { Kolm } from "@kolmogorov/kolm"; const k = await Kolm.load("./support-triage.kolm"); const r = await k.run({ subject, body }); // r.output, r.score, r.receipt
// iOS · Apple Neural Engine import Kolm let k = try Kolm.load(name: "support-triage") let r = try await k.run(["subject": subject, "body": body]) // r.output, r.score, r.receipt
// Android · Snapdragon Hexagon / NPU import ai.kolm.Kolm val k = Kolm.load("support-triage.kolm") val r = k.run(mapOf("subject" to subject, "body" to body)) // r.output, r.score, r.receipt
# server, agent, batch from kolm import Kolm k = Kolm.load("support-triage.kolm") r = k.run(subject=subject, body=body) # r.output, r.score, r.receipt
03Anatomy

What's inside a .kolm.

A single signed zip. Everything the artifact needs to run, packaged once, content-addressed, verifiable offline. The frontier models that produced it are the compiler; the file itself is the binary.

  • model.gguf
    The base. A small open-weight model (Qwen, Llama, Phi, Hermes) at INT4. The ground the artifact stands on.
  • lora.bin
    The personality. A LoRA adapter distilled from k-sample frontier output verified by your tests. Up to 100 MB.
  • recipes.json
    The shortcut. Deterministic-token drafts that cover the structured part of the task. Speculative decoding without a draft model.
  • index.sqlite-vec
    The corpus. A multimodal vector index of your data (text, images, audio, PDFs) that grounds every call.
  • tests.jsonl
    The contract. The synthesized verifier and the held-out evaluation set the artifact had to pass before it shipped.
  • manifest.json
    The header. Task description, base model hash, recipe registry version, K-score breakdown.
  • signature.sig
    The seal. HMAC chain over every component, anchored to the public Kolmogorov registry. Tampering is detectable.

06K-score

07Compounding

Every compile makes the next one cheaper.

kolm watches the live MCP traffic. Cache hits get faster and free. Inputs outside coverage go back to the frontier and the next compile absorbs them. The cache strictly grows.

01

First call

Frontier model serves it. Receipt is recorded. Inputs and outputs land in the corpus.

02

Repeat call

Cache hit. Local artifact answers in ~80 ms. The new receipt chains to the original.

03

Drift detected

Verifier flags an input outside coverage. Frontier resolves it. Coverage gap is logged.

04

Recompile

Next nightly run absorbs the gap. K-score ticks up. The cache strictly grows.

08Moat

What gets harder for everyone else as kolm grows.

Five compounding layers. None of them is a feature; each one strengthens the next. The compiler is what's hard to copy.

Layer 01

Compilation network effects.

The recipe registry indexes deterministic-token patterns by embedding-shape. More compiles bend the curve: faster builds, smaller artifacts, more cache coverage on every future compile.

Layer 02

Artifact format gravity.

Once your pipeline emits .kolm files, switching means recompiling and re-testing every artifact. The format is open (RS-1, MIT). The compiler that produces it is ours.

Layer 03

Compliance posture.

BAA, SOC 2 Type II, on-prem mirror, optional on-chain anchor. Procurement in healthcare, finance, and defense has legal requirements that select for the only certified option.

Layer 04

Runtime embedding.

The kolm runtime ships inside the app, the device, the equipment. Removing it means a new release. Every shipped product extends the install base for years.

Layer 05

Developer mindshare.

Open spec (RS-1), open registry, open SDK. The first format with a K-score on the cover. The community we earn becomes the default credentialing system for on-device AI.
