What ships in v0.2 today.

Four new pillars land in this release. Each one is real code with a passing test file you can run today; nothing here is a roadmap claim. Compiled LoRA, INT4, ONNX, and GGUF bundling are on the gates list, not in this build.

  1. kolm moe compose

    Mixture-of-experts, kolm-style.

    Combine N expert .kolm files into one composite with a deterministic router (keyword regex, intent field, or first-match). Per-expert sha256 and the router spec live in the receipt chain.

    tests/wave144-moe-compose.test.js

  2. kolm tokenize

    Pure-JS byte-level BPE.

    Train, encode, decode, inspect. Round-trips any UTF-8 input via the byte-level fallback. kolm compile --tokenizer tokenizer.json embeds it inside the artifact; the manifest records its spec, vocab size, and sha256.

    tests/wave144-tokenizer*.test.js

  3. kolm extract

    Text out of anything.

    Plain text, JSON, JSONL, HTML, and a built-in PDF text-layer extractor that inflates FlateDecode streams. Images opt in via --ocr (tesseract) or --vision (Anthropic). Zero native deps for the base path.

    tests/wave144-extract.test.js

  4. kolm doc check

    Completeness gate for documents.

    Built-in specs for claim packets, denial letters, prior-auth requests, EOBs, and appeal letters. Verdict pass / warn / fail; exit 0 on pass-or-warn, exit 2 on fail. Composable in any shell pipeline or CI gate.

    tests/wave144-doc-check.test.js

Common questions.

The six things people ask before they try it.

Can I improve my AI over time?
Yes. Capture real use (with consent), recompile with better examples. kolm grades each candidate against a held-out test set and refuses to ship a regression, so each version meets or beats the last. how the loop works →
Does it work with Claude Code, Cursor, Continue?
Yes. Run kolm install <harness> and your .kolm shows up as an MCP tool inside the agent. Stdio by default; HTTP optional. install the harness →
What is actually inside the .kolm file?
Your small model, your examples, the deterministic recipes, and an evaluator that grades it on held-out cases. One zip-format artifact, fully open spec. anatomy of a .kolm →
What if I want to leave kolm later?
The file is yours. The runtime is MIT. The spec is open. Run it forever without us, on any hardware you control. rs-1 open spec →
Does my data leave the device?
Inference is local. The deterministic recipe runtime is in-process and the benchmark harness ships a zero-egress monitor. Compilation happens against your approved data; we publish a threat model that spells out exactly what each step touches. threat model →
What does it cost?
The CLI runtime is free and MIT. Cloud compile + registry starts at $9/mo; team and enterprise tiers add seats, BAA, and SLAs. see pricing →
