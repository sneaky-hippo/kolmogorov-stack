The CLI quickstart for kolm.ai · the AI compiler

Five commands. Signup, login, capture, compile, verify. No editor switch, no web UI, no SaaS handoff. Every command exits with a code your shell can branch on, every artifact carries a signed receipt your auditor can replay, and the whole flow runs on your hardware.

target time: 90 seconds · prerequisites: node 18+ · works offline after first login

Install

One npm install via the scoped tarball. The CLI ships from the same package the device shell uses, so the SDK and the CLI agree on every wire format.

$ npm i -g @kolm/cli
$ kolm version
kolm 7.2.4 (node 20.10.0, darwin-arm64)

Five-step flow

step 1Sign upkolm signup

Provisions an anonymous tenant with a free quota. Stashes the key in ~/.kolm/config.json at 0600. With --email you also get a paid plan upgrade URL.

$ kolm signup --email you@example.com
tenant:  ten_yc8e9f2a   plan: free   quota: 10000   base: https://kolm.ai
key:     ks_test_abc...4321
step 2Verify identity & quotakolm whoami --json

Anytime you want to know which tenant the local CLI is acting as, ask. The --json envelope is the stable script-parseable shape: {logged_in, base, cli_version, key_fingerprint, tenant{id,plan,quota,seats,email}, _raw}.

$ kolm whoami --json | jq .tenant
{
  "id": "ten_yc8e9f2a",
  "plan": "free",
  "quota": 10000,
  "seats": 1,
  "email": "you@example.com"
}
step 3Capture from your real traffickolm tail captures

Point any existing OpenAI-compatible client at the kolm proxy base. Every prompt+response pair lands in your tenant's durable capture store and streams live to your terminal.

$ export OPENAI_BASE_URL=https://kolm.ai/v1/bridges/openai
$ export OPENAI_API_KEY=$(kolm key fingerprint --raw)
$ kolm tail captures --namespace summarizer
[live] cap_018a3e... summarizer    qwen-3.5-27b   234ms  "Summarize this PR:..." -> "This PR refactors..."
step 4Compile from captureskolm distill --from-captures

Once a namespace crosses ~1,000 captured pairs (kolm tells you), compile them into a .kolm artifact. The compiler enforces the receipt chain (spec → seeds → split → train → recipes → evals → export → signatures → rekor) and refuses to ship if any link is missing.

$ kolm distill --from-captures summarizer --tier=3090
job_w8e2 distill running. 64 GPU-min budget. follow with: kolm watch job_w8e2
$ kolm watch job_w8e2
[done] artifact_summarizer_v3.kolm  k_score=0.91  size=148MB  receipts:9/9 signed
step 5Verify and runkolm verify && kolm run

Verify replays the receipt chain offline against the artifact bytes — never trust an artifact you cannot verify. Run dispatches the artifact to the right runtime (JS, WASM, native, GGUF, or ONNX) based on what is inside.

$ kolm verify artifact_summarizer_v3.kolm
verify OK. 9/9 receipts signed, 0 drift breach, k_score 0.91 within tolerance.
$ kolm run artifact_summarizer_v3.kolm --prompt "Summarize this PR:..."
This PR refactors the auth middleware to fail-closed on token expiry...
