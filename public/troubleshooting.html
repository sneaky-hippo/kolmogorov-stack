troubleshooting

When kolm refuses to ship.

Most failures fall into one of nine categories. The compiler is strict on purpose: a signed artifact that misses its quality gate is worse than no artifact. Each row below names the symptom, the cause, and the exact fix. If your problem is not here, open an issue with the compile id.

error reference status open an issue
01compile failed

The job ran, but the K-score didn't pass.

The compiler refused to sign the artifact because at least one gate (accuracy, coverage, size, p50 latency) fell below the floor. Three causes account for almost every case.

too few examplesThe verifier could not synthesize a stable test trace. Add 6–12 more positive examples and 2–4 negatives. Re-run kolm compile.
examples disagreeTwo examples imply opposite outputs for inputs that look similar. Run kolm doctor <job_id> to see which pair caused the contradiction. Resolve and re-submit.
output_spec too tightSchema rejected most teacher samples. Loosen the schema or remove an unused required field. The compile log shows which path failed validation most often.
02k-score under floor

Score is 0.62, you need 0.74.

Ship gate is composite. You can pass three sub-gates and still fail if one is under floor.

$ kolm score summary.kolm
K-score          0.62  blocked
  accuracy       0.84  ok      >= 0.80
  coverage       0.71  ok      >= 0.65
  size_bytes     0.93  ok      >= 0.50
  p50_latency_us 0.18  failed  >= 0.40    <-- fix this

Latency floor failed. Either pick a smaller base model with --base qwen2.5-3b-instruct, or reduce recipe-pack depth with --recipe-pack-depth 32. Re-run; the receipt chain regenerates from the new pack.

03signature drift

The artifact verifies locally but not on the registry.

A receipt mismatched its registry anchor. Three sub-cases.

RECIPE_RECEIPT_SECRET unsetServer is signing with the dev fallback. Set RECIPE_RECEIPT_SECRET on Railway, redeploy, recompile. The CLI returns 503 not_configured until set.
local clock driftReceipts include issued_at. If your machine is >5min off NTP the registry rejects on submit. Sync system time and retry.
SDK version too oldPre-v0.1.0 receipts had a different chain shape. npm i -g github:sneaky-hippo/kolmogorov-stack@latest and recompile. Old artifacts stay valid; only fresh submissions need the new chain.
04base-model mismatch

"Manifest declares qwen2.5-7b but loader has qwen2.5-3b."

The artifact's manifest.json/base_model field does not match the local runtime. The .kolm doesn't ship weights; it ships a pointer plus a LoRA delta. The base must be present locally.

$ kolm doctor summary.kolm
manifest.base_model:  qwen2.5-7b-instruct@hf:Qwen/Qwen2.5-7B-Instruct
local cache:          qwen2.5-3b-instruct  (mismatch)
fix:                  kolm pull qwen2.5-7b-instruct
                      # or recompile with --base qwen2.5-3b-instruct
05quota exhausted

You hit the monthly compile cap.

Free is 1 compile/month, Pro is 25, Teams is 10/seat. The CLI returns 403 quota_exhausted with a reset date.

$ kolm compile examples.jsonl
error: quota_exhausted (resets 2026-06-01)
plan:  pro
used:  25 of 25

Upgrade at /pricing, or wait for the reset. Compiles do not roll over.

06auth fails locally

"Invalid API key" from a key you just created.

Two causes account for ~95% of these.

wrong base URLCLI defaults to https://kolm.ai. If you previously set kolm config base http://localhost:8787, the new key from prod won't work locally. Run kolm config base https://kolm.ai && kolm login.
shell ate the prefixKeys start with ks_. If you pasted into .bashrc with export KOLM_KEY=ks_... without quotes, the underscore-prefixed value can hit shell expansion on some setups. Quote it: export KOLM_KEY="ks_...".
07kolm serve refuses to start

MCP server fails the bind step.

Three usual suspects.

port already in useDefault 11455 conflicts with a stale instance. kolm serve --mcp --http --port 11456. Or kill the old: lsof -i :11455 && kill <pid>.
artifact not in cachekolm serve reads from ~/.kolm/artifacts. If you compiled on another machine, copy the .kolm in or pass --artifact ./summary.kolm directly.
signature invalidIf the file was modified after sign, serve refuses. kolm verify summary.kolm will tell you which member is dirty.
08verify offline

You want to verify a .kolm with no network.

The signature chain is HMAC-SHA256 with the registry's public anchor cached locally. kolm verify --offline uses the cached anchor only; if the artifact references a registry version newer than your last kolm pull anchors, it returns warn: anchor_stale but still tells you whether the chain is internally consistent.

$ kolm verify --offline summary.kolm
manifest          ok    sha256:1f2e…
recipes.json      ok    sha256:7a4c…
lora.bin          ok    sha256:9b11…
index.sqlite-vec  ok    sha256:0d56…
receipt chain     ok    chained-from registry@v2026-04-30
anchor            warn  cached 9d ago, run `kolm pull anchors`
09still stuck

Open an issue with the compile id.

If your symptom is not above, paste the compile id from kolm compile's output and the last 50 lines of ~/.kolm/log/compile.log into a GitHub issue. We answer within one business day.
