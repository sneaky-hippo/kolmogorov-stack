04privilege

Why the receipt matters at deposition.

A signed receipt is the firm's record that this artifact was the one that produced this output for this input on this date. It binds the model state, the recall sources, and the input/output hashes into a single chain. If opposing counsel asks how the AI was used, the firm produces the artifact and the receipt. Reproducibility is the audit defense.

artifact hashsha256 of the .kolm zip. Pinned in the firm's matter management system at deployment. Cannot drift.
recall provenanceEvery recall hit is content-addressed. The receipt names which playbook section was retrieved.
input/output hashesHMAC-chained. Tampering with output after the fact breaks the chain on verify.
retention policyReceipts are local. Firm retention rules apply. No third-party archives to subpoena.

06why legal AI is different

Three problems cloud AI cannot solve for a firm.

The reason most firms have written cloud-LLM bans into their information security policy is not vendor friction. It is the structural mismatch between Model Rule 1.6 confidentiality and a third-party inference API that logs prompts.

ABA Model Rule 1.6 confidentialityA lawyer must not reveal information relating to representation. A cloud prompt is a disclosure to a third-party processor, and most firms cannot get the informed consent required for every matter touched. Architecture that keeps document text on firm hardware sidesteps the consent question entirely.
Work-product doctrineMental impressions, legal theories, and case strategy live in the prompts associates write. If those prompts persist on a vendor's logging plane, the firm has a discoverability problem that did not exist before the tool was adopted. A local artifact has no remote log.
Conflict and matter wallsInformation barriers between matters are an ethical obligation, not a feature flag. Cloud inference does not respect firm walls by default. A compiled artifact per matter type runs only against the documents the lawyer hands it and writes its receipt to the firm's matter management system.
07how kolm helps

Three things a signed artifact gives the firm.

Each is a property of the compile output, not a configuration setting an associate could turn off.

No document text leaves the firm at run timeThe .kolm runs on the partner's laptop or the firm's secure compute. Counterparty drafts, deposition transcripts, and matter files stay on the device that opened them. There is no inference API to log, throttle, or subpoena.
Frozen model state, defensible at auditThe model weights, recall index, and recipe pack are pinned inside the artifact at compile time. The firm controls when the artifact updates. If an opposing counsel asks which model produced a memo, the answer is a sha256 hash with a date, not a vendor release note.
HMAC receipt per outputEvery run writes a content-addressed receipt: input hash, output hash, recall sources, model identity. The receipt lives in the firm's matter management system. Re-verification is byte-exact on the same artifact.
08a matter walkthrough

NDA redline, from intake to signed output.

Concrete example. A senior associate receives an inbound NDA from a counterparty on a corporate matter. The firm has a redline policy compiled as nda-redline-v2026.02.kolm. Here is the run.

01 intakeAssociate opens the matter file in their document management system. The counterparty draft (12.4 KB .docx) is checked into the matter. Conflict check is already cleared by the firm's intake workflow. No documents have left the firm boundary.
02 compileAlready done at the firm level, last quarter. The redline policy was compiled from 142 historical NDAs reviewed and signed off by the corporate practice group. K-score 0.81 cleared the ship floor of 0.74. The artifact is pinned in the firm's MDM and distributed to every partner laptop.
03 reviewAssociate runs kolm run nda-redline-v2026.02.kolm --in counterparty-draft.docx --out redlined.docx. The artifact returns a tracked-changes file with 23 markup proposals: indemnity scope tightened, IP assignment carve-out added, MFN deleted, term capped at 24 months. Recall surfaces the playbook section behind each change. Runtime is 14.2 seconds on the partner's M3 Pro. Zero network calls.
04 shipPartner reviews each tracked change, accepts or rejects as a human lawyer, and sends the redlined draft back to the counterparty under cover of the firm's letterhead. The receipt (receipt-9b110d56.json) attaches to the matter file. If the deal closes and the agreement is later challenged, the firm can re-verify which policy version produced which markup on which date.
09common questions

What partners and GCs ask first.

Direct answers. If the answer is "depends on the firm's GC," we say that.

Does running kolm on client documents waive privilege?That is a determination for the firm's GC and ethics counsel. The architecture is designed to avoid the third-party-disclosure event that most privilege-waiver analyses flag. The .kolm runs on firm-controlled hardware. No document text is transmitted to a third-party API at run time. The receipt is local. We provide the architecture and the audit trail. The privilege opinion is the firm's call.
Can the firm use this for matters under a protective order?Most protective orders restrict disclosure to third parties and require secure handling. A local artifact does not transmit document content to anyone. The firm should still review the specific order language with the GC. We provide a deployment that produces receipts the firm can attach to the matter record.
What about discovery review? Can opposing counsel demand the model?The firm produces what was used. Because every run writes a receipt with the artifact's sha256, the firm can identify exactly which model state generated which output and provide that artifact for inspection if required. The model is not a remote service the firm cannot freeze in time.
Does the firm need to train its own model?No. Compile uses a base model the firm selects (Qwen, Llama, Mistral, or a frontier model via API key at compile time only). The firm's gold examples and playbook tune the artifact during specialization. The teacher model sees the training set during compile, not production matter at run time.
How does this fit conflict checks and matter walls?Per-matter or per-practice-group artifacts are the default deployment pattern. A corporate redline artifact never sees litigation files; a litigation discovery artifact never sees corporate matter. Distribution is handled by the firm's MDM. The artifact has no cross-matter memory because it has no remote storage.
What does the firm do when a frontier model deprecates?Nothing immediately. The artifact's behavior is frozen at the version compiled. Frontier deprecations affect future compiles, not in-flight artifacts. The firm decides when to recompile. Old artifacts remain re-verifiable indefinitely because the model state is inside the .kolm.
