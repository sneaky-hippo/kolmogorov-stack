Where kolm plugs in.

One CLI. Ten surfaces. kolm compile works the same way from Claude Desktop, Cursor, VS Code, your CI pipeline, your Python REPL, and the bare terminal. The artifact it produces is the same byte-exact .kolm in every case.

Agents & editors

If your agent speaks MCP, it can drive kolm. If your editor speaks Cursor rules or Claude Code skills, it can drive kolm.

Claude Code (MCP)

shipped

Run kolm install claude-code --apply and the installer writes mcpServers.kolm into ~/.claude/settings.json. Claude can then synthesize, run, and inspect recipes as MCP tools over stdio - no manual JSON editing.

npm i -g github:sneaky-hippo/kolmogorov-stack
kolm install claude-code --apply
# writes ~/.claude/settings.json mcpServers.kolm: { command: "kolm", args: ["serve","--mcp"] }

CI / CD

Compile on every push. Gate the build on K-score. Publish the artifact as a build attachment.

GitHub Actions

shipped

Composite action at .github/actions/kolm-compile. Drops into any workflow; uploads the .kolm as a build artifact.

- uses: sneaky-hippo/kolmogorov-stack/.github/actions/kolm-compile@main
  with:
    task: 'classify support tickets'
    examples: ./labels.jsonl
    api-key: ${{ secrets.KOLM_API_KEY }}
source

GitLab CI / CircleCI / Buildkite

shipped

Agent frameworks

First-class LangChain and LlamaIndex adapters. Same .kolm artifact, same receipt chain, just wired in the framework your agent already lives in. Every call returns the receipt alongside the text so audit, drift detection, and replay work the same as in the bare CLI.

LangChain.js

shipped

Drop-in KolmLLM that satisfies the LangChain LLM interface. invokeWithReceipt(prompt) returns { text, receipt } so chains keep the receipt chain intact. Subprocess mode for local .kolm files, HTTP mode for hosted artifacts.

npm i @kolm/langchain
import { KolmLLM } from '@kolm/langchain';
const llm = new KolmLLM({ artifactPath: './ticket-classifier.kolm' });
const { text, receipt } = await llm.invokeWithReceipt('refund #1234');
walkthroughreceipt schema

LangChain (Python)

shipped

Same surface, Pythonic. invoke_with_receipt(prompt) returns a tuple. Works in LangGraph nodes, LangServe handlers, and plain scripts.

pip install kolm-langchain
from kolm_langchain import KolmLLM
llm = KolmLLM(artifact_path='./ticket-classifier.kolm')
text, receipt = llm.invoke_with_receipt('refund #1234')
walkthroughreceipt schema

LlamaIndex.TS

shipped

Implements the LlamaIndex LLM protocol. complete(prompt) returns { text, raw } where raw.receipt carries the kolm receipt chain. Ready to plug into a QueryEngine.

npm i @kolm/llamaindex
import { KolmLLM } from '@kolm/llamaindex';
const llm = new KolmLLM({ baseUrl: 'https://kolm.example', apiKey: 'k_...' });
const r = await llm.complete('summarize this ticket');
walkthrough

LlamaIndex (Python)

shipped

Python LlamaIndex adapter exposing the same invoke_with_receipt surface as the LangChain one. Lets QueryEngines and SubQuestion agents speak to a compiled .kolm without losing the audit trail.

pip install kolm-llamaindex
from kolm_llamaindex import KolmLLM
llm = KolmLLM(artifact_path='./classifier.kolm')
r = llm.complete('summarize this ticket')  # r.raw.receipt is the kolm receipt

Workflow automation

No-code triggers and downstream wiring. The MCP-over-HTTP shim already covers webhooks; native Zapier and Make.com modules ship next quarter and will surface the same receipt chain that the CLI does.

Zapier app

coming Q3 2026

Native Zapier app with two actions: "Run a kolm artifact" and "Compile a new kolm from captured calls". Output payload includes text, k_score, and the full receipt chain so Zaps can branch on quality. Until then, the same flow works via the kolm HTTP API as a generic webhook in Zapier.

# Zapier webhook today (coming Q3 2026 as a native app):
POST https://api.kolm.ai/v1/run
authorization: Bearer $KOLM_API_KEY
{ "artifact": "ticket-classifier", "input": "{{trigger.body}}" }
