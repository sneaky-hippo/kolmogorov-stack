kolm  /  migrate  /  from OpenPipe

Replace OpenPipe's capture-then-hosted-endpoint with kolm capture-then-compile.

OpenPipe wraps your OpenAI / Anthropic SDK calls and logs every prompt/completion pair to its backend, fine-tunes from the logged traffic, then serves the result as a hosted endpoint. The capture loop is the right shape. The output shape is what changes: kolm captures the same traffic into local seeds.jsonl and compiles a signed artifact you keep. Same teacher, same student base, same eval, same auto-fine-tune trigger. The deliverable is a file with a receipt instead of a URL with a usage meter.

Wave 188 · from OpenPipe · capture-loop migration

The two capture loops, side by side.

OpenPipe today: SDK wrap→OpenPipe backend→auto fine-tune→hosted endpoint URL
kolm tomorrow: kolm capture→local seeds.jsonl→kolm distill→signed .kolm file

The input traffic looks identical. What changes is where the captured pairs land (your disk vs OpenPipe's DB), where the fine-tune runs (your GPU or one-line cloud worker vs OpenPipe-managed), and what you receive (a portable file vs a hosted endpoint). Your existing OpenPipe-side data exports cleanly; the migration is one CLI session.

The four-step migration.

01 · flip the capture

Replace the OpenPipe SDK wrap with kolm capture.

OpenPipe's SDK wrap is a drop-in for the OpenAI / Anthropic client. kolm capture is the same shape: same base URL pattern, same auth header forwarding, but the pairs land in a local seeds.jsonl namespace instead of OpenPipe's hosted DB.

# one-line proxy that captures every pair into a namespace
$ kolm capture \
    --provider openai \
    --as ticket-classifier \
    --namespace tickets

  LISTEN 127.0.0.1:7117
  capturing → ~/.kolm/captures/tickets/seeds.jsonl
  threshold:  1000 pairs · required before distill unlocks
02 · import historical data

Bring your OpenPipe project into seeds.jsonl.

OpenPipe lets you export a project's dataset as JSONL. kolm reads the canonical {"input","output","tags"}; a rename step gets you there.

# export your OpenPipe project
$ openpipe dataset export \
    --project my-tickets > openpipe.jsonl

# reshape into the canonical kolm format
$ kolm import openpipe \
    --in openpipe.jsonl --out seeds.jsonl
  shipped
  WROTE 12847 rows
03 · distill on your terms

kolm distill swaps in for the OpenPipe auto-tune.

Same teacher you were already paying for — OpenAI or Anthropic API call. Same student base — Llama 3.x family OpenPipe leans on. Local GPU or one-line cloud worker; deterministic flags pinned into the manifest.

$ kolm distill \
    --namespace tickets \
    --teacher=openai:gpt-5 \
    --student-base=llama-3.2-1b \
    --redact=phi --eval-gate=0.85

  PASS  K-score 0.892 · train/holdout disjoint
  PASS  PHI redactor · no leakage
  WROTE ticket-classifier.kolm · 184 MB
04 · verify and ship

The deliverable is a file, not a URL.

Run kolm verify once to confirm 24/24 checks. Then run the artifact wherever you want — local inference, your own LoRAX cluster, BYO serving. No OpenPipe meter ticking.

$ kolm verify ticket-classifier.kolm
  PASS  24/24 checks
  PASS  Ed25519 + Sigstore validated

# run locally, no network required
$ kolm run ticket-classifier.kolm \
    --input "Refund request for order #12345..."
  ⇒ {"intent":"refund","priority":"high"}

What survives, what kolm adds.

Survives the migration unchanged

  • Capture-loop pattern. Same drop-in client wrap idea OpenPipe pioneered, same per-call logging.
  • Teacher choice. Whichever OpenAI or Anthropic SKU you were already calling stays the --teacher.
  • Student base family. The Llama 3.x bases OpenPipe defaults to are first-class kolm --student-base options.
  • Train / eval split logic. OpenPipe's holdout becomes kolm's frozen holdout. The numerics carry over.
  • Auto-trigger threshold. OpenPipe's "fine-tune when N pairs captured" is mirrored by kolm's distill threshold (default 1000).

kolm adds on top new

  • Local artifact. The deliverable is a file on disk, not a hosted endpoint.
  • Air-gap deploy. Verifier runs offline; runtime has no network dependency.
  • PHI redactor (Wave 157). 18 HIPAA Safe Harbor + 3 kolm extensions before the teacher call.
  • K-score gate with external + adversarial holdouts (W164). Eval signal isn't just captured-traffic accuracy.
  • Signed receipt chain. Ed25519 (W161) + Sigstore / Rekor (W162). Auditor walks the chain offline.

Where staying with OpenPipe is the right answer.

Stay with OpenPipe when the deliverable you actually want is a hosted endpoint URL, not a file. OpenPipe's auto-fine-tune-on-N-pairs trigger plus their hosted serving is the shortest path from production traffic to a cheaper-than-GPT endpoint. If no one on your team needs to mail the artifact to a phone, an edge box, or an air-gapped VPC, the file format earns nothing.

Stay with OpenPipe if the RL-for-Agents pivot they shipped in 2025 is what you came for — reinforcement-learning agent fine-tuning is orthogonal to kolm's capture-then-distill flow. The 2025-2026 OpenPipe roadmap is genuinely interesting in that lane and kolm is silent there.
