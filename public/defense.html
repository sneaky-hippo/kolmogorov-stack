01Why this matters

Cloud AI on classified work is
a non-starter.

Disconnected teams need local inference, deterministic evidence, and a transfer story that can be reviewed by security. Cloud APIs alone do not solve classification, data-flow, or authorization questions.

Cross-domain
Zero egress

often permitted from sensitive networks. kolm's signed artifact format gives reviewers a concrete object to inspect before transfer.

Tactical edge
No link

guaranteed at the forward node. kolm's target architecture is local execution with no cloud dependency; every deployment still needs hardware proof.

Provenance
Every byte

requires attestation. HMAC-chained manifest covers base model hash, adapter delta, recipe pack, recall index, evals.

02How it works

Compile on the low side.
Run on the high side.

A customer-led pilot can compile from approved low-side examples, emit a signed artifact, and verify that artifact before any transfer. High-side corpus handling, cross-domain movement, and authorization remain under the customer's controls.

-

Open-weight base policy.

Regulated deployments should review every model license, export classification, and weight source before packaging an artifact.

-

Signed transfer object.

Artifact is one file. Manifest hash, recipe hash, eval hash, and receipt hash are bound by HMAC today; model and adapter hashes harden further as the specialization bridge matures.

-

Edge target requirements.

Each hardware path needs named-device benchmarks for latency, memory, thermal behavior, and fallback before it is represented as production-ready.

03Use case

Reference deployment:
analyst assistant in an enclave.

A 2.1 GB artifact, base model plus 84 MB adapter, on a workstation inside an enclave. Indexes the analyst's own document set locally, drafts in the institution's voice, defers anything the verifier rejects. No external link, no cloud, no telemetry.

compile pilot · customer-controlled network · signed artifact

From an unclassified gold set to a signed binary on the workstation.

Compile can happen from an approved gold set. The artifact is signed, K-scored, written to media, and verified by the customer before deployment. The signature chain detects modification; it does not replace an authorization package.

01 - gather

Unclassified gold set

Scrubbed example pairs and held-out evals on the low side. Frontier teacher reachable through your egress proxy.

02 - compile

kolm compile

Synthesize a verifier from examples. Specialize an open-weight base on your data. Index your private context. Sign.

03 - transfer

Signed sneakernet

Artifact written to write-once media. Hash chain printed for cross-domain ATO. Verified on arrival, signature matches, deploy.

04 - run

Detached operation

Workstation runs offline. Receipts log to local audit. Evidence packets exportable on demand for inspection.
