Typed output specs

An output spec is a machine-readable description of what shape a response must take: JSON schema for structured outputs, OpenAPI for tool-calling, Pydantic or TypeScript types for SDK-style consumers. A spec lets the trainer score conformance directly, the runtime constrain decoding, and the auditor verify that a captured response satisfied the contract the buyer signed.

What it is

Treating model outputs as a typed object goes back to function-calling in OpenAI's June 2023 release and earlier to the structured-prediction literature. JSON Schema (draft 2020-12) is the dominant lingua franca; OpenAPI 3.1 wraps it for tool definitions. At inference time the spec becomes a grammar: token-level constrained decoding via finite-state machines (Willard and Louf's Outlines, 2023; Dong et al. XGrammar, 2024) guarantees the output parses without the latency overhead of post-hoc retries. At training time the spec becomes a reward: shape-conformance pass rate is a verifiable signal that can drive RL (the BFCL benchmark, Yan et al. 2024, scores tool calls this way). At gate time the spec becomes a contract: the K-score includes a shape-conformance component the auditor can replay against the same schema.

How kolm uses it

Output specs are first-class in kolm v0.2.6. The recipe declares a output_spec block (JSON schema, OpenAPI fragment, or named built-in like email_address or iso8601_date). The capture proxy validates every captured response against the spec and tags non-conforming rows so the trainer can either drop them or learn to repair them. The trainer runs SFT on the parsed-and-reformatted responses, so the student learns the canonical shape, not whatever variant the teacher produced. The gate adds a shape-conformance score to the K-score weighted by the recipe's shape_weight. Constrained decoding at runtime is opt-in via spec_enforce: true; the default is unconstrained sampling because constrained decoding can interact badly with prompt formats the student learned during distillation. The receipt records the spec hash, the conformance rate on the holdout, and which decoding mode was used.
