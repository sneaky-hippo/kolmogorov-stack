Holdout set disjointness

A holdout is a fraction of labelled examples that the trainer never sees and the gate uses to grade. The grade is only meaningful if the holdout stays disjoint from the training corpus at every level the model can memorize: exact byte, near-duplicate, n-gram overlap, and semantic neighborhood. Kolm checks all four every time it grades.

What it is

The textbook definition of generalization error requires train and test draws to be independent. In practice every contemporary dataset suffers from some leakage: web crawls inherit duplicates, synthetic-data pipelines memorize their seeds, judge models echo their training set. The literature on benchmark contamination (Sainz et al., 2023; Zhou et al., 2023) finds measurable inflation on most public benchmarks once contamination is corrected for. The standard mitigation stack is byte-equality (the cheapest), MinHash-LSH near-duplicate detection (Broder, 1997), n-gram containment (the OpenAI evals heuristic uses 13-grams), and embedding cosine-distance thresholds in a frontier encoder space. Each check catches a different leakage mode; running only the cheapest understates contamination.

How kolm uses it

The gate stage in v0.2.6 runs three disjointness checks before emitting a K-score: SHA-256 byte-equality (rejects identical rows), MinHash with Jaccard threshold 0.85 (rejects near-duplicates), and 13-gram containment with a 50% threshold (matches the OpenAI evals contamination heuristic). Embedding-space disjointness is on the v0.3 roadmap; the current implementation uses the cheaper KL surrogate described in /research/eval-drift. The holdout itself is regenerated at every compile from the latest seed prompts, never recycled across builds, and its hash is recorded in the receipt so an auditor can re-verify the disjointness check offline. Rows that fail any check are quarantined and reported, not silently dropped.
