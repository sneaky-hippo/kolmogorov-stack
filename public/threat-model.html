threat model

What it defends. What it doesn't.

A signed artifact is only useful if the threat scope is honest. The attacks the receipt chain catches, the ones it doesn't, and the primitives behind both.

Primitive: the receipt chain

Every .kolm artifact emits a chain of HMAC-SHA256 hashes anchored on the manifest. The verifier walks the chain backwards from output to manifest hash; any tampered link breaks verification.

// at compile time
manifest_hash   = SHA256(manifest.json)
model_hash      = SHA256(model.gguf)
lora_hash       = SHA256(lora.bin)
recipes_hash    = SHA256(recipes.json)
recall_hash     = SHA256(recall.sqlite-vec)

artifact_root   = HMAC-SHA256(secret, manifest_hash || model_hash ||
                              lora_hash || recipes_hash || recall_hash)

// at run time, every output gets a fresh receipt
output_receipt  = HMAC-SHA256(secret, artifact_root ||
                              SHA256(prompt) || SHA256(output) ||
                              timestamp || run_nonce)

// verification (v0.1: shared-secret HMAC, issuer or shared-secret holder)
// public-key (Ed25519) verifier is a roadmap tier
verify(output, receipt, artifact, secret) => ok | tampered

What kolm defends against

defends · t1

Artifact tampering

Modifying model.gguf, lora.bin, recipes.json, or recall.sqlite-vec inside a .kolm breaks the chain; kolm inspect fails immediately.

defends · t2

Output substitution

Forged outputs don't chain to a valid artifact_root; verifier rejects.

defends · t3

Manifest forgery

A fake manifest's hash diverges from artifact_root; verification fails on the first link.

defends · t4

Recall index poisoning

Editing the bundled sqlite-vec index changes recall_hash; chain breaks. (Why we ship the index inside the artifact.)

defends · t5

Replay across artifacts

A valid receipt from A copied to B fails: receipts embed artifact_root, A's root ≠ B's root.

defends · t6

Quality regression smuggling

A trojan-trained LoRA fails the per-artifact K-score gate at compile time; manifest records this artifact's K-score and held-out test set hash.

What kolm does not defend against

out of scope · n1

Model-level adversarial prompts

A jailbroken prompt will be signed as-is. Receipts prove provenance, not safety. Use guardrail layers separately.

out of scope · n2

Compromised compile environment

If the machine running kolm compile is compromised, the attacker can sign a valid artifact with poisoned weights. Use a clean build; consider air-gapped compile for high-stakes work.

out of scope · n3

Side-channel timing inference

Inference-timing attacks on shared hardware may infer input or output. Receipts are an integrity primitive, not confidentiality. Use trusted hardware (SEV, TDX) for adversarial multi-tenant.

out of scope · n4

Human at the keyboard

Receipts prove what the artifact said, not that a human acted on it correctly. Audit trail ends at the model output; downstream business logic is yours.

out of scope · n5

Recipe sandbox escapes (v0.1)

v0.1 executes JS recipes in node:vm with a frozen lib global. node:vm is defence-in-depth, not a hard security boundary. Synthesized recipes are scanned for canonical escape primitives (process, require, constructor, Function, eval) and rejected on match. For untrusted third-party recipes, run kolm under a hardened isolation layer (isolated-vm, Wasmtime, Firecracker). Roadmap: switch the default sandbox to isolated-vm.

What an attacker would have to compromise

To forge a legitimate-verifying artifact, an attacker needs simultaneously: (a) the user's signing secret or the registry's, (b) a SHA-256 second-preimage on the manifest, (c) if co-signed, a second registry compromise.

On-chain anchored artifacts (enterprise, roadmap) will also require rewriting the anchored block at chain-attack cost once rs-3 anchoring ships.

Reporting

Found a way around any of T1 through T6? Email security@kolm.ai with repro steps. Bounty brackets on trust. Coordinated disclosure: 90 days standard, 14 days for actively exploited.

Honest scope

kolm is an integrity primitive, not a safety primitive. Receipts prove what was said by which artifact, not that the output is safe or policy-aligned.

Pair with prompt-side guardrails, output filters, and downstream business logic.
