Enterprise

The enterprise tier is the same kolm.ai stack you can run on a laptop, packaged for procurement. On-prem deploy, retention policies measured in years, an uptime SLA, named support, and SSO / SCIM that fits inside an Okta or Entra org. The product surface is unchanged. The contract surface is what differs.

On-prem deploy

The enterprise tarball is the same binary that ships in the @kolm/cli npm package plus a docker-compose for the cloud-side services (sync relay, team coordinator, audit aggregator). Air-gapped installs are supported; the install bundle includes all model weights you license.

curl -O https://download.kolm.ai/enterprise/kolm-onprem-7.x.tar.gz
tar xzf kolm-onprem-7.x.tar.gz
cd kolm-onprem
./bootstrap --domain kolm.internal.example --license ./license.key

Audit log retention

The audit log default is unlimited. Enterprise customers usually pick a policy that aligns with their compliance regime.

RegimeTypical retention
HIPAA6 years
SOX7 years
GDPR (data subject access)30 days after request
SOC 21 year minimum, 7 typical

Configure with one line in ~/.kolm/config.json:

{
  "audit_retention_days": 2555
}

SLA

The hosted control plane carries a 99.9% monthly uptime SLA. Self-hosted installs do not require a control plane to operate; the proxy, lake, and runtime keep working with zero control-plane connectivity.

SSO and SCIM scaffolding

OIDC and SAML 2.0 are supported via the control plane. SCIM 2.0 provisions and deprovisions members in lockstep with your IdP.

# In ~/.kolm-control/config.json on the enterprise control plane
{
  "auth": {
    "oidc": {
      "issuer": "https://login.example.com",
      "client_id": "kolm-prod",
      "client_secret_env": "KOLM_OIDC_CLIENT_SECRET"
    },
    "scim": {
      "enabled": true,
      "bearer_env": "KOLM_SCIM_BEARER"
    }
  }
}
