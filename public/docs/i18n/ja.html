60秒でkolmをビルド

このページのすべての例は、2026-05-11に kolm.ai で実際にキャプチャされたレスポンスです。任意のブロックをコピーしてターミナルに貼り付ければ動作します。動作しない場合は issue を登録してください。

コードブロック、CLI コマンド、API エンドポイント、環境変数名、JSON キーは言語に依存しないため、英語のまま維持しています。

クイックスタート

サインアップしてキーを受け取り、2 つの例でマスキング (redaction) 関数をコンパイルし、実際の入力に対して実行します。6 行で完了します。

$ curl -sX POST https://kolm.ai/v1/signup \
    -H 'content-type: application/json' \
    -d '{"email":"you@example.com","plan":"free"}'
$ export KOLM_KEY=ks_...                            # 上記レスポンスから取得
$ curl -sX POST https://kolm.ai/v1/synthesize \
    -H "authorization: Bearer $KOLM_KEY" -H 'content-type: application/json' \
    -d '{"name":"redact","positives":[{"input":"token=sk_live_xxx","expected":"[REDACTED]"}]}'
$ curl -sX POST https://kolm.ai/v1/run \
    -H "authorization: Bearer $KOLM_KEY" -H 'content-type: application/json' \
    -d '{"concept_id":"cpt_...","input":"my token=sk_live_abc123"}'

40 マイクロ秒、HMAC 1 つ、署名済みレシート (signed receipt) 1 つ。これだけです。

認証 (Authentication)

authorization ヘッダーに Bearer トークンを使用します。トークンは POST /v1/signup または /signup から取得します。Free プランには月 10,000 回の呼び出しが含まれます。

curl -H "authorization: Bearer ks_..." https://kolm.ai/v1/account

CLI のインストール

npm i -g github:sneaky-hippo/kolmogorov-stack
kolm login

サインアップ (Sign up)

POST/v1/signupno auth
curl -X POST https://kolm.ai/v1/signup \
  -H 'content-type: application/json' \
  -d '{"email":"you@example.com","plan":"free"}'
注意: API キーは 1 度だけ表示されます。安全に保管してください。紛失時は POST /v1/account/rotate-key でローテーションできます。

Synthesize (合成)

入力/期待出力ペアのリストを、分類・マスキング・変換を行うコンパイル済み JavaScript 関数に変換します。concept_id と品質スコアを返します。

POST/v1/synthesizebearer
curl -X POST https://kolm.ai/v1/synthesize \
  -H "authorization: Bearer $KOLM_KEY" \
  -H 'content-type: application/json' \
  -d '{ "name":"redact-secrets", "positives":[ { "input":"token=sk_live_xxx", "expected":"[REDACTED]" } ] }'

K-score ゲートはデフォルト 0.85 です。ゲート未満のコンパイルは失敗した軸を明示し accepted: false を返します。例を追加するか spec を調整して再送信してください。

Run (実行)

コンパイルされた concept を新しい入力に対して実行します。出力と共に、同じキーを持つ誰もが再検証可能な署名済みレシートを返します。

POST/v1/runbearer
curl -X POST https://kolm.ai/v1/run \
  -H "authorization: Bearer $KOLM_KEY" \
  -H 'content-type: application/json' \
  -d '{ "concept_id":"cpt_mp11ol06a...", "input":"my token=sk_live_abc123" }'

レシート (Receipts)

今日提供されるもの: 各 .kolm アーティファクトは、アーカイブ内容 (manifest + レイヤーハッシュ) 全体に対する単一の HMAC-SHA256 署名を保持します。各 /v1/run レスポンスは source_hash · input_hash · output_hash · version_id · issued_at を含む _kolm メタデータブロックを返し、発行者または共有テナントレシートシークレット保有者であれば誰でも再検証できます。目標アーキテクチャ: リクエスト間の呼び出しごとの HMAC チェーンと Ed25519 公開鍵レシートはロードマップ上にあり、まだ提供されていません。

.kolm フォーマット

5 つのファイルを含む署名付き zip です: spec.json (レシピ)、policy.json (ゲート)、recall/ (オプションのサイドカー)、verifier.js (低コスト出力確認)、receipt.json (HMAC チェーン)。完全なスキーマは /spec にあります。

K-score

モデル単位ではなく、アーティファクト単位。 コンパイルされた各 .kolm は、それ自身の評価セットで自身の K-score を取得します。このスコアはそのアーティファクトを評価するものであり、その下のベースモデルを評価するものではありません。単一のスカラーで、デプロイ基準は 0.85 です。計算式は 0.40·A + 0.15·(S + L + C + V) で、A は精度 (accuracy)、S は特異度 (specificity)、L は遅延 (latency)、C はコスト (cost)、V は検証可能性 (verifiability) です。全内訳は /k-score にあります。

CLI

8 つの動詞がループ全体をカバーします。API と同じ認証を使用します。kolm login はキーを ~/.kolm/config.json に保存します。

kolm init              # kolm.yaml + examples/ をスキャフォールド
kolm compile           # synthesize · 署名 · .kolm 書き出し
kolm run file.kolm     # 実行してレシート発行
kolm tune --airgap     # キャプチャされた実行から LoRA アダプタを学習
kolm rag attach docs/  # BM25 サイドカーをビルド
kolm serve --mcp       # Claude Code / Cursor 用のローカル MCP サーバー
kolm install claude-code   # エージェントハーネスを接続
kolm doctor            # env、キー、ポート、バージョンを点検

MCP serve

kolm serve --mcp はローカルアーティファクトを stdio を介して MCP ツールとして公開します (MCP のデフォルト方式)。クライアントはこれをサブプロセスとして実行し、kolm install <harness> --apply は claude-code / cursor / continue / cline の正しい場所に spawn 設定を書き込みます。--http を渡すと 127.0.0.1:8765 にオプションの HTTP JSON-RPC エンドポイントが追加されます。

リモートアクセス

デフォルトのデプロイはローカルです。kolm serve --local はランタイムをあなたのマシンにバインドし、ノートパソコン・スマートフォン・コーディングエージェントはあなたが制御するネットワーク経由でアクセスします。プロンプトと出力は決して当社のインフラを経由しません。経由するインフラ自体が存在しないからです。

ベンダーを信頼せずにホスト型アクセスを望むチームには、同じ .kolm を TEE (Trusted Execution Environment、信頼実行環境) 内で実行することもできます。ハードウェアエンクレーブ (AWS Nitro、GCP Confidential、Azure SEV) がモデル、重み、リクエストデータを封印し、kolm もクラウドプロバイダーも読み取れないようにします。相対的に安全で、相対的にプライベートです。暗号学的保証ではありませんが、クラウドホスト型推論に対する最も率直な表現です。

OpenAI 互換性

ランタイムは localhost:8765 で /v1/chat/completions、/v1/models、/v1/embeddings を公開します。OpenAI 互換クライアントをそこに接続するだけです。tool call とストリーミングはドキュメント通りに動作します。すべてのリクエストは署名済みレシートを発行します。

コンプライアンスパック (Compliance packs)

コンプライアンスパックは (recipe.json + evidence.md + dpia.md + mrm.md + controls.md) のバージョン管理されたバンドルで、どのマスキングが必須か、どのフィールドを検証すべきか、監査人が何を尋ねるかを符号化します。それによってエンジニアリングチームが法務チームの承認した同じ目標に対してビルドすることになります。

今日 3 つのパックが提供されており、それぞれ data/compliance-packs/<pack>/ 配下にあります: HIPAA (PHI マスキング、K 基準 0.92)、SR 11-7 (Fed Model Risk Management)、SOX-lite (AI のための ITGC)。

この翻訳について

日本語版は要約のみを扱います。コードブロック、CLI コマンド、API エンドポイント、JSON 形式、エラーコードは言語に依存しないため英語のままです。完全なリファレンスとすべてのチュートリアル、最新の例については、正本の英語ドキュメントを参照してください: kolm.ai/docs。
