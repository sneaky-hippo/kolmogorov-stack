Quickstart

Five minutes from npm install to your first artifact. You install the CLI, you point your existing OpenAI-compatible client at the local kolm proxy, you watch the lake fill, and you run kolm lake stats to see what you have. Everything is local. Nothing leaves your machine unless you turn cloud sync on.

1. Install the CLI

Node 18 or newer. The CLI ships as a scoped npm package. The same binary backs kolm, kolm connect, kolm lake, kolm distill, and kolm run.

npm i -g @kolm/cli
kolm version
# kolm 7.x.y (node 20.x, darwin-arm64)

2. Start the local proxy

The connect proxy is an OpenAI-compatible HTTP server on 127.0.0.1:8787. Every /v1/chat/completions, /v1/messages, and /v1/embeddings call you route through it is captured into the local lake before it is forwarded to the real upstream provider.

kolm connect start
# proxy listening on http://127.0.0.1:8787/v1
# captures landing in ~/.kolm/events/events.sqlite

3. Point your client at the proxy

The proxy is wire-compatible with OpenAI, Anthropic, OpenRouter, and Google AI Studio. Set one env var and your existing SDK keeps working.

export OPENAI_BASE_URL=http://127.0.0.1:8787/v1
export OPENAI_API_KEY=sk-your-upstream-key  # forwarded unchanged
# now your existing app sends through kolm

Per-provider setup is broken out in detail: see OpenAI, Anthropic, OpenRouter, Gemini.

4. Watch captures land

Send a request through your app the way you normally would. The proxy mirrors the response back to your client and writes a 35-field event to the local lake. Use kolm tail captures to see them stream live.

kolm tail captures --namespace default
# [live] cap_018a3e... default  gpt-4o-mini  234ms  $0.0008  in=412 out=88

5. Run lake stats

kolm lake stats aggregates the local event store and prints what you would otherwise be paying observability vendors for. By namespace, by model, by hour, by cost.

kolm lake stats --window 24h
# events:    1842
# spend:     $4.21
# top model: gpt-4o-mini (61% of calls)
# duplicate-pair rate: 27% (eligible for cache or local replacement)

Where to go next

Once you have a few hundred captures in the lake, the rest of kolm.ai opens up.

  • Privacy wires the 17 detector classes so no PHI/PII leaves your laptop.
  • Optimizer reads the lake and prints the 11 cost-cut opportunity types it finds.
  • Datasets turn raw captures into train/val/holdout splits with sha256-mod disjointness.
  • Training auto-detects the task and picks a path profile + backbone.
  • Distillation ships the result as a portable .kolm artifact.
  • Runtime swaps your live calls onto the local artifact when it wins.
Privacy →17 detector classes, VAR_ placeholders, allow/redact/block/override actions. Lake schema →The canonical 35-field event row and the SQLite layout backing it. Optimizer →The 11 opportunity types kolm.ai finds in your traffic.
