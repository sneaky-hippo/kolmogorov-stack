Privacy

Every prompt and response that goes through the proxy is scanned by 17 detector classes before it is written to the lake. You set the per-class policy in ~/.kolm/runtime/policy.json. Detected spans are replaced with VAR_* placeholders, blocked classes 4xx the request, and the full audit trail is appended to ~/.kolm/runtime/audit.log.

The 17 detector classes

Each detector returns 0..N spans per text body with a confidence score and a class id. The class id is what the policy engine matches on.

Policy actions

Each class is configured with one of four actions. The action runs before the request leaves the proxy and before the response is returned to the client.

  • allow - pass through unchanged. The span is still logged but the original text reaches the upstream.
  • redact - replace each detected span with a deterministic VAR_* placeholder. The mapping is kept in the local lake row, never sent upstream.
  • block - 451 the request with a structured error body. Nothing is forwarded.
  • override - log only. Use this when you have explicit user consent for the class on this namespace.

policy.json schema

The policy file lives at ~/.kolm/runtime/policy.json and is hot-reloaded on every request. Per-namespace overrides take precedence over the default.

{
  "version": 1,
  "default": {
    "ssn": "block",
    "credit_card": "block",
    "api_key": "block",
    "jwt": "redact",
    "email": "redact",
    "phone": "redact",
    "mrn": "redact",
    "npi": "redact",
    "icd10": "allow",
    "cpt": "allow",
    "ip_v4": "allow"
  },
  "namespaces": {
    "billing": {
      "credit_card": "redact",
      "iban": "redact"
    },
    "support": {
      "email": "redact",
      "phone": "redact"
    }
  }
}

Audit log

Every detection, every action, every override is appended to ~/.kolm/runtime/audit.log as one JSON line. Rotates daily, never overwritten. kolm privacy audit tails it with structured filters.

kolm privacy audit --window 24h --class ssn
# 2026-05-18T09:14:21Z  ns=billing  class=ssn  action=block  count=1
# 2026-05-18T09:18:02Z  ns=support  class=ssn  action=redact count=2

Dry-run a policy

Before you flip an allow to a block, see what would have been blocked in the last 24 hours.

kolm privacy dry-run --window 24h --policy ~/.kolm/runtime/policy.proposed.json
