Datasets

A dataset in kolm.ai is a deterministic, frozen view over the event lake. You pick a namespace, you pick a window, you split by sha256_mod for guaranteed disjointness, you label what needs labelling, and you freeze. Every training and eval run pins to a dataset id so results are reproducible months later.

Build a dataset

One command. The workbench reads the lake, applies the selector, splits, and writes the manifest to ~/.kolm/datasets/<id>/.

kolm datasets build summarizer-v1 \
  --namespace summarizer \
  --window 30d \
  --split 0.80/0.10/0.10 \
  --min-rows 1000

Splits with sha256-mod disjointness

The split function is int(sha256(prompt_canonical), 16) % 100. Train is buckets 0..79, val is 80..89, holdout is 90..99. Any future call with the same canonical prompt always falls into the same bucket, so a row cannot accidentally move from val to train when you rebuild.

# A prompt with sha256-mod = 87 always lands in val
# A prompt with sha256-mod = 92 always lands in holdout
# Re-running the build the next day cannot move it.

Label queue

For tasks that need human labels, the workbench surfaces unlabelled rows in priority order (high uncertainty first, then frequency). Labels are stored alongside the row in ~/.kolm/datasets/<id>/labels.sqlite.

kolm datasets label summarizer-v1 --kind classification --classes urgent,normal,low
# 12 rows pending. press y/n/s/q.
# [1/12] "Production is down, customers..." -> u
# [2/12] "Quick question about billing..." -> n

Freeze

Freezing computes a sha256 over the row ids + labels + split assignments and stamps a manifest. Any training run that pins this id is reproducible bit-for-bit.

kolm datasets freeze summarizer-v1
# frozen as ds_018e... sha256=8f3a... rows=4204 train=3363 val=420 holdout=421

Export

For external tooling. Emits JSONL with one example per line, one file per split.

kolm datasets export summarizer-v1 --out ./out
# ./out/train.jsonl  (3363 rows)
# ./out/val.jsonl    (420 rows)
# ./out/holdout.jsonl(421 rows)
# ./out/manifest.json

Next steps

Hand the frozen dataset id to training or distillation. Run evals against the holdout once you have an artifact. Start from /docs/quickstart if the lake is still empty.
