Storage

kolm.ai is local-first. The lake, the artifacts, the receipts, the audit log, the device map all live under ~/.kolm/ on the machine that captured them. Storage modes control which fields get persisted and, if cloud sync is on, which fields get synced. Default is redacted_local: redacted bodies on disk, no cloud bytes.

The 5 storage modes

ModeLocally storedSent to cloud
metadata_onlyRow metadata, no message bodiesnothing
redacted_local (default)Redacted message bodies + VAR_ map, no raw bytesnothing
raw_localBoth raw and redacted bodies, both encrypted at restnothing
redacted_cloud_syncSame as redacted_localRedacted bodies only, per-class gated
raw_cloud_syncSame as raw_localRaw + redacted, per-class gated

Local-first paths

Everything kolm.ai writes lives under ~/.kolm/. The folder is the unit of backup; copy it to keep your state.

~/.kolm/
  config.json             # local config: base URL, retention, optimizer thresholds
  device.json             # detected hardware
  events/
    events.sqlite         # the lake (WAL mode)
  runtime/
    policy.json           # privacy + runtime policy
    audit.log             # one JSON line per detection / action
  artifacts/
    summarizer-v1.kolm    # installed artifacts
  datasets/
    ds_018e.../           # frozen datasets
  jobs/
    jobs.jsonl            # training / distill job ledger
  keys/
    signing.ed25519       # local signing key (0600)
    signing.pub           # public key

What stays local in every mode

Some fields never leave the machine, regardless of mode.

  • The signing private key under ~/.kolm/keys/.
  • The privacy_var_map column in the lake (the VAR_ to original mapping).
  • The upstream OPENAI_API_KEY / ANTHROPIC_API_KEY / etc. The proxy forwards them but never persists them to the lake.
  • The full audit.log. Sync sends summarized counters only.

Set the mode

kolm storage mode set redacted_local
kolm storage mode set raw_local --namespace forensics
kolm storage mode show
# default     redacted_local
# namespaces:
#   forensics  raw_local

Encryption at rest

When mode is raw_local or raw_cloud_sync, the messages_raw_json and response_raw_json columns are encrypted with a per-machine AES-256-GCM key under ~/.kolm/keys/at-rest.aes. Lose the key and the raw rows are unreadable.

Vacuum

Reclaim space after a retention sweep.

kolm storage vacuum
# scanning events.sqlite ... 1.2 GB before
# rebuilding ... 412 MB after
# done in 18s

Next steps
