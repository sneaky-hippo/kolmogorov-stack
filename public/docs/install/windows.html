Install kolm on Windows

The kolm CLI ships as a single npm package and runs natively on Windows. PowerShell 5.1 or PowerShell 7 both work. Windows 10, Windows 11, and Windows Server all supported. If you prefer the Linux experience, install on WSL2 and follow the Linux guide.

Prerequisites

Node.js 20 or newer. Check your version in PowerShell with node --version. If you do not have Node:

Five minutes

Step 1

Install via npm

npm i -g kolm

Global install puts the kolm.cmd shim on your path. If PowerShell does not find it after install, restart the shell (the global npm bin folder is added to PATH only on new sessions).

Step 2

Verify

kolm --version
kolm --help

You should see a version line and the one-screen verb list. If execution policy blocks the .cmd shim, run PowerShell as admin and set Set-ExecutionPolicy -Scope CurrentUser RemoteSigned.

Step 3

Sign in

kolm signup --email you@company.com
# or, if you already have a key:
kolm login --key ks_...

The signup verb provisions a tenant and an API key, then writes them to $env:USERPROFILE\.kolm\config.json. The login verb takes an existing key directly.

Step 4

Run the doctor

kolm doctor
kolm doctor --detect-hw

The first form sanity-checks config, cloud, Docker Desktop if present, and any project-level kolm.yaml. The second reads nvidia-smi (if installed) to recommend a model tier based on the discrete GPU.

Step 5

Build the first artifact

kolm build first --from chatbot
kolm list
kolm run $env:USERPROFILE\.kolm\artifacts\first.kolm 'hello'

One-shot scaffold plus seeds plus compile plus verify. Drops a signed .kolm under %USERPROFILE%\.kolm\artifacts\.

PowerShell notes

PowerShell parses some characters differently from bash. When passing JSON or single-quoted strings to kolm compile, use a single-quoted here-string to keep PowerShell from expanding $:

kolm compile @'
classify a support ticket as billing, technical, or other
'@

For arguments that contain - or @ (which PowerShell parses as operators), use the stop-parsing token: kolm run art.kolm --% --json 'input here'.

Where files land

PathHolds
%USERPROFILE%\.kolm\config.jsonTenant + API key + base URL.
%USERPROFILE%\.kolm\artifacts\Signed .kolm files.
%USERPROFILE%\.kolm\events.sqliteCaptured (input, output) pairs from kolm capture.
%USERPROFILE%\.kolm\models\Pre-cached GGUF weights from kolm models pull.
%USERPROFILE%\.kolm\logs\runs.jsonlAppend-only run history. Tail with kolm logs.

Upgrade

kolm upgrade           # check for a newer release
kolm update            # self-install the latest from github
