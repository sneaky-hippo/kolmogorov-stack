API

The kolm.ai REST surface is the same whether you are talking to the local proxy on 127.0.0.1:8787 or the hosted control plane on kolm.ai. Every endpoint is JSON in, JSON out, bearer-auth on protected routes, and rate-limited per tenant. Every endpoint has a stable response envelope.

Base URL

Local proxy: http://127.0.0.1:8787. Hosted: https://kolm.ai. Self-hosted enterprise: your domain.

export KOLM_BASE=http://127.0.0.1:8787
curl -s $KOLM_BASE/v1/health | jq .

Auth

All /v1/* routes except /v1/health require a bearer key. Local proxy uses the fingerprint of ~/.kolm/keys/signing.pub; hosted uses the issued tenant key.

curl -s $KOLM_BASE/v1/lake/stats \
  -H "Authorization: Bearer $(kolm key fingerprint --raw)"

/v1/chat/completions

OpenAI-compatible chat. Captures every call to the lake before forwarding.

curl $KOLM_BASE/v1/chat/completions \
  -H "Authorization: Bearer $KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4o-mini","messages":[{"role":"user","content":"hi"}]}'

/v1/messages

Anthropic-compatible messages. Captures every call.

curl $KOLM_BASE/v1/messages \
  -H "x-api-key: $ANTHROPIC_KEY" \
  -H "anthropic-version: 2023-06-01" \
  -d '{"model":"claude-haiku-4-5","max_tokens":256,"messages":[{"role":"user","content":"hi"}]}'

/v1/capture/*

MethodPathPurpose
GET/v1/capture/streamSSE live tail of captures, filterable by namespace
GET/v1/capture/healthDurability probe; returns x-kolm-capture-durable
POST/v1/capture/promotePromote in-memory captures to durable storage
 
curl -N $KOLM_BASE/v1/capture/stream?namespace=summarizer \
  -H "Authorization: Bearer $KEY"

/v1/lake/*

MethodPathPurpose
GET/v1/lake/statsCost, latency, error rollups
GET/v1/lake/eventsList events with filters
GET/v1/lake/events/:idSingle event
 
curl "$KOLM_BASE/v1/lake/stats?window=24h&by=namespace" -H "Authorization: Bearer $KEY"

/v1/optimize/*

curl "$KOLM_BASE/v1/optimize?window=7d&min_confidence=0.8" -H "Authorization: Bearer $KEY"

/v1/datasets/*

MethodPathPurpose
POST/v1/datasetsBuild a new dataset from a selector
GET/v1/datasetsList datasets
GET/v1/datasets/:idDataset manifest
POST/v1/datasets/:id/freezeFreeze the dataset for reproducibility
 
curl -X POST $KOLM_BASE/v1/datasets \
  -H "Authorization: Bearer $KEY" -H "Content-Type: application/json" \
  -d '{"name":"summarizer-v1","namespace":"summarizer","window":"30d","split":[0.8,0.1,0.1]}'

/v1/labels/*

curl $KOLM_BASE/v1/labels?dataset=ds_018e -H "Authorization: Bearer $KEY"
curl -X POST $KOLM_BASE/v1/labels -H "Authorization: Bearer $KEY" \
  -d '{"row_id":"r_18a","label":"urgent"}'

/v1/synth/*

Generate synthetic seed examples for cold-start tasks.

curl -X POST $KOLM_BASE/v1/synth \
  -H "Authorization: Bearer $KEY" \
  -d '{"kind":"classification","classes":["urgent","normal","low"],"n":120}'

/v1/sim/*

Simulate a candidate policy against captured traffic, no live mutation.

curl -X POST $KOLM_BASE/v1/sim \
  -H "Authorization: Bearer $KEY" \
  -d '{"policy":"local_first","window":"7d","artifact":"summarizer-v1"}'

/v1/bakeoff/*

curl -X POST $KOLM_BASE/v1/bakeoff \
  -H "Authorization: Bearer $KEY" \
  -d '{"dataset":"ds_018e","contestants":["cache","rule","prompt_only","gemma-3n-e2b","qwen-0.5b","phi-mini","claude-haiku-4-5","gpt-4o-mini"]}'

/v1/training/plan

curl -X POST $KOLM_BASE/v1/training/plan \
  -H "Authorization: Bearer $KEY" -d '{"dataset":"ds_018e","tier":"3090"}'

/v1/runtime/*

MethodPathPurpose
POST/v1/runtime/registerRegister an artifact for a namespace
POST/v1/runtime/policySet the policy (per-namespace optional)
GET/v1/runtime/statsReplacement stats by stage
 
curl -X POST $KOLM_BASE/v1/runtime/policy -H "Authorization: Bearer $KEY" \
  -d '{"policy":"local_first","namespace":"summarizer"}'

/v1/devices/*

curl $KOLM_BASE/v1/devices -H "Authorization: Bearer $KEY"
curl -X POST $KOLM_BASE/v1/devices/install -H "Authorization: Bearer $KEY" \
  -d '{"artifact":"summarizer-v1","transport":"scp","target":"user@host:/srv/kolm"}'

/v1/sync/*

curl -X POST $KOLM_BASE/v1/sync/state -H "Authorization: Bearer $KEY" \
  -d '{"state":"redacted_only"}'
curl $KOLM_BASE/v1/sync/audit?window=7d -H "Authorization: Bearer $KEY"

/v1/team/*

curl -X POST $KOLM_BASE/v1/team/invite -H "Authorization: Bearer $KEY" \
  -d '{"email":"alice@example.com","role":"contributor"}'
curl $KOLM_BASE/v1/team/approvals -H "Authorization: Bearer $KEY"

Error envelope

Every 4xx and 5xx response carries the same shape.

{
  "error": {
    "code": "privacy_blocked",
    "message": "ssn class set to block in policy.json",
    "request_id": "req_018k",
    "details": { "class": "ssn", "namespace": "billing" }
  }
}
