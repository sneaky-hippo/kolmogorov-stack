Dev-agent proxy install

Every code-writing AI you use already speaks an OpenAI-compatible API. Point them at http://127.0.0.1:8787 and the local kolm proxy captures the full round-trip: prompt, response, latency, cost, whether you accepted it, which file you applied it to. Then it tells you which model was the cheapest one that still got the task done. One install, six agents covered.

Why route through a proxy

You already have one editor, two chat agents, and a fresh coding CLI you tried this week. Each one bills separately, each one stores its history somewhere different, none of them tell you which model worked best for the kind of code you actually write. The kolm proxy is a single capture point. Every request goes through it, every response comes back through it, and your laptop builds a private record you can query, filter, and compile down into a smaller cheaper artifact that handles your house-style prompts at a fraction of the rented cost.

Every install writes a timestamped backup of the original config (.kolm-backup-<ts>) and registers a marker so you can later run kolm uninstall <agent> and the proxy step disappears cleanly. kolm install all covers the lot in one shot.

Five minutes

Step 1

Start the proxy

Boot the local daemon so the proxy is listening on 127.0.0.1:8787. The first run picks a free port if 8787 is taken and prints what it actually used.

kolm connect
# kolm proxy listening on http://127.0.0.1:8787
# state dir: ~/.kolm
Step 2

Install for every agent at once

kolm install all
# patched codex      (~/.codex/config.toml)
# patched claude     (~/.claude/settings.json)
# patched aider      (~/.aider.conf.yml)
# patched gemini-cli (~/.gemini/cli.json)
# cursor             (manual: paste env vars into Settings » Advanced)
# windsurf           (manual: paste env vars into Settings » Models)

Two of the IDE agents (Cursor and Windsurf) don't expose a documented config file for the base URL, so we print the two env vars you paste in. The CLI also drops shell wrappers in ~/.kolm/bin/ as a fallback so you can always launch them through kolm-cursor / kolm-windsurf and skip the IDE settings step.

Step 3

Wrap an ad-hoc command

For anything you forgot to install, or a tool that isn't on the supported list yet, kolm wrap exports the right env vars and execs the command. Same effect, no permanent file edit.

kolm wrap claude --print "explain this stack trace"
kolm wrap aider --model gpt-4o-mini src/router.js
Step 4

Confirm the proxy sees traffic

kolm tail captures --limit 5
# 14:02:13  claude-3.5  anthropic   324 tok  1.2s  ok
# 14:02:31  gpt-4o      openai      181 tok  0.6s  ok
# ...

If you see your last 5 prompts there, the proxy is wired. If not, restart the agent and try again — most CLIs cache the old base URL until you re-source your shell.

Step 5

See which agent is winning

kolm agents stats --window 7d
# agent          requests  accept%  median_latency  total_cost
# claude-code        421     82.7%      980 ms        $2.41
# codex              198     71.2%      640 ms        $0.88
# cursor             612     64.3%      710 ms        $4.10
# aider              57      88.6%     1.4 s          $0.31

Acceptance is inferred from the next event in the same session: if you didn't follow up with a correction inside 90 seconds, it counts as accepted. Confidence in the heuristic is exposed under --json for the dashboard.

Shell init alternative

If you don't want kolm to edit any agent's config and would rather just shadow the env vars in your shell, kolm shell-init prints the right export lines for your detected shell.

# bash / zsh
eval "$(kolm shell-init)"

# fish
kolm shell-init | source

# powershell
kolm shell-init | iex

The output sets OPENAI_BASE_URL, ANTHROPIC_BASE_URL, OPENROUTER_BASE_URL, and the matching *_API_KEY values to a local pass-through key. The proxy keeps your real upstream keys server-side so they never appear in agent settings files.

Uninstall

kolm uninstall claude-code
# restored ~/.claude/settings.json from .kolm-backup-1715980412

kolm uninstall all
# restored 4 of 6 configs (cursor + windsurf were manual installs — remove env vars yourself)

Privacy posture

Captures are written to ~/.kolm/events.sqlite by default and nothing leaves the device. The privacy membrane (17 detector classes) tags each captured prompt with which classes it touched, and the per-class blocklist on the cloud-sync settings page acts as a kill-switch that runs before anything is uploaded. See /docs/privacy and /docs/cloud-sync for the full model.
