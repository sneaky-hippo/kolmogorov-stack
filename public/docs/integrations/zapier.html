Zapier integration

Zapier has no native kolm app yet. You call a deployed artifact via the Webhooks by Zapier action. Bearer auth in the header, JSON body in, JSON envelope out. The same shape as the REST API.

Prerequisites

  • A kolm API key. Get one from /signup or via kolm signup --email you@company.com.
  • An artifact you have already compiled and published. The walk-through uses <owner>/<name> as the handle.
  • A Zapier account on a Starter plan or above (Webhooks is a premium action).

Five minutes

Step 1

Add a Webhooks by Zapier action

In a new or existing Zap, click the plus to add an action, search for Webhooks by Zapier, and pick Custom Request. This is the generic HTTP client; you set the URL, headers, and JSON body yourself.

Step 2

Configure the request

Method:        POST
URL:           https://kolm.ai/v1/artifacts/<owner>/<name>/run
Data Pass-Through?  No
Data:          {"input": "{{step_1__text}}"}
Wrap Request In Array?  No
File:          (leave blank)
Unflatten:     Yes
Basic Auth:    (leave blank)
Headers:
  Authorization: Bearer kolm_...
  Content-Type:  application/json

Replace <owner>/<name> with your artifact handle. The {{step_1__text}} token is whatever the upstream step exposes; click the plus on the field to pick from the Zap's data sources.

Step 3

Test the action

Click Test & Continue. Zapier shows the parsed JSON response in a tree view. Downstream actions can then reference output, k_score, and other fields with the standard Zapier mapping picker.

Step 4

Wire to a downstream step

Add the next action (Gmail, Slack, Airtable, whatever) and map the kolm output into its fields. Zapier remembers the response shape from the test, so the picker shows the top-level keys directly.

Step 5

Turn on the Zap

Once the steps look right, toggle the Zap on. Every trigger fires the chain. Zapier logs the request and parsed response per run; failures land in the Zap history with the upstream status code (401 means bad bearer, 404 means wrong handle, 422 means missing required input).

Trigger from a webhook

If you want kolm to be the first step rather than a middle one, swap the trigger for Webhooks by Zapier > Catch Hook. Zapier assigns a unique inbound URL; point your upstream system at that URL and every POST kicks off the Zap.
