Connect OpenAI

The kolm.ai local proxy speaks the OpenAI wire format natively. Point your existing OpenAI SDK, LangChain, or raw fetch at http://127.0.0.1:8787/v1 and every chat.completions, responses, and embeddings call gets mirrored into the local lake before forwarding upstream. No SDK rewrite, no proxy changes, no streaming break.

One env var

The OpenAI SDK reads OPENAI_BASE_URL for the API endpoint and OPENAI_API_KEY for the upstream key. The proxy forwards the key unchanged.

export OPENAI_BASE_URL=http://127.0.0.1:8787/v1
export OPENAI_API_KEY=sk-your-upstream-key
kolm connect start

Python SDK

from openai import OpenAI
client = OpenAI(base_url="http://127.0.0.1:8787/v1")

r = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Summarize this PR diff..."}],
)
print(r.choices[0].message.content)

TypeScript SDK

import OpenAI from "openai";
const client = new OpenAI({ baseURL: "http://127.0.0.1:8787/v1" });

const r = await client.chat.completions.create({
  model: "gpt-4o-mini",
  messages: [{ role: "user", content: "Summarize this PR diff..." }],
});
console.log(r.choices[0].message.content);

Streaming

Streaming is preserved end-to-end. The proxy reads the upstream SSE stream, forwards each data: frame to the client unchanged, and writes the assembled completion to the lake on stream close.

const stream = await client.chat.completions.create({
  model: "gpt-4o-mini",
  stream: true,
  messages: [{ role: "user", content: "Write a haiku about disk I/O." }],
});
for await (const chunk of stream) process.stdout.write(chunk.choices[0]?.delta?.content || "");

LangChain

LangChain wraps the OpenAI SDK so setting OPENAI_BASE_URL is enough for ChatOpenAI, OpenAIEmbeddings, and the rest. No code changes.

from langchain_openai import ChatOpenAI
llm = ChatOpenAI(model="gpt-4o-mini", openai_api_base="http://127.0.0.1:8787/v1")
print(llm.invoke("List three database indexing strategies.").content)

Raw fetch

If you do not use a vendor SDK, the proxy speaks raw HTTP. Identical request body, identical response body.

curl http://127.0.0.1:8787/v1/chat/completions \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4o-mini","messages":[{"role":"user","content":"hello"}]}'

Namespaces

Tag each call with a namespace by adding the x-kolm-namespace header. The lake groups by namespace, the optimizer rolls up costs by namespace, the distiller compiles namespace-scoped artifacts.

curl http://127.0.0.1:8787/v1/chat/completions \
  -H "x-kolm-namespace: summarizer-v2" \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -d '...'

Verify it works

One round trip and one kolm tail captures. You should see the call land within a second.

kolm tail captures --namespace summarizer-v2 --limit 5
