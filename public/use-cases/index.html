Solutions

What you can build with kolm.

If a task is repetitive and a frontier model can do it well, you can compile it into a signed local artifact that you own. Below are the shapes we see most often. Yours probably fits.

Some teams need a vertical lane because their data has compliance baggage. Others are general product, research, or engineering teams who want their AI to live on hardware they control. Both paths use the same runtime, the same K-score, and the same audit chain.

Verticalized paths

When the industry sets the constraints.

Four anchor verticals where the architecture matters from day one. The frontier API was never on the table because the data cannot leave the perimeter. The compiled .kolm runs locally, the receipt chain is examinable, and the deployment shape lines up with how your auditors already think.

Clinical AI that runs inside your hospital network.

Triage prior-auth, summarize encounter notes for sign-off, code visits to ICD-10. The model runs on hardware your CISO already trusts, PHI stays inside the customer-hosted bridge your environment at run time, and every output ships with a receipt your clinical informatics committee can re-verify. BAA-bounded deployment is the architecture, reviewed per Enterprise engagement.

Common questions

Before you book a call.

Five questions that come up on every first call. The answers below mirror the FAQPage schema for this page.

Which vertical fits my team if I'm not in healthcare, finance, legal, or edge?+
The vertical pages are anchor templates. If a frontier model can already do your task and you want it local, signed, and free of a per-token bill, the generic quickstart fits. Most teams who land here are not regulated. Pick the closest team shape from the second band on this page or read the cookbook.
Do I need an Enterprise plan to use kolm in a regulated industry?+
For PHI under HIPAA, customer-controlled deployment is required and a BAA review is part of onboarding. For SR 11-7 model risk reviews in finance, the signed artifact and HMAC receipt chain are designed to be examinable by SEC, FINRA, and OCC. For attorney-client work, the artifact runs inside the firm boundary at run time. The Free and Pro plans let you prototype against the same architecture before you commit.
Does the .kolm artifact run offline?+
Yes. Once compiled, the artifact runs on your hardware with no network call to a frontier API at run time. That is the property edge, healthcare, finance, and legal teams care about. Receipt verification and registry lookup are optional and can be done later when the box is back online.
How is data protected at run time?+
Data never leaves the machine the artifact runs on. We describe this architecturally rather than as a crypto claim. The model is local, the inputs are local, the receipts are local. For remote access scenarios, the TEE deployment path runs the artifact inside a trusted execution environment so cloud providers cannot read traffic.
Can I see what people are already compiling?+
The public registry at /leaderboard shows a demo catalog of artifacts with K-score, byte size, and signature status. The cookbook at /cookbook walks through working recipes for the most common shapes: triage, redline, intake, summarization, classification, and tool-call routing.
