Threshold alerts

Fire when a namespace crosses 100 / 500 / 1,000 captured pairs. Counts use the durable W212 capture store.

Email delivery requires RESEND_API_KEY on the deploy. The test button fires the same path as the real threshold crossing, so it exercises WebPush + email + capture-handler header injection end to end.

Browser push

VAPID-signed WebPush. We send a "tickle" (empty payload) — the service worker renders a generic "open /captures" notification.

checking WebPush configuration...

    What gets sent

    • WebPush: title = kolm: namespace "<ns>" hit N captures; body cites whether recipe (<1000) or specialist (>=1000) is now unlocked.
    • Email: same subject + plain-text + HTML body with a link to /captures?namespace=<ns>.
    • Response header: x-kolm-distill-ready: true on the capture response that crosses the threshold AND every capture afterward in that namespace.
    • Response header: x-kolm-distill-threshold: 100|500|1000 on the single capture that crossed.

    Why default off

    kolm.ai does not push or email until you opt in. The capture loop generates real notification volume — one tenant running 5 namespaces past 1,000 pairs would otherwise fire 15 emails. We hold the toggle until the operator says yes.
