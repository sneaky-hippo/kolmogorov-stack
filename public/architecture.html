architecture

How the compiler works.

kolm is one product, four engines, and one signed artifact. The CLI is what you use; the engines are what the CLI orchestrates. This page is the honest picture: which subsystems are shipped, which are scaffolded, and how the pieces compose at compile and at run time. If you want the on-disk format, read /anatomy. If you want the spec, read /docs/rs-1.md. This is the system view in between.

01four engines

The compiler is a composition.

Each engine does one thing. Together they turn a task description and a gold set into a signed file you can run anywhere.

RecallMultimodal vector substrate. Embed text and JSON via sqlite-vec on-device; index by tenant; query at compile and at run. Image, audio, and PDF embedders ship in the next quarterly release.
DistillVerified inference at scale. K-sample a frontier teacher (Anthropic, OpenAI, your provider), deterministically verify each sample against the synthesized test, produce high-quality labels.
Decompose (RSD)Recipe Speculative Decoding. Extract the deterministic-token subset of a model's behavior into a draft pack indexed by prefix-shape and embedding-shape. Used at run time for 2–3× wall-time speedup on covered tokens.
RunLocal execution. llama.cpp + sqlite-vec + the LoRA delta + the recipe pack, all four composed at every token. Receipts written for every output. macOS, Linux, and Windows today; iOS and Android adapters land Q4 2026.

03registry

What the public registry actually stores.

Not the artifact. Not your data. Not your weights. The registry stores three things: the schema versions, the receipt anchors, and the deterministic-pattern set that everyone benefits from. It is small, public, and append-only.

Schema versionsHashes of manifest-v0.1.json, receipt-v0.1.json, and the RS-1 spec. The chain anchors here so any artifact can be verified against the exact schema it was compiled against.
Receipt anchorsPer-artifact HMAC tip. Submitted at compile time. Anchors prove the artifact existed at this hash on this date and was signed by a known tenant.
Pattern setDeterministic-token patterns mined from accepted compiles: a draft pack indexed by prefix-shape and embedding-shape. This is the compounding moat: every public compile shrinks the marginal cost of every future compile.
What's NOT hereYour gold examples. Your LoRA weights. Your recall index. Your inputs at run time. Your outputs at run time. The registry only holds the public, hashable, non-revealing parts.
04K-score gate

The compiler refuses to ship below floor.

K-score is a composite gate. Every artifact that ships passes all four sub-gates; an artifact that fails any one is rejected at compile, not at deploy. This is the difference between a build system and a "best-effort" inference wrapper.

K-score (composite)
 |
 +-- accuracy        sub-gate floor 0.80   measured on hold-out
 +-- coverage        sub-gate floor 0.65   verifier hits / inputs
 +-- size_bytes      sub-gate floor 0.50   target SKU class fit
 +-- p50_latency_us  sub-gate floor 0.40   target hardware class fit

ship  iff  every sub-gate >= floor
fail  if   any single sub-gate < floor

Floors are not magic numbers; they are tier-specific defaults you can tighten or loosen via flags. /k-score has the full breakdown.

05determinism

Why the .kolm zip is byte-stable.

Two compiles of the same gold set, the same flags, and the same teacher state must produce the same .kolm sha256. We get this by canonicalizing every layer.

JSON canonicalizationManifests and receipts use sorted keys, no whitespace, UTC timestamps. Deterministic across operating systems.
ZIP canonicalizationMembers in lexical order, mtime fixed at the registry epoch (2026-01-01T00:00:00Z), no extended attributes, deflate level 6.
LoRA canonicalizationTensor order fixed by parameter name; bf16 with byte-stable dump. Trainer seed is part of the manifest.
Recipe packPattern entries sorted by hash. Registry version pinned at compile.
06receipts

What the receipt actually proves.

A receipt binds (artifact_hash, input_hash, output_hash, recall_hits, runtime_id, issued_at) into a single HMAC chain anchored to the registry. It is local by default; you mirror upstream when you want shared audit.
