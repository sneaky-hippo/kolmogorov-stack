v2.1

2026-05-18current

k-score-2 axes. The five v2 K-score axes (R real-world independence, F fairness, T teacher-fidelity, E eval-source label, Z drift) are promoted from scaffolding to first-class. The reference verifier checks each populated axis; the manifest declares which axes are computed; the bench page surfaces axis means.

  • R axis: external + adversarial holdouts (Presidio, i2b2 2014, MIMIC-III for PHI; CMS / NCQA for claims). Verifier check #16.
  • F axis: equal performance across declared subgroups in seeds.jsonl tags. Verifier check #17.
  • T axis: teacher-delta for distilled_model. Verifier check #18.
  • E axis: eval-source label (synthetic_starter, tenant_captured, external_public, adversarial_pair, auditor_witnessed). Verifier check #19.
  • Z axis: drift snapshot/report/cron mechanism with supersession-chain hand-off. Verifier check #20.
  • §7.1 + §7.2 demoted from "wave forthcoming" to cross-references at the implementation-confirmed subsections §7.6 (W157) and §7.7 (W158).
  • v2.0 verifiers accept v2.1 manifests by ignoring unknown fields; v2.1 is additive and backward-compatible at the read path.

v2.0

2026-05-17frozen

supersession + teacher-fidelity. Major revision. Adds the supersession-chain block that lets a new artifact cite the prior artifact CID it replaces, plus the teacher-fidelity T axis for distilled_model recipes. Closes the tautological-eval loophole by making seeds.jsonl train/holdout split a hard verifier gate.

  • Four-class taxonomy frozen: rule, synthesized_rule, compiled_rule, distilled_model. Verifier check #3 rejects misdeclared artifacts.
  • Three-signature cascade: HMAC-SHA256 + Ed25519 + optional sigstore with Rekor anchor. Verifier checks #4-#6.
  • Supersession block: supersession_of field cites prior artifact CID; supersession reason from a fixed taxonomy (drift_detected, scheduled_rebuild, security_patch, recipe_revision, policy_change, tenant_request). Verifier check #15.
  • Teacher-fidelity (T axis): T = teacher accuracy on same holdout; reported as A/T ratio.
  • seeds.jsonl train/holdout deterministic split. Verifier rebuilds split and confirms disjointness; tautological-eval artifacts rejected pre-signature.
  • Canonical-JSON serialization frozen across JavaScript, Python, Rust reference verifiers; interop test suite enforces byte-identical CIDs.
  • Breaking against v1.x: v1.x manifests claiming the v2.x schema string are rejected. New artifacts must declare version: "2.0" and ship a four-class artifact_class.

v1.1

2026-02-28frozen

drift block. Minor revision. Adds an optional drift block to the manifest so an artifact can declare its drift posture (tolerances + detection cadence) at build time, paving the way for the v2.0 supersession chain.

  • Drift block in manifest with tolerances, detection_cadence, and snapshot_corpus_hash fields.
  • Default tolerances frozen: eval_score ±0.02 (warn) / ±0.05 (breach), k_score.composite ±0.03 / ±0.08, k_score.<axis> ±0.05 / ±0.10.
  • Verifier check #14 added: drift block schema validation when present.
  • Backward compatible: v1.0 verifiers accept v1.1 manifests by ignoring the drift block.

v1.0

2026-01-15frozen

initial RS-1 spec. First public freeze of the Reproducible Signed-artifact format. Defines the .kolm ZIP layout, the canonical-JSON manifest, the K-score formula (A/S/L/C/V), HMAC-SHA256 signature, and the runtime inference receipt.

  • .kolm ZIP layout: manifest.json, spec.json, recipes.json, evals.json, seeds.jsonl, receipt.json plus class-dependent payload.
  • Canonical-JSON: sorted keys, no whitespace, UTF-8, integers as integers.
  • K-score v1 composite: K = 0.40A + 0.15S + 0.15L + 0.15C + 0.15V. Compile gated by recipe-declared min_k.
  • HMAC-SHA256 signature over canonical-JSON(body).
  • Runtime receipt: {manifest_cid, input_sha, output_sha, ts, k_score, signature}.
  • Reference verifier in JavaScript (Node 20+), pure stdlib.

Compatibility policy

Minor versions (v2.0 to v2.1) are additive and backward-compatible at the verifier level; older verifiers ignore unknown fields. Major versions (v1.x to v2.x) require a supersession block in the new artifact pointing at the prior artifact's CID; the verifier rejects v1.x manifests whose schema string claims v2.x.

The change-log entries above are the source of truth for what landed in each version. The corresponding diff link points at the spec PR that landed the change. All spec PRs go through the RFC process described at RS-1 §10.1 How to participate.

How to propose a change

Any normative change to the spec goes through a numbered RFC pull request against docs/rfcs/ in the kolmogorov-stack repo. The current RFC template lives at docs/rfcs/0000-template.md. RFCs land with a 14-day public comment window; the change log records the version bump on merge.
