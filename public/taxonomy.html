Evidenced taxonomy · four classes · verifier-enforced

The four kolm recipe classes.

Every .kolm artifact declares exactly one class. The verifier rejects any artifact whose declared class does not match the bytes on disk. Shipping a distilled_model artifact with no weights is a verifier-rejected lie, not a marketing footnote.

Why this page exists

Before Wave 144, a .kolm artifact could claim "distilled model" while shipping an empty model.gguf. The audit caught it. Wave 144 closed it: every recipe carries a class field, the artifact rolls up to the most-permissive class of its recipes, and the verifier blocks the build if the claim does not match the bytes. This page is the canonical reference for what each class means. The strings on this page are imported directly from src/recipe-class.js so marketing copy and verifier rejection reasons can never drift.

class · rule

Rule recipe

Deterministic finite-state transformation. No model, no teacher, no neural net.

A hand-authored JavaScript function that runs on input and returns output. Same input always returns the same output. There is no model file because there is no model. Examples: phone-number normalizer, SSN redactor, ICD-10 lookup, denial-code classifier with explicit rules.

teacher required: no
weights required: no
quantization: not applicable
runtime: sandboxed JS via node:vm
class · synthesized_rule

Synthesized rule recipe

LLM-emitted rule code, AST-validated against a constrained DSL, then compiled like a rule recipe.

A frontier model (the teacher) emits candidate JavaScript that satisfies the spec. The candidate is parsed into a constrained AST (no I/O, no FFI, bounded loops), fuzzed against the train split, and gated on K-score against the holdout. The final artifact contains rule code, not weights — but the rule code was authored by an LLM, not by a human. Verifiable distinction: this is rule code generation, not knowledge distillation.

teacher required: yes (records teacher_vendor)
weights required: no
quantization: not applicable
runtime: sandboxed JS via node:vm
class · compiled_rule

Compiled rule recipe

Rule or synthesized-rule recipe lowered to C / Rust / WASM and compiled to a native binary.

A rule-class recipe whose source has been emitted as C99 (gcc/clang) and Rust (rustc 2024 edition) and compiled to a deterministic native binary. The manifest carries the toolchain pinning so a verifier can rebuild and confirm binary_hash. There is still no model; the speed-up is from native code, not from quantization.

teacher required: no
weights required: no
quantization: not applicable
runtime: native binary (ELF / Mach-O / PE) or WASM (wasm32-wasi)
class · distilled_model

Distilled model recipe

A small open-weight base model LoRA-fine-tuned on teacher outputs, quantized, and shipped as GGUF / ONNX bytes.

A real neural network. A teacher model (frontier API or large open-weight) generates training pairs from the train split. A small base model (e.g., Llama-3.2-1B, Qwen2.5-0.5B, SmolLM2-360M) is LoRA-fine-tuned on those pairs. The adapter is quantized (GPTQ / AWQ to INT4 or INT8) and exported as GGUF or ONNX. The artifact carries real weights — model.gguf is non-empty and verifies against manifest.hashes.model_pointer. K-score is computed against the holdout, and the receipt chain captures teacher_vendor, teacher_model, student_base, distillation_method.

teacher required: yes
weights required: yes
quantization: applicable (INT4 / INT8)
runtime: onnxruntime / llama.cpp / candle
class · workflow_capsule

Workflow capsule

A graph of recipes (LLM / tool / branch / artifact nodes) compiled to a frozen, replayable workflow IR.

An artifact whose execution is described by a workflow IR (src/workflow-ir.js, spec wir-v1). The IR is a DAG of input / llm / tool / branch / artifact / const / output nodes. The artifact carries the IR alongside a seed cache of (input -> output) pairs captured at compile time; the runtime is cache-first (replays cached outputs byte-for-byte when input matches a seed) and fail-loud (throws if it would need to call an executor but no executor is wired). The artifact-level workflow_ir_hash MUST match the IR shape carried in the zip; verifiers reject any mismatch. Source: traced from a real production session (kolm capture → kolm compile --ir) or hand-authored.

teacher required: no
weights required: no
quantization: not applicable
runtime: workflow-IR interpreter + caller-supplied node executors

Class decision tree

When you do not declare recipe.class explicitly, kolm walks this tree. Source of truth: inferRecipeClass() in src/recipe-class.js. The walk is most-specific first — a recipe with weights is never demoted to a rule even if it also carries DSL.

1. Recipe has workflow_ir / workflow_ir_hash?
   └─ YES → workflow_capsule     (rank 4 · IR shape required · lineage.workflow_ir_hash verified)
   └─ NO  → continue

2. Recipe has weights_file / gguf_file / onnx_file?
   └─ YES → distilled_model      (rank 3 · weights required · non-empty model_pointer)
   └─ NO  → continue

3. Recipe has compiled_targets / native_bin?
   └─ YES → compiled_rule        (rank 2 · toolchain pin · binary_hash verifiable)
   └─ NO  → continue

4. Recipe has synthesized_by / teacher_vendor?
   └─ YES → synthesized_rule     (rank 1 · teacher attribution required)
   └─ NO  → continue

5. Default
   └─       → rule                 (rank 0 · floor · hand-authored JS, deterministic)

An artifact bundling multiple recipes rolls up under CLASS_RANK (rule=0, synthesized_rule=1, compiled_rule=2, distilled_model=3, workflow_capsule=4). The artifact_class is the MAX of its recipe classes — see rollupArtifactClass().

What the verifier rejects

Source: validateArtifactClass(manifest) in src/recipe-class.js. Each card is a verifier reason — the build fails before signing if the manifest claim does not match the bytes on disk.

artifact_class · distilled_model

Empty model_pointer. manifest.hashes.model_pointer is missing or equals e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855 (the sha256 of an empty buffer). A distilled_model artifact MUST ship real model bytes.

artifact_class · distilled_model

Missing base_model. manifest.base_model is null or the literal string 'none'. The receipt chain must record which open-weight base was LoRA-fine-tuned.

artifact_class · compiled_rule

Missing compiled_targets. manifest.compiled_targets block absent. A compiled_rule artifact must carry the toolchain pin so a verifier can rebuild and confirm binary_hash.

artifact_class · synthesized_rule

Missing teacher attribution. manifest.training records neither teacher_vendor nor teacher_model nor synthesized_by. Synthesized rules must name which teacher emitted the source.

artifact_class · workflow_capsule

Missing workflow_ir_hash. manifest.lineage.workflow_ir_hash is absent. A workflow_capsule artifact must carry the IR shape hash so the runtime can prove the IR has not been tampered with between sign and replay.

artifact_class · any

Unknown class. manifest.artifact_class is not one of rule / synthesized_rule / compiled_rule / distilled_model / workflow_capsule. The enum is frozen in RECIPE_CLASSES.

artifact_class · rule

No positive requirement. rule is the floor — if your recipe is hand-authored JS with no teacher, no weights, no compiled targets, it is a rule recipe and the verifier accepts it without further checks.

Where this fits

/spec/rs-1

Canonical RS-1 spec. The receipt chain wraps the artifact_class into the signed manifest.

/k-score

The frozen-eval gate. Every class can produce a K-score; only distilled_model is gated against the holdout differently.

/drift

The lifecycle surface. Drift detection and supersession apply to every class; only distilled_model triggers eval-drift checks against weights.
