The wrap pattern

One import. One wrap. Your existing SDK calls go through Recipe untouched. Recipe inspects each call, decides its lane, and returns the same SDK shape your code already expects.

If you remove the wrap, your code keeps working. There is nothing to migrate away from. That is the entire interface contract.

// Before
import Anthropic from '@anthropic-ai/sdk';
const ai = new Anthropic();

// After — one line.
import { recipe } from '@kolmogorov/recipe';
const ai = recipe.wrap(new Anthropic());

// Use it normally — every call auto-routed.
const r = await ai.messages.create({
  model: 'claude-opus-4-7',
  messages: [{ role: 'user', content: prompt }],
});

The three-lane router

Every call goes through a tiny router that decides which lane to take. The decision takes <100 µs.

                            ┌─────────────────────────┐
                            │   recipe.wrap(client)   │
                            └────────────┬────────────┘
                                         │
                            ┌────────────▼────────────┐
                            │   classify call shape   │
                            └────────────┬────────────┘
                                         │
              ┌──────────────────────────┼──────────────────────────┐
              │                          │                          │
   small + structured?          verifiable hard task?         everything else
              │                          │                          │
              ▼                          ▼                          ▼
        Lane 1: Recipe         Lane 2: Verified Inf       Lane 3: Passthrough
        ────────────────       ─────────────────────      ──────────────────
        cached recipe          k-shot frontier sample     forward to provider
        OR synthesize          + recipe verifier          (no interception)
        ────────────────       ─────────────────────      ──────────────────
        35 µs · $0             ~600 ms · $k×opus          ~600 ms · $opus
        deterministic          accuracy → 1               whatever it was
        signed receipt         signed receipt             (still records receipt)

Lane 1 — Deterministic recipe

The call shape ("classify spam vs ham", "extract phone number", "language ID") is small and structured. Recipe checks the registry; if a verified recipe exists for the call shape, it executes it directly — no model call, microsecond response.

If no recipe exists yet, Recipe synthesizes one in the background from the call's examples, and starts using it immediately on the next equivalent call. The model call you just made becomes the labeled example — you bootstrap your own recipe.

  • Latency: ~35 µs (compiled JS function, one V8 invocation)
  • Cost: $0 per call after one-time synthesis (~$0.10)
  • Determinism: bit-exact same input ⇒ bit-exact same output, forever
  • Receipt: HMAC-SHA256 over {source_hash, input_hash, output_hash, runtime_version, time}

Lane 2 — Verified Inference

The call has a structured but non-trivial answer (code generation, math, structured extraction with checks). A single shot from a frontier model is 91% reliable; that's not enough for production. Recipe samples k candidates in parallel and runs each through a deterministic verifier — the first one that passes is returned. P(success) = 1 − (1 − p·v)k.

For Opus 4.7 on HumanEval (p≈0.91): k=4 → 99.99%. k=8 → 99.9999%. Same model. Bounded compute. Accuracy approaches 1.

The verifier is itself a recipe. Recipes verify recipes. The receipt binds the verifier's source hash to the chosen output's source hash — you can prove not just what ran, but why it was accepted. See the proof →

Lane 3 — Passthrough

The call is open-ended generation: "write a marketing email," "summarize this PDF," "draft a contract." Recipe does not get in the way. The frontier model is the right tool. We forward the call untouched.

A passthrough call still gets a receipt — the source_hash is null (no deterministic source ran), but the input/output/timestamp/runtime are still attested. You get a tamper-evident audit log of every AI interaction in your system, regardless of lane.

Receipts on every output

Every lane returns a receipt: an HMAC-signed, canonical-JSON object that proves (this exact source) ran on (this exact input) and produced (this exact output) at (this exact time). Anyone can verify offline. Forever. Without us.

Self-host or hosted

Hosted

One npm install, no infra.

Use recipe.wrap(client) against our hosted runtime. Free tier covers most projects. Your usage data is tenant-isolated; signed receipts let you walk away with a complete audit trail at any time.

Self-host

Single Node binary, no dependencies.

Clone the repo, run node server.js. The runtime is one process, the verifier is open MIT-licensed, the spec is RS-1. Vendor neutrality is built in — we provide a reference implementation, not a moat.
