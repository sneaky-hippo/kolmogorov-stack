How it works

A rule-class recipe authored as a sandbox-safe mirror of src/phi-redactor.js (single-source-of-truth pattern, W295/W258-ML-4). The artifact is a single .kolm zip containing the recipe source, an ed25519 receipt chain, and an evaluation block measured against a held-out slice of 60 synthetic-but-realistic claims notes. The recipe is deterministic: the same note always returns the same redacted output with the same token-to-original map. Tokens preserve structure so a downstream teacher API never sees the raw PHI, and reinject() on the teacher response restores fidelity.

Classes covered (all 18 HIPAA Safe Harbor + 3 kolm extensions)

  • NAME, GEO, DATE, PHONE, FAX, EMAIL, SSN, MRN, HPID, ACCT, LIC, VEH, DEV, URL, IP, BIO, OTHER — the Safe Harbor 17 (plus general "BIO" coverage).
  • NPI, DEA, MEDICAID — healthcare-provider identifiers required for claims submission but not technically PHI under 45 CFR 164.514(b)(2).
  • Bare names + bare addresses — W343 additions so unlabeled claims narratives still mask "Sandra Pham" and "415 Oak St" without a "Patient:" or "Address:" prefix.
