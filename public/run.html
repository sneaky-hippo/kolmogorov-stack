Bundle into your iOS app.

Drop the .kolm into your app bundle or download it lazily. The Swift SDK loads it, indexes the user's chosen folder, and gives you a single generate call.

  • iOS 17+. Apple Silicon only.
  • Neural Engine path on supported chips.
  • No telemetry. No network calls.
SPM github.com/kolm-ai/kolm-swift
App.swift
import Kolm

// load + verify on first launch
let kolm = try Kolm.load(
  bundle: "support-writer.kolm",
  index: .documents
)

// generate. local. signed. cite-able.
let reply = try await kolm.generate(
  task: "draft a polite reply to the latest email",
  recall: 8
)

print(reply.text)        // the reply
print(reply.sources)     // which local files it used
print(reply.receipt)     // HMAC chain you can verify

Run it in the browser.

The web runtime uses WebGPU on Chrome and Edge, falls back to wasm simd. The .kolm is fetched once and cached by the service worker.

  • WebGPU first. WASM SIMD fallback.
  • Indexes FileSystemHandle trees locally.
  • Works offline after first load (PWA).
npm npm i @kolm-ai/runtime
app.ts
import { Kolm } from "@kolm-ai/runtime";

// load + verify. cached by the service worker.
const kolm = await Kolm.load("/artifacts/support-writer.kolm", {
  index: await showDirectoryPicker()
});

// generate. local. signed. cite-able.
const reply = await kolm.generate({
  task: "draft a polite reply to the latest email",
  recall: 8
});

console.log(reply.text);      // the reply
console.log(reply.sources);   // which local files it used
console.log(reply.receipt);   // HMAC chain you can verify

What the runtime does. In order.

Five steps. Every one of them is verifiable, and every one of them is offline once the artifact has been downloaded.

01
Parse manifest
Read the declared base, adapter, recipe pack, tools, and permissions from the manifest.json at the root of the zip. Reject if any required field is missing.
02
Verify signature
ed25519 over each component hash, anchored to the kolm public registry. Receipt chain over (artifact_hash, eval_set_hash, eval_score, judge_id). Refuse to load on mismatch.
03
Load locally
Map the base into the platform's tensor runtime (Metal, NNAPI, WebGPU, llama.cpp). Stream the adapter delta. Open the sqlite-vec index in read-only.
04
Index private data
If the user has granted access to a folder, the runtime embeds new files into the local index. The corpus never leaves the device.
05
Generate
Recall k chunks. Draft tokens from the recipe pack where possible. Generate. Cite the chunks. Stamp the output with a verifiable receipt.

Their data does not leave.

Step 04 happens on device. The kolm cloud never sees the user's emails, photos, files, transcripts, or messages. Compile in the cloud. Run on the edge.
