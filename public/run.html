02surface map

Current, beta, and roadmap are separate.

This is the page enterprise buyers should see before they ask for platform proof.

Current: Node CLI

Local inspect, eval, run, and MCP serve are implemented in this repo and covered by the test suite.

Beta: hosted APIs

Compile, registry, verified wrap, receipt verification, readiness, and tenant-scoped auth run through the Express API.

Roadmap: native adapters

iOS, Android, and browser runtimes need published packages, real hardware benchmarks, and store-policy notes before public launch claims.

03target requirements

What a platform target must prove.

signature

Verify locally.

The adapter must reject manifest, layer, and receipt mismatches before execution.

policy

Declare egress.

The runtime must expose its network behavior and fail closed when the artifact forbids egress.

benchmark

Measure devices.

Each target needs p50/p95 latency, memory, battery, thermal, and app-size data on named hardware.

fallback

Document degradation.

Teams must know exactly what happens when a device cannot run the selected model path.
