A 30-second install, a one-line compile, and an honest spec. Everything below works today.

Install #

Two ways to get the CLI on your machine.

npm

# requires Node 20+
$ npm i -g @kolmogorov/kolm
$ kolm --version
kolm v0.1.0

curl (no Node needed)

$ curl -fsSL https://kolm.ai/install.sh | sh
$ kolm --version
kolm v0.1.0

First-run prompts for your model API key (Anthropic by default). The key stays in ~/.kolm/config; we never see it.

CLI: kolm compile #

Compile turns a task description into a signed .kolm artifact.

$ kolm compile "answer my emails in my voice, draft never send"
  recall   indexed 1,283 messages from ~/Mail
  distill  64 verified labels @ k=8 sample
  decomp   recipe pack: 142 patterns (covers 71%)
  sign     HMAC chain: 7e3f...c2a1
  ready    inbox-reply-v1.kolm  18.3 KB

Flags

FlagEffect
--examples FILEJSONL of (input, output) pairs to seed Distill.
--corpus DIRFolder to recall over. Defaults to current directory.
--base MODELOpen base. qwen-2.5-3b, llama-3-8b, hermes-3-8b, etc.
--decomposeRun the recipe extractor for run-time speedup. Default-on for code, JSON, math.
--out FILEOutput artifact path. Default: ./<task-slug>.kolm.
--watchStream the four engines' progress. Default-on in TTY.

CLI: kolm run & kolm serve #

# one-shot
$ kolm run inbox-reply-v1.kolm "hi, can we move to 4pm?"
Sure, 4pm works. Calendar invite incoming.

# MCP server (any MCP client can call this)
$ kolm serve inbox-reply-v1.kolm --mcp --port 4711
listening on stdio + tcp://127.0.0.1:4711

Every run verifies the artifact's signature chain before loading. If the signature fails, the artifact won't load and you'll see SIGCHK_FAIL.

SDKs #

JavaScript / TypeScript (browser + Node)

import { kolm } from "https://kolm.ai/sdk.js";

const reply = await kolm.run("inbox-reply-v1.kolm", {
  input: "hi, can we move to 4pm?"
});

Swift (iOS, macOS)

import Kolm

let artifact = try await Kolm.load("inbox-reply-v1.kolm")
let reply = try await artifact.run("hi, can we move to 4pm?")

Kotlin (Android)

val artifact = Kolm.load("inbox-reply-v1.kolm")
val reply = artifact.run("hi, can we move to 4pm?")

Python (server, CLI, scripts)

from kolm import Kolm

artifact = Kolm.load("inbox-reply-v1.kolm")
reply = artifact.run("hi, can we move to 4pm?")

All SDKs verify the receipt chain on load and re-sign every output, so consumers can independently audit "this output came from this artifact."

RS-1 spec #

RS-1 is the open spec for the recipe registry: how a deterministic-token subset of a model's behavior is captured, indexed, and replayed. MIT-licensed. Contributions welcome.

Recipe object

{
  "id": "email.greeting.formal",
  "shape": { "prefix": "Dear", "context": "<0.81|formal_register>" },
  "draft": " {salutation},",
  "trust": 0.94,
  "hash": "7e3f…c2a1"
}

The full RS-1 grammar (1.4k lines, EBNF) lives at github.com/kolmogorov/rs-1.

  "base": { "model": "qwen-2.5-3b-instruct", "sha256": "…" },
  "lora": { "sha256": "…", "params": 12_582_912 },
  "recipes": { "version": "rs-1", "count": 142, "sha256": "…" },
  "recall": { "backend": "sqlite-vec", "vectors": 1283, "sha256": "…" },
  "k_score": 0.94,
  "signed_at": "2026-05-06T18:30:00Z",
  "signature": { "alg": "HMAC-SHA256", "chain": ["…", "…"] }
}

Receipts v0.1 #

Every output emits a receipt: an HMAC-SHA256 chain over the artifact hash, the input hash, and the model output bytes. Receipts can be verified offline and anchored to the public Kolmogorov registry.

{
  "artifact": "inbox-reply-v1.kolm",
  "artifact_sha256": "7e3f…c2a1",
  "input_sha256": "…",
  "output_sha256": "…",
  "timestamp": "2026-05-06T18:34:11Z",
  "signature": "HMAC-SHA256:…",
  "prev": "<previous receipt id>"
}

Receipts cost zero bandwidth at runtime — they're computed and emitted with the output. Verifying a receipt requires only the artifact's public signing key.

REST API #

For tools that can't run the CLI, the cloud exposes the same compile pipeline over HTTPS.

EndpointBody
POST /v1/compile{ task, examples?, corpus_namespace?, base_model? } → { job_id }
GET /v1/compile/:job_id{ status, k_score?, artifact_url? }
POST /v1/run{ artifact_url, input } → output + receipt
POST /v1/embed{ items: [...] } → vectors
POST /v1/recall{ namespace, query, k? } → matches
GET /v1/registry/exportFull RS-1 registry, signed.

K-score #

K-score is the headline quality metric for a compiled specialist. Range [0, 1], threshold ≥ 0.90 required to publish. Defined as:

K = (verified_pass_rate × recall_grounding) / (1 + size_penalty)

Where verified_pass_rate is the fraction of held-out examples passed by the deterministic verifier, recall_grounding is the fraction of outputs that cite at least one corpus chunk, and size_penalty grows with artifact size beyond 50 MB.
