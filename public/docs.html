docs

Build against the current product.

These docs describe what the repo ships today: the Node CLI, the hosted compile API, signed recipe-mode .kolm artifacts, HMAC receipt metadata, and local MCP serving from ~/.kolm/artifacts.

quickstart github RS-1 spec
01install

Install and authenticate.

Until a public package-manager release exists, install the CLI from the GitHub repository or link a local checkout.

Install from GitHub.

npm i -g github:sneaky-hippo/kolmogorov-stack
kolm --version

Link a checkout.

git clone https://github.com/sneaky-hippo/kolmogorov-stack
cd kolmogorov-stack
npm install
npm link
kolm version

Save cloud config.

kolm config base https://kolm.ai
kolm login
# paste your ks_... API key
02compile path

Compile, run, inspect, serve.

The CLI command surface is intentionally small. These examples use only implemented flags and transports.

$ kolm compile "classify support tickets" \
  --examples ./support.examples.jsonl \
  --data ./support-policy \
  --base-model qwen2.5-coder-7b-instruct-q4_0 \
  --out ./artifacts

$ kolm run ./artifacts/support-tickets.kolm '{"input":"checkout fails on Android"}'
$ kolm inspect ./artifacts/support-tickets.kolm
$ kolm eval ./artifacts/support-tickets.kolm
$ kolm benchmark ./artifacts/support-tickets.kolm --runs 5 --out ./benchmark.json
$ kolm score ./artifacts/support-tickets.kolm
$ kolm serve --mcp --http --port 11455
03CLI reference

Every implemented command.

If a command is not listed here, do not treat it as supported by the current CLI.

kolm loginprompts for a ks_... API key and writes ~/.kolm/config.json
kolm configreads or sets base and api_key
kolm compileposts a task to /v1/compile, polls the job, and downloads the artifact
kolm runexecutes a local artifact with a JSON input or bare string input
kolm inspectprints manifest, recipes, K-score, and signature metadata
kolm evalre-runs embedded evals using the artifact runner
kolm benchmarkruns embedded evals repeatedly, blocks runtime network calls, and emits reproducible benchmark JSON
kolm scoreprints the artifact K-score summary
kolm servestarts MCP serving for artifacts in ~/.kolm/artifacts; use --mcp, optional --http and --port
kolm publishcurrently a placeholder; public gallery publishing is not implemented
kolm versionprints the CLI version
04API reference

Hosted API surface.

The CLI uses bearer auth. Browser sessions can use the httpOnly kolm_session cookie set by sign-in routes.

POST /v1/signupbody { email, name? }; returns tenant metadata and a one-time API key
POST /v1/signinbody { api_key }; validates key and sets kolm_session
POST /v1/session/loginsame API-key session flow for app clients
GET /v1/accountreturns tenant id, name, plan, quota, usage, and remaining compile budget
POST /v1/account/rotate-keyreturns a new API key; new keys are stored as hashes at rest
POST /v1/compilebody { task, examples?, corpus_namespace?, base_model? }; returns a compile job id
GET /v1/compile/:idreturns status, progress, K-score, and artifact download URL after completion
GET /v1/compile/:id/.kolmdownloads the compiled artifact for the authenticated tenant
GET /v1/registry/exportexports public recipe metadata and source for current registry entries
GET /sdk-current.jsoncurrent browser SDK SHA, SRI hash, URL, and byte size
05artifact format

Open schemas for the artifact contract.

The current .kolm file is a zip artifact with manifest, model pointer metadata, recipe data, placeholder LoRA/index members where appropriate, eval data, and HMAC receipt metadata. Native LoRA training, populated vector indexes, and external receipt anchoring remain product-hardening work, not current defaults.
