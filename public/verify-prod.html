Live drift indicator

Verifying an artifact tells you it is what it claims to be on the day it was signed. It does not tell you whether the world has moved out from under it. Once a .kolm is in production, an operator needs a second loop — kolm drift detect — that re-scores the live artifact against a frozen baseline and reports whether any K-score axis has crossed its tolerance band. The card below documents what that loop emits so the same auditor who dropped the file above can reason about the artifact's current health, not just its provenance.

Pure documentation surface. The browser verifier does not call any drift API; the kolm CLI runs the detection locally against your baseline. See /drift for the full lifecycle ladder.

Run it locally

$ kolm drift detect --artifact bundle.kolm --baseline baseline.snapshot.json --window 7d

[kolm drift] loaded current    drift-snapshot-v1  artifact=bundle.kolm
[kolm drift] eval_score                  delta=-0.004  within   (warn 0.02 / fail 0.05)
[kolm drift] k_score.composite           delta=+0.011  within   (warn 0.03 / fail 0.08)
[kolm drift] k_score.R                   delta=-0.062  drift    (warn 0.05 / fail 0.10)
[kolm drift] k_score.F                   delta=+0.018  within   (warn 0.05 / fail 0.10)
[kolm drift] external_holdout_hash       within   (exact match)
[kolm drift] tenant_shadow_corpus_hash   within   (exact match)
[kolm drift] artifact_hash               informational  (new build, expected)
[kolm drift] verdict: drift   breach_count=0  drift_count=1
[kolm drift] wrote drift-report-v1 -> drift-report.json
$ echo $?
0

Exit code is 0 on within/drift and 2 on breach so CI can gate on it (kolm drift detect ... || rollback.sh). Same JSON output as drift-report-v1 in the receipt chain.

The six axes & their floors

Every axis below is a constant in src/drift-supersession.js (W167) DEFAULT_TOLERANCES. Override per-axis with --tolerances tol.json; the verifier re-derives whichever values the report carries.

eval_score

Normalized scalar in [0, 1]. Tightest band: warn 0.02 / fail 0.05. Tightest because eval is a single number — even small swings move it meaningfully.

k_score.composite

Weighted sum of axes. warn 0.03 / fail 0.08. Slightly looser than eval because float noise across architectures adds 1e-4 per axis.

k_score.<axis>

Per-axis K-score (R, F, E, Z, T, A, S, L, C, V — whichever the manifest carries). warn 0.05 / fail 0.10. Loosest of the numeric bands.

external_holdout_hash

Exact-match comparison. Any byte change in the external holdout corpus immediately reports breach — the score was measured on a corpus that no longer exists.

tenant_shadow_corpus_hash

Exact-match comparison. Same semantics as external_holdout_hash but for the tenant's private shadow corpus. Hash mismatch → breach.

artifact_hash

Informational only. A new artifact always has a new hash by design — the field is reported in the signal so it appears in the report, but it never gates the verdict.

What gets checked

01 . manifest

Parse manifest.json. Confirm RS-1 schema version. Read declared model + adapter + verifier.

02 . CID

SHA-256 every file. Sort by canonical JSON. Recompute cidv1:sha256:hex(manifest). Match against declared CID.

03 . HMAC chain

Walk the receipt chain. Verify HMAC-SHA256 over (cid, input_sha, output_sha, ts) using the public verifier key.

04 . K-score

Recompute K = 0.40·A + 0.15·S + 0.15·L + 0.15·C + 0.15·V from this artifact's eval block. Check the gate.

05 . provenance

Confirm builder, base model digest, training timestamp, compile container image hash.

06 . signature

Verify the issuer signature against the kolm issuer pubkey baked into this page.

25 . corpus URL licensing gate

For each declared corpus source, confirm source_url parses and license sits in SAFE_LICENSES or AMBER_LICENSES (src/licensing-allowlist.js, wave 194). DENY (proprietary, scraped, tos-violated, unknown) fails; amber passes with a manual-review note.

Why this is a separate product

The CLI verifier is for compilers and operators. kolm-verify in the browser is for the auditor, the procurement reviewer, the insurance underwriter, the regulator. People who want to confirm an artifact is what it claims to be, without installing anything.

Same verification code path. Free forever. Works offline (download the page, run from a USB stick).

Works on your platform

The verifier uses crypto.subtle.digest, FileReader, and a stdlib ZIP reader. No external scripts, no upload, no installer. We've tested it on:

desktop

Chrome 119+, Edge 119+, Firefox 121+, Safari 17+ on macOS, Windows 10/11, Ubuntu 22.04+.

mobile

iOS Safari 17+ (iOS 17.4+), Chrome on Android 10+. Drop a .kolm from Files / Photos / Downloads.

offline

Save the page (right-click · Save As), copy to USB / air-gap network, open the .html locally. Same checks, no network.
