Overall verdict.

A drift report's verdict is the worst status across all signals: any breach → breach, any drift (no breach) → drift, otherwise within. The verifier (check #24) gates ship on this single value, the cron exits non-zero on breach, and the homepage K-score card surfaces the pill so operators see it at a glance.

The six tolerance axes.

Every drift report carries one signal per axis. Numeric axes (eval_score, k_score.composite, per-axis k_score.<axis>) classify delta against warn + fail bands. Hash axes (external_holdout_hash, tenant_shadow_corpus_hash) require exact match; artifact_hash is informational only (a new artifact ALWAYS has a new hash).

Default tolerances live in DEFAULT_TOLERANCES in src/drift-supersession.js. The recipe can override per axis at compile time; the active tolerance set is persisted into the drift report so a verifier replaying the comparison uses the same thresholds. The page is recipe-class neutral: the same six axes apply whether the artifact is class rule, synthesized_rule, compiled_rule, or distilled_model.

If you need to retire this artifact, pick one reason.

A successor's manifest carries supersession_provenance.reason declaring exactly one of the six values below. The enum is frozen in the backend (SUPERSESSION_REASONS in src/drift-supersession.js) so a verifier can reason about the chain without parsing free text. Builder + verifier both enforce the reason-specific evidence guards listed under each card.

drift_detected

M+3 detected breach; predecessor's K-score axes moved past tolerance against a tenant baseline. evidence required drift_signals array OR drift_report_hash (one or both); builder rejects without evidence; verifier rejects at ingest.

scheduled_rebuild

M+2 cadence triggered routine re-distillation. No drift detected; just the calendar said it was time. evidence required none beyond the cadence config block already on the manifest.

security_patch

Upstream model or runtime CVE; retire stale weights even if every drift signal sits inside the warn band. evidence required notes recommended (CVE id + advisory URL); authorized_by recommended.

recipe_revision

spec.json changed shape: new input field, new output schema, new redactor class. Not a pure retrain; the contract surface moved. evidence required notes recommended (link to spec diff); spec.json hash captured separately in seeds/recipe chain.

policy_change

Legal or compliance policy update: new redactor class added, license terms shifted, regulator request. evidence required authorized_by recommended (compliance-ops identifier); notes recommended (policy reference).

tenant_request

Explicit tenant or operator command to retire the artifact (wrong recipe shipped, customer requested rebuild). evidence required authorized_by recommended (operator identifier); notes recommended (ticket reference).

Reason-specific guard: reason === 'drift_detected' REQUIRES drift_signals or drift_report_hash per buildSupersessionBlock in the W167 backend. The builder throws at compile time; the verifier rejects at ingest. The retirement claim is auditable.

The receipt chain this report binds into.

The drift report is one node of the 14-node receipt chain. Each node hash binds into the next via artifact_hash_input; the last two nodes (supersession and drift_report) are the W167 lifecycle extensions. Any byte tamper drifts artifact_hash and breaks every downstream signature including the Rekor anchor.

spec → seeds → split → train → recipes → evals → external_holdout → tenant_shadow → auditor_attestation → supersession → drift_report → export → signatures → rekor

The two new nodes (supersession, drift_report) only emit on opt-in. Every pre-Wave-167 artifact lacks both blocks; the verifier emits informational pass with an upgrade hint instead of failing. No legacy .kolm regresses. When this page shows real values, both checks (#23 Supersession chain and #24 Drift report) are actionable.

What to do next.

Three paths keyed off the verdict. The example above renders the within-band path; the other two are documented here so operators know what is coming the day this surface goes red.

verdict within

No action required.

The artifact is inside every tolerance band on every axis. Verifier check #24 emits pass. Nothing ships, nothing rebuilds, no successor needed. The next scheduled detection run (per the cron config block) will recompute against the same baseline.

# nothing to run
$ echo verdict within
verdict within
verdict drift

Schedule a re-distillation against current traffic.

Movement past the warn band but inside the fail band. The artifact can keep shipping; check #24 emits WARN pass. The honest action is to capture a fresh seed batch and rebuild before the next cadence interval, so the new artifact is anchored to the current distribution rather than racing the fail band.

$ kolm distill --reuse-seeds \
    --baseline acme-jan.kolm \
    --capture ./captures/may.jsonl \
    --out acme-may-rebuild.kolm
verdict breach

Compile a successor with supersession bound in.

Fail band crossed on at least one axis. Verifier check #24 gates ship. Cron exits 2 so CI fails loud. The successor's manifest must carry supersession_provenance naming this artifact's hash, the reason drift_detected, and the breached signals inlined (or the drift report hash as evidence).

$ kolm compile --supersession-of=\
    sha256:abc…def \
    --supersession-reason drift_detected \
    --supersession-drift-report \
    drift-2026-05-17.json
