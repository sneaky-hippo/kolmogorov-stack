proof surface

Benchmarks that survive diligence.

kolm does not win by claiming a generic runtime is faster than Apple, Google, Microsoft, Meta, or MLC. It wins when a compiled .kolm artifact proves the task is shippable: smaller than the budget, fast enough on the target device, accurate on held-out examples, covered by recall, signed by receipts, and silent on the network at runtime.

01benchmark contract

Measure the artifact, not the slogan.

Each benchmark run produces JSON with artifact hash, device label, eval accuracy, p50/p95 latency, receipt integrity, and blocked runtime egress attempts. Any score without that evidence is marketing copy, not proof.

Quality

Task accuracy on locked held-out examples, refusal correctness where applicable, and grounded citation checks for recall-backed answers.

Footprint

Artifact bytes on disk, expanded runtime memory, model warm load, and added app bundle size per platform target.

Latency

Cold start, warm p50, p95, first-token latency, throughput, and battery-relevant sustained run behavior.

Coverage

Percent of structured tokens covered by deterministic recipes, recall hit rate, and fallback frequency.

Integrity

Manifest hash, signature verification, HMAC receipt chain, and verifier pass/fail for every measured output.

Privacy

Runtime egress must be zero. The harness records DNS/socket attempts and fails any run that contacts a network endpoint.

02device matrix

The minimum public matrix.

A production artifact must name its target devices. The first public matrix should cover mainstream Apple silicon, Android fragmentation, browser limits, and server fallback.

iOS / macOSCore ML target, Apple silicon, Neural Engine where available, Instruments trace attached.
AndroidLiteRT / NNAPI / vendor NPU path where supported, plus CPU fallback on a low-memory Android device.
WebWASM/WebGPU capability check, browser memory ceiling, asset streaming behavior, and offline cache verification.
DesktopmacOS, Windows, Linux CPU baseline with optional GPU delegate when supported by the artifact.
ServerSame artifact behind an API boundary for regulated teams that need VPC deployment before mobile rollout.
03competitive context

Where kolm must be honest.

Core ML, LiteRT, ONNX Runtime Mobile, ExecuTorch, and MLC LLM are serious infrastructure. kolm should not pretend otherwise. The differentiated benchmark is compile-to-artifact UX, personalization boundary, registry quality gates, and signed runtime receipts.

04publication rules

No fake charts.

Numbers should only ship when they are reproducible by a customer. The current local harness is available through kolm benchmark; hardware-specific public charts should still include raw JSON, artifact hash, and device notes.

Investor-grade gate

Publish a chart only when the artifact, device manifest, seed, test set hash, raw JSON, and receipt verifier are available. If any component is private, label the benchmark as internal and keep it out of acquisition or fundraising material.
