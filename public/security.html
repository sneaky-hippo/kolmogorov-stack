01shipped today · verifiable

What we have today.

Eight controls running in production. Each is linked to the artifact, endpoint, or document that proves it. If a control is on this list, an auditor can verify it without taking our word.

HIPAAlive

HIPAA Security Rule mapping, 22 standards.

45 CFR 164.308 through 164.316 mapped citation by citation to the kolm control that satisfies each one. Required vs Addressable flagged per regulation. Read the mapping.

GDPRlive

Article 28 DPA, EU SCCs incorporated.

Customer is Controller, kolm is Processor. EU Standard Contractual Clauses Module 2 by reference. UK IDTA addendum on request. Founder-signable within 48 hours. Read the DPA.

BAAlive

BAA with PHI Schedule 1, 10 clauses.

Purpose, Permitted Uses, Safeguards (4-layer), 10-business-day Breach Notification, Subprocessor flow-down, 30-day Return/Destruction, Audit Rights, Mitigation, Security Rule mapping, Survival. Default retention is 30 days, configurable down to 7 days per tenant. Read the schedule.

Subprocessorslive

Full inventory, 30-day change notice.

PHI is redacted by the customer-hosted bridge before any subprocessor is contacted. Vercel, Railway, Stripe, Resend, Cloudflare, GitHub never receive PHI. Anthropic, OpenAI, Google are opt-in only with your key. Email subscription for change alerts. Read the inventory.

Receiptslive

HMAC-SHA256 receipt chain, 4 rings.

Compile, run, eval, audit rings each carry an HMAC-SHA256 signature bound to the manifest, recipes, eval set, and receipt body. Tamper breaks verification. kolm inspect re-runs the chain offline. Read the spec.

K-score gatelive

Cryptographic gate, ge 0.85 baseline, ge 0.95 PHI.

K = 0.40A + 0.15S + 0.15L + 0.15C + 0.15V (accuracy, size, latency, cost, verification). The V component reads cve_audit.ok; an unpatched Critical CVE refuses the gate. Read the formula.

Compliance pkglive

One-call signed bundle for auditors.

GET /v1/account/compliance-package returns a signed JSON bundle: tenant, controls, BAA status, subprocessor inventory, HIPAA / SOC 2 / HITRUST / GDPR posture, receipt records, audit log, signed-at timestamp. API docs.

Isolationlive

Tenant boundary tested, cross-tenant 400.

Every record carries a tenant ID. Cross-tenant reads return 404 (existence-hiding) and cross-tenant writes return 400. Verified in scripts/sensitive-data-readiness.sh on every release. Read the architecture.

03decided not to pursue · with reasons

What we will not pretend to.

A short list of frameworks our buyers occasionally ask about and we have decided not to pursue. Each row says why. Saying it up front saves a procurement cycle.

CMMC Level 1+not pursuing

Defense and government are not our ICP.

CMMC compliance overhead would force pricing changes that hurt our healthcare, finance, and legal buyers. If you need a defense-controlled environment, run kolm self-hosted on your own GovCloud or on-prem infrastructure. The runtime supports it; we will not attest to CMMC ourselves.

FedRAMPnot pursuing

Same reasoning as CMMC.

FedRAMP Moderate or High is a multi-year, seven-figure program built for vendors whose ICP is federal civilian or DoD. That is not us. Government primes can deploy the runtime air-gapped and bring their own ATO. Defense deployment notes.

ISO 27001not pursuing today

SOC 2 Type II overlaps about 80 percent.

Honest answer: SOC 2 Type II covers most of what ISO 27001 covers, US buyers ask for SOC 2 vastly more often than ISO, and dual-attesting costs about 1.6x for the second one. We will revisit post-SOC 2 if EU buyers consistently request it.

PCI-DSSnot applicable

We never touch payment data.

Stripe handles all cardholder PII for billing. The kolm control plane sees a Stripe customer ID and a price plan, nothing more. Out of scope by design, not by attestation gap. See Stripe row.

04auditor · verification quickref

How an auditor verifies any of this.

Four lookups your auditor will run on day one. Each returns a verifiable artifact, not a slide.

VerificationHow to run it
Compliance package GET /v1/account/compliance-package with a tenant API key returns a signed JSON bundle covering controls, BAA status, subprocessor inventory, HIPAA / SOC 2 / HITRUST / GDPR posture, receipt records, audit log, signed-at timestamp. One file, one artifact, attestation chain inside.
Receipt chain kolm inspect <artifact>.kolm re-runs HMAC-SHA256 verification over the manifest, recipes, eval set, and receipt body using the tenant signing secret you archived at compile time. Tamper breaks the chain. Format spec.
Audit log GET /v1/audit/log?limit=N tenant-scoped, returns the 4-ring receipt-chain entries with HMAC signatures, sha256, K-score, and timestamps. Payload-free by design; no PHI in the log. Audit log docs.
SOC 2 Type I summary letter Email security@kolm.ai. Type I attestation letter target Q3 2026; pre-attestation we share the Halborn pentest report, Vanta posture snapshot, and CAIQ-Lite on MNDA, typical turnaround 1 business day.
05vulnerability disclosure · safe harbor

Found a flaw? Tell us first.

Continuous responsible disclosure.

Email security@kolm.ai; PGP-encrypt anything sensitive. Canonical contact and fingerprint at /.well-known/security.txt.

In scope: kolm.ai, compile pipeline, public registry and hub, runtime, SDK, CLI, RS-1 spec implementations, the receipt verifier, the K-score gate, the auth path, the Stripe billing path.

Out of scope: social engineering, denial of service, spam, third-party services we do not control (vendor issues should go to the vendor first).

Safe harbor: good-faith research consistent with this scope and our security.txt will not be pursued legally. Credit on the hall of fame with researcher consent.
