Three lanes, one boundary. Artifact crosses once. Receipts never cross.

In

Encrypted at rest in the compile pipeline. Deleted post-compile. Eval set retained 30 days for K-score audit, then purged. Regulated data requires per-deployment review.

Through

API key is pass-through; we never persist your key, prompts, or completions. Teacher outputs become labels and dissolve into the LoRA.

Out

One signed file. You download it. Build inputs deleted. Users run it offline; receipts never leave their device.

02Three layers of provenance

Signed at compile.
Verified by receipt.

Each layer independent. Inspect offline with one command; receipt re-verification needs the shared tenant secret you archived at compile time (Ed25519 public-key receipts are roadmap).

Layer 01

HMAC-SHA-256 over model pointer, recipes, eval set, manifest, receipt body. Tampering breaks the chain. kolm inspect prints it; kolm inspect re-runs against the artifact you hold.

$ kolm inspect support-triage.kolm
ok manifest.signature_alg hmac-sha256
ok receipt.signature_alg  hmac-sha256
ok anchors []
Layer 02

Registry record.

Public metadata via registry export, diffable against artifact bytes. Each entry carries manifest hash and signature algorithm.

$ curl -fsSL https://kolm.ai/v1/registry/export
ok public recipe metadata export
Layer 03

External anchor extension.

Receipt schema carries an anchor field for transparency-log roots and third-party timestamps. Enterprise add-on; empty on Free / Pro.

$ kolm inspect support-triage.kolm
anchors  []
ok reserved extension field

08Penetration testing

Halborn external review.
Findings shipped.

Pre-release penetration test conducted 2026-04 by Halborn (formerly Halborn Security). 7 findings: 0 Critical / 1 High (fixed pre-publish) / 3 Medium (fixed) / 3 Informational (acknowledged). Full report on request to security@kolm.ai with NDA.

Scope2026-04-08 to 2026-04-22

14-day engagement.

Runtime API surface (54 endpoints), compile pipeline, registry, Stripe billing path, cross-tenant isolation, JWT + API key auth, receipt verification, RS-1 spec implementation.

MethodologyOWASP Top 10 + ASVS 4.0

Authenticated + unauthenticated.

Two phases: black-box external scan, then credentialed deep dive with source-code review. Mutation-based fuzzing on the receipt verifier and the manifest parser. Threat model handed off pre-engagement.

Findings0C / 1H / 3M / 3I

One High: receipt-replay window.

Receipts could be replayed within their 5-minute issuance window if intercepted. Fix: bound replay to (cid, nonce) tuple at receipt issuance; verifier rejects duplicate nonce. Patched in v10.4 before public launch.

Retest2026-05-06

All P0/P1 findings closed.

2-week remediation window. Halborn retest report attached to the original engagement. 6 of 7 findings closed by 2026-05-06; remaining Informational acknowledged in /security/researchers.

Cadenceannual + on major

Pentest before every major.

Re-test on v12, v13, or sooner if the runtime API surface changes by >25%. Continuous bug bounty between formal engagements.

Report accesson NDA
