RS-1-receipts · MIT · verifiable offline

Security by receipt.

Every .kolm artifact is HMAC-signed over its emitted components; receipts verify from the artifact bytes. Base-model, LoRA, recall-index, and external-registry claims become production evidence only when the corresponding bridge ships.

01Architecture · data flow

Compile runs in our cloud (or your VPC on Enterprise). Runtime is the Node CLI and MCP server, on-device. Post-compile only the manifest hash crosses the boundary: not the artifact, not the corpus, not the receipts. Customer-hosted compile and native offline runtimes pending contract review.

Three lanes, one boundary. Artifact crosses once. Receipts never cross.

In

Encrypted at rest in the compile pipeline. Deleted post-compile. Eval set retained 30 days for K-score audit, then purged. Regulated data requires per-deployment review.

Through

API key is pass-through; we never persist your key, prompts, or completions. Teacher outputs become labels and dissolve into the LoRA.

Out

One signed file. You download it. Build inputs deleted. Users run it offline; receipts never leave their device.

02Three layers of provenance

Signed at compile.
Verified by receipt.

Each layer independent. Verify offline, single command, no service.

Layer 01

HMAC-SHA-256 over model pointer, recipes, eval set, manifest, receipt body. Tampering breaks the chain. kolm inspect prints it; kolm verify re-runs against the artifact you hold.

$ kolm inspect support-triage.kolm
ok manifest.signature_alg hmac-sha256
ok receipt.signature_alg  hmac-sha256
ok anchors []
Layer 02

Registry record.

Public metadata via registry export, diffable against artifact bytes. Each entry carries manifest hash and signature algorithm.

$ curl -fsSL https://kolm.ai/v1/registry/export
ok public recipe metadata export
Layer 03

External anchor extension.

Receipt schema carries an anchor field for transparency-log roots and third-party timestamps. Enterprise add-on; empty on Free / Pro.

$ kolm inspect support-triage.kolm
anchors  []
ok reserved extension field
