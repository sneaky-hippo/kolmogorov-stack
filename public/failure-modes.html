Failure modes

What happens when something goes wrong.

Compilers fail in clean ways. So does kolm. Below: every error path, the exit code, the receipt status, and the action you should take.

01compile-time

Compile-time failures (the artifact never ships).

Every compile is gated by the K-score floor and the verifier. If the artifact does not meet the gate, the compile fails closed - you never get a signed binary that under-performs.

FM-001fail-closedk_score < floor

K-score below the gate.

The compiled artifact would not pass your eval set at the K-score floor (default 0.85). The compile fails closed: no signature, no receipt, no .kolm emitted. The job report shows the eval breakdown and the highest scoring sub-recipe.

exit code22 (kolm: gate-not-met) http status422 Unprocessable Entity artifactnot produced receiptnone actionadd more gold examples, lower the floor explicitly with --k-score-floor, or split the task 
$ kolm compile "..." --examples gold.jsonl
[compile] k_score=0.71 below floor=0.85
[compile] FAIL - artifact not signed.
[hint] add 30+ more examples or run: kolm compile ... --k-score-floor 0.70
FM-002degradedcorpus.unreadable

Corpus has unreadable files.

One or more files in the corpus directory failed to parse: corrupt PDF, unsupported audio codec, broken image, encrypted file without key. The compile continues with the readable subset; the failed paths are listed in compile.log and in the manifest's skipped field.

exit code0 (with warnings) http status200 with warnings: [...] artifactproduced (subset) receiptissued; corpus_completeness field reflects reality actionre-encode or remove the failing files; re-compile if completeness matters
FM-003degradedfrontier.quota_exceeded

Frontier API key out of quota.

Your Anthropic / OpenAI / Google key returned 429 mid-compile. The job pauses for up to 30 minutes and resumes from the last checkpoint when quota replenishes; if it does not, the job fails with a partial-progress report.

exit code23 (kolm: frontier-quota) http status503 Service Unavailable (with retry_at) artifactnot produced receiptnone actiontop up the frontier key, or pass --frontier-key with a different vendor's key
FM-004fail-closedexamples.format_invalid

Examples don't match the spec.

JSONL parse error, missing required fields, mismatched output schema. The compile fails fast before consuming frontier tokens.

exit code2 (kolm: bad-input) http status400 Bad Request artifactnot produced receiptnone actionrun kolm lint examples.jsonl for the per-line error report
FM-005fail-closedbase_model.unavailable

Base model not found in registry.

The base model tag (qwen3-7b, llama-3-8b, etc.) is not in the public Kolmogorov registry. The compile fails fast.

exit code2 (kolm: bad-input) http status400 Bad Request actionrun kolm models to see the registry list, or specify a sha-pinned local path
02run-time

Run-time failures (after a signed artifact ships).

Once a .kolm is signed, runtime failures are bounded: signature mismatch (corruption / tampering), input-out-of-distribution (refusal vs. wrong answer), or environment incompatibility (RAM, codec, OS).

FM-101fail-closedsignature.invalid

Signature does not verify.

The artifact's HMAC chain does not match the registry's published key. The runtime refuses to load. Possible causes: corrupt download, tampering, expired registry secret rotation.

exit code31 (kolm: signature-invalid) artifactrefused at load actionre-download from the original tenant's signed source; if persistent, file an issue with the manifest hash 
$ kolm run patient-intake.kolm "..."
[run] FAIL - signature does not match published HMAC chain.
[run] manifest_hash=sha256:...  expected=...
[hint] this artifact may be tampered. Do not trust output.
FM-102degradedverifier.rejected

Verifier rejects the output.

The artifact ran, but the embedded verifier scored the output below the artifact's K-score floor. The runtime returns the output with a low-confidence flag and a refusal-by-default policy unless the caller passes --allow-low-confidence.

exit code42 (kolm: low-confidence) artifactran successfully receiptissued; verifier_score field below the gate actiontreat as out-of-distribution; route to a human or escalate to a frontier call
FM-103fail-closedruntime.oom

Out of memory.

The artifact's base + LoRA + recall index exceeded available RAM. The runtime fails fast at load with the required vs. available footprint.

exit code12 (kolm: oom) actionre-compile with a smaller base (--base qwen3-3b) or a quantized variant (--quant int4)
FM-104fail-closedruntime.unsupported_arch

Unsupported architecture.

The artifact was compiled with a target tag (e.g., cuda-sm89) that does not match the local GPU/CPU. The runtime fails fast with the required vs. detected target.

exit code14 (kolm: arch-mismatch) actionre-compile with --target portable for the slowest-but-everywhere build
03registry

Registry & receipt failures.

The public Kolmogorov registry anchors signature verification. If it goes down, already-issued artifacts continue to run; new compiles and verifications block until it recovers.

FM-201warningregistry.unavailable

Registry is down.

The public registry returns 5xx or times out. Already-loaded artifacts continue to run on cached signatures. New compiles block; new kolm run against an unfamiliar artifact pauses for up to 5 minutes, then runs in unverified mode if you pass --allow-unverified.

artifact (loaded)continues artifact (cold load)blocks 5 min, then refused unless --allow-unverified compilequeued; runs when registry recovers receipt verifyoffline path: works against archived public key; online: blocks actioncheck /status; for air-gapped deployments, the registry is mirrored at compile time
FM-202warningreceipt.secret_rotated

Receipt secret rotated.

The HMAC secret on the registry was rotated. Receipts issued before the rotation cannot be re-verified by the public endpoint. They remain verifiable offline against the archived secret bundled in the original artifact.

http status410 Gone (on /v1/receipts/verify) actionuse kolm verify --offline receipt.json artifact.kolm
FM-203fail-closedreceipt.malformed

Receipt body is malformed.

Required fields missing or types wrong. Always treat as not-verified.

http status400 Bad Request actionre-export the receipt from the artifact's receipts/ dir
04auth

Auth & quota failures.

FM-301fail-closedauth.bearer_invalid

Bearer key not recognized.

Either the key was rotated, the tenant was deleted, or the request was made against the wrong host.

http status401 Unauthorized actionkolm login to refresh, or POST /v1/account/rotate-key
FM-302fail-closedquota.exhausted

Compile quota exhausted.

Free: 3 compiles per month. Starter: 25. Pro: 200. Teams: 1000. Enterprise: unlimited. Existing artifacts keep running; new compiles block.

http status402 Payment Required actionUpgrade or wait for the monthly reset
FM-303fail-closedrate_limit.signup

Signup rate-limited.

10 signups per IP per 24 hours. Prevents enumeration / abuse.

http status429 Too Many Requests actionwait for the rolling window or sign up from a different network
05action

If you hit something not on this page.

Open an issue with the compile id and the last 50 lines of ~/.kolm/log/compile.log. We answer within one business day. /troubleshooting covers the common ones.
