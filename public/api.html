Authentication

Bearer-token auth on every /v1 endpoint that touches tenant state. Public endpoints (health, public registry, receipts/verify) have no auth.

Bearer keys
X-API-Key: ks_<your-key>
# or, equivalently (for SDK compatibility):
Authorization: Bearer ks_<your-key>

X-API-Key is the canonical header. Authorization: Bearer is also accepted for SDK compatibility. Issued at signup. Rotate via POST /v1/account/rotate-key. Stored as a scrypt hash - we never see your raw key after rotation.

Browser sessions
Cookie: kolm_session=<httpOnly-cookie>

Web dashboard uses an httpOnly session cookie. SameSite=Lax, 30-day rolling. POST /v1/session/login issues; POST /v1/session/logout revokes.

HTTP error codes

Documented endpoints aim to return errors as JSON: { "error": "<message>", "code": "<machine_code>" }. Some routes still return ad-hoc { "error" } shapes; the generated route manifest is the source of truth. A request_id envelope field and matching response header are target architecture. No request-id middleware ships today, so do not key alerting on it yet.

2xxSuccess. 3xxRedirect or not-modified. 400Bad request: malformed body, missing fields, invalid types. 401Missing or invalid bearer / session. 403Authenticated but not authorized (wrong tenant, signup disabled). 404Not found. 409Conflict: duplicate id, signature mismatch, version skew. 422Semantically wrong: k-score below floor, verifier rejects. 429Rate-limited or quota exhausted on current plan; upgrade or wait for reset. Retry-After header indicates when. 500Server error. File an issue with the request path, payload shape, and timestamp; correlated request_id headers are target architecture. 503Upstream dependency unavailable (frontier API down, secret unset).

Account & plans

Self-serve provisioning, plan changes, cancellation, deletion. No mailto, no scoping calls.

POST/v1/signuppublic

Provisions a new tenant. Returns the bearer key plus the Stripe Payment Link for paid plans (or trial mode if the link isn't configured).

Request
{
  "email": "you@example.com",
  "name":  "your-org",
  "plan":  "free | starter | pro | teams | enterprise"
}
Response (201)
{
  "tenant": {
    "id": "...",
    "name": "your-org-x4f8",
    "plan": "pro",
    "quota": 200000,
    "seats": 1,
    "trial_ends_at": "2026-06-08T00:00:00Z"
  },
  "api_key": "ks_...",
  "billing_url": "https://buy.stripe.com/...",
  "billing_required": true
}
Errors
400valid email required 403public signup disabled (INVITE_ONLY=true) 42910 signups per IP per 24 hours exceeded

POST/v1/account/change-planbearer

Self-serve upgrade or downgrade. Returns the new Stripe Payment Link if the plan requires one. Free is always free.

Request
{ "plan": "teams" }
Response (paid target, pending checkout)
{
  "ok": true,
  "plan": "pro",
  "pending_plan": "teams",
  "billing_url": "https://buy.stripe.com/...",
  "billing_required": true,
  "message": "Complete payment at  to activate the Teams tier."
}

Switching to free downgrades immediately and returns {ok, plan: "free", quota, seats, billing_url: null, billing_required: false}. Paid targets return the shape above; the tenant flip happens when the Stripe webhook confirms payment.

Errors
400unknown plan 401missing/invalid bearer
POST/v1/account/cancelbearer

Drops the tenant to the free tier at the end of the paid period. The key keeps working; quotas drop to free limits at period end.

Response
{ "ok": true, "downgrades_to": "free", "effective_at": "2026-06-01T00:00:00Z" }
POST/v1/account/deletebearer

Soft-deletes the tenant. The key stops working immediately. Artifacts already issued continue to verify.

Request
{ "confirm": "DELETE my-org" }
Response
{ "ok": true, "deleted_at": "2026-05-08T17:30:00Z" }

Compile

The orchestrator: synthesize a verifier, k-sample the frontier, observe deterministic patterns, sign, bundle. Returns a job id; polls become .kolm files. Adapter/LoRA fit is a roadmap tier; v0 emits recipe-tier artifacts with signed model-pointer records.

POST/v1/compilebearer

Starts a compile job. Synchronous compiles return when the artifact is ready; async returns a job id and you poll.

Request
{
  "task":             "summarize patient intake notes for triage",
  "examples":         [{"input": "...", "output": "..."}],
  "corpus_namespace": "tenant/patient-corpus",
  "base_model":       "qwen3-7b",
  "k_score_floor":    0.85,
  "mode":             "async"
}
Response (202)
{ "job_id": "cmp_8f2a...", "status": "queued", "poll": "/v1/compile/cmp_8f2a..." }
Errors
400invalid task / examples / base_model 422k_score below floor - artifact gated, not signed 429quota exhausted on current plan (or rate-limited)
GET/v1/compilebearer

List recent compile jobs for the current tenant.

Response
{ "jobs": [
  { "id": "cmp_...", "status": "ready", "k_score": 0.91, "size_mb": 142, "created_at": "..." },
  { "id": "cmp_...", "status": "running", "stage": "k-sample", "created_at": "..." }
]}
GET/v1/compile/:idbearer

Job status. Stages: queued → k-sample → verify → lora-fit → bundle → sign → ready. ready means the artifact is downloadable.

Response (ready)
{
  "id":           "cmp_8f2a...",
  "status":       "completed",
  "k_score":      0.913,
  "size_mb":      142,
  "manifest":     "sha256:...",
  "signature":    "hmac-sha256:...",
  "artifact_url": "/v1/compile/cmp_8f2a.../.kolm"
}
GET/v1/compile/:id/.kolmbearer

Streams the signed .kolm archive. Content-Type: application/zip. Includes manifest, recipes, evals, signature, receipt.

GET/v1/artifactsbearer

List artifacts the tenant has compiled. Includes hash, size, base model, k-score, signing time.

Run

Local-first. Use kolm run on the artifact when you can. The HTTP run endpoint is for the cloud runtime - same code path, different host.

POST/v1/runbearer

Execute a registered version (or concept) against an input. Returns the output, latency, cache flag, and an optional HMAC-bound receipt.

Request
{
  "concept_id": "cpt_8f2a...",   // or "version_id": "ver_..."
  "input":      { "note": "patient presents with..." },
  "use_cache":  true,             // optional, defaults true
  "receipt":    true              // optional, defaults true
}
Response
{
  "output":     { "triage": "urgent", "confidence": 0.91 },
  "version_id": "ver_...",
  "latency_us": 142000,
  "cache_hit":  null,
  "receipt": {
    "spec":            "rs-1",
    "input_hash":      "sha256:...",
    "output_hash":     "sha256:...",
    "version_id":      "ver_...",
    "runtime_version": "0.2.0",
    "issued_at":       "2026-05-08T17:30:00Z",
    "hmac":            "..."
  }
}
POST/v1/wrap/verifiedbearer

k-sample a frontier model and pick the candidate that passes a deterministic verifier (test cases or recipe-as-judge). Drop-in shape for messages.create; verified is the kolm extension. Used during compile and as a standalone "verified inference" surface.

Request
{
  "messages":          [{ "role": "user", "content": "extract invoice line items as JSON" }],
  "system":            "You are a code generator...",   // optional
  "model":             "claude-opus-4-7",
  "max_tokens":        2048,                              // optional
  "temperature":       0.7,                               // optional
  "verified":          { "k": 4, "test_cases": [...] },  // or { judge_recipe_id, expected }
  "corpus_namespace":  "tenant/patient-corpus"            // optional Recall grounding
}
Response
{
  "id":          "wrap_...",
  "model":       "claude-opus-4-7",
  "role":        "assistant",
  "stop_reason": "end_turn",                  // or "verifier_unsatisfied"
  "content":     [{ "type": "text", "text": "..." }],
  "_kolm": {
    "verified":      true,
    "chosen":        { "source": "...", "passes": 4, "total": 4 },
    "candidates":    [...],
    "cost_usd":      0.0123,
    "elapsed_ms":    1840,
    "recall_chunks": 0,
    "receipt":       { "spec": "rs-1", "primitive": "verified-inference", ... }
  }
}
POST/v1/recallbearer

Top-k retrieval over a tenant corpus. Multimodal: text, image, audio, video, PDF.

Request
{
  "namespace": "tenant/patient-corpus",
  "query":     "patients reporting chest pain in the last week",
  "k":         8
}
Response
{
  "namespace": "tenant/patient-corpus",
  "n":         3,
  "chunks": [
    { "path": "...", "snippet": "...", "score": 0.91 }
  ]
}
POST/v1/embedbearer

Tokenize-and-ingest absolute paths from a server-mounted corpus into a tenant namespace. Routes to the right embedder by modality. Server-mounted absolute paths only; SaaS upload is on the Sprint 2 roadmap.

Request
{
  "namespace": "tenant/patient-corpus",
  "paths":     ["/abs/path/to/doc1.pdf", "/abs/path/to/notes.md"],
  "force":     false
}
Response
{ "indexed": 2, "skipped": 0, "namespace": "tenant/patient-corpus" }

Capture & distill

Drop-in proxy endpoints that mirror upstream Anthropic and OpenAI shapes. Every call you forward through us is recorded as an (input, output) pair under a tenant namespace. At 1,000 pairs the namespace is eligible for auto-distill (preview) · we hand off to a training bridge and ship you a signed .kolm you keep forever; LoRA packaging requires the bridge and may return 503 in v0. Your frontier API key never touches our store; we forward it via x-upstream-api-key and strip it before logging.

POST/v1/capture/anthropicbearer

Mirror of the Anthropic Messages API. Forward your request through this URL with your real Anthropic key in x-upstream-api-key; we proxy upstream, return the upstream response verbatim, and record the (prompt, completion, model, latency) tuple under x-kolm-namespace.

Headers
X-API-Key:           ks_<your-kolm-key>
x-upstream-api-key:  sk-ant-<your-anthropic-key>
anthropic-version:   2023-06-01
x-kolm-namespace:    support-classifier   # optional; defaults to "default"
Request
{
  "model":      "claude-sonnet-4-6",
  "max_tokens": 1024,
  "messages":   [{ "role": "user", "content": "Classify: checkout fails on Android" }]
}
Response
// Upstream Anthropic response, verbatim. Plus two response headers:
//   x-kolm-capture-id: obs_8f2a...
//   x-kolm-namespace:  support-classifier
Errors
400missing messages array 401missing or invalid kolm bearer key 502upstream Anthropic unreachable or returned non-JSON
POST/v1/capture/openaibearer

Mirror of the OpenAI Chat Completions API. Same shape as /v1/capture/anthropic with the OpenAI request body. Your real OpenAI key MUST come in x-upstream-api-key · we never fall back to the Authorization header here.

Headers
X-API-Key:          ks_<your-kolm-key>
x-upstream-api-key: sk-<your-openai-key>
x-kolm-namespace:   ticket-router   # optional
Request
{
  "model":    "gpt-4.1",
  "messages": [{ "role": "user", "content": "Route: invoice export request" }]
}
Response
// Upstream OpenAI response, verbatim. Plus two response headers:
//   x-kolm-capture-id: obs_3e1b...
//   x-kolm-namespace:  ticket-router
Errors
400missing messages array 401missing or invalid kolm bearer key 502upstream OpenAI unreachable
GET/v1/labels/synthesize-corpusbearer

Download captured pairs for a namespace as JSONL or a JSON envelope. Pass ?count_only=1 for a cheap header-style probe (returns count, ready_to_distill, threshold). The CLI command kolm capture status wraps this.

Query
namespace   string   default "default"
format      string   "jsonl" | "json"   default "jsonl"
limit       int      1..50000   default 10000
count_only  bool     "1" | "true"
Response (jsonl, default)
// Content-Type: application/x-ndjson
// Header: x-kolm-count: 4287
{"input":"...","output":"...","model":"claude-sonnet-4-6","provider":"anthropic","latency_us":812000,"created_at":"2026-05-09T..."}
{"input":"...","output":"...","model":"gpt-4.1","provider":"openai","latency_us":740000,"created_at":"2026-05-09T..."}
...
Response (count_only)
{
  "namespace":         "support-classifier",
  "count":             4287,
  "ready_to_distill":  true,
  "threshold":         1000
}
POST/v1/specialists/auto-distillbearerpreview

Preview. Kick off an adapter training run against a namespace's captured corpus. Locked under 1,000 pairs (returns 400 with a count). Shipped today: the route counts pairs by corpus_namespace and, when KOLM_TRAINER_BRIDGE_URL is configured, forwards a job to the external bridge with a callback URL. Target architecture: a server-owned status endpoint, captured-job ledger, signed-artifact return path through /v1/compile/:id/.kolm, and triage-binding (only kept non-error rows counted). Without the bridge the route returns 503.

Request
{
  "namespace":   "support-classifier",
  "base_model":  "qwen2.5-coder-7b-instruct",
  "target_size": "phi-3-mini"
}
Response (200, bridge configured)
{
  "ok":          true,
  "job_id":      "distill_4f8a...",
  "namespace":   "support-classifier",
  "base_model":  "qwen2.5-coder-7b-instruct",
  "target_size": "phi-3-mini",
  "pair_count":  4287,
  "bridge":      "external"
}

Note: the bridge owns delivery today; a server-side /v1/specialists/auto-distill/:id/status endpoint is target architecture, not shipped.

Errors
400not enough captures · { count, threshold: 1000, message } 401missing bearer 502training bridge returned a non-2xx 503distill_bridge_not_configured · on-prem trainer ships in Wave 2; email hello@kolm.ai for hosted access

Receipts

Every /v1/run output ships with an HMAC-bound receipt. In v0.1, the issuer or any holder of the shared tenant receipt secret can re-verify offline; Ed25519 public-key receipts are a roadmap tier. This endpoint is the server-side recompute for convenience.

POST/v1/receipts/verifypublic

Public endpoint that recomputes the receipt HMAC server-side using the service's receipt secret. Unauthenticated convenience verifier · not the same as independent public-key verification (Ed25519 receipts are roadmap).

Request
{ "receipt": { "kolm_version": "0.1", "artifact_hash": "...", "chain": [...], "signature": "..." } }
Response
{
  "verified": true,
  "reasons":  [],
  "mode":     "drive-by"   // or omitted for full-receipt mode
}
Errors
400malformed receipt 409signature does not match (tampered) 410receipt secret rotated - this receipt cannot be re-verified by us; verify offline against your archived secret

Registry

The public artifact and recipe index. Read-only on this surface; submit through the CLI or /v1/publish.

GET/v1/registry/publicpublic

Browse the public registry: featured recipes, base-model fingerprints, RS-1 spec entries.

GET/v1/registry/exportpublic

Bulk export of the registry as NDJSON. Rate-limited to 60/min/IP. Cache headers honored.

Health

Probe endpoints. Use /health for liveness, /ready for readiness (dependencies + secrets).

GET/healthpublic

Liveness probe. Always 200 unless the process is dead.

Response

GET/readypublic

Readiness probe. Returns the state of all upstream dependencies (frontier API, registry, embedder, store, receipt secret). 200 when ready, 503 when a required check fails.

Response
{
  "status":          "ready",
  "production_like": false,
  "checks": [
    { "name": "receipt_secret",  "ok": true,  "required": true,  "label": "RECIPE_RECEIPT_SECRET" },
    { "name": "anthropic_key",   "ok": true,  "required": false, "label": "ANTHROPIC_API_KEY" },
    { "name": "store",           "ok": true,  "required": true,  "label": "store" }
  ]
}
