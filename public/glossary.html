Glossary - plain definitions

The kolm vocabulary.

Everything you need to read the docs without flipping back. If a term used on this site is not here, mail dev@kolm.ai.

Artifact

.kolm file

The output of a kolm compile. A signed ZIP containing the manifest, recipe pack, recall index, verifier, receipt chain, and (optionally) base model + LoRA. Portable; runs on phone, laptop, browser, server, or edge box. Same byte-exact file in every place.

See the anatomy →

BYOK

bring-your-own-key

The pricing model where you pay your own frontier model bill (Anthropic, OpenAI, Google) directly. kolm proxies the call, never resells tokens, and never marks up frontier inference. You own the relationship with your teacher.

Compile

kolm compile

The action that produces a .kolm artifact. Takes a task description, examples, optional corpus, and a frontier API key. Synthesizes a verifier, runs k-sample inference under it, packages the result, signs it. The expensive frontier work happens here, once.

Distillation

The process of teaching a small student model the behavior of a large teacher model on a specific task. The teacher answers your examples; the verifier accepts or rejects each answer; the accepted pairs become training data for a LoRA. The result is a small, task-specific model that behaves like the frontier on your data.

Frontier teacher

The big model you bring at compile time. Anthropic Claude, OpenAI GPT, Google Gemini, your own Hermes-3-405B endpoint. The teacher is consulted once at compile, not on every runtime request. You pay for inference at compile time; runtime is free.

K-score

release score

A blended quality score from 0 to 1 across five sub-axes: accuracy, coverage, size, latency, cost. Measured on a held-out test set. One number on the artifact cover. Five you can defend underneath.

/k-score →

K-gate

release threshold

The minimum K-score required for an artifact to ship. Default 0.70. You can raise it (regulated workloads) or lower it (research preview). An artifact under the gate fails the build; it does not promote silently.

.kolm file

artifact, specialist

The file extension for a compiled kolm artifact. ZIP-format, manifest-first. Mounts cleanly under any sandbox. Inspect without running with kolm inspect <path>.kolm.

Manifest hash

The SHA-256 of manifest.json inside the artifact. The first link in the receipt chain. If anyone replaces the artifact with a different one, the manifest hash differs and verification fails immediately.

MCP

Model Context Protocol

The Anthropic-led protocol for letting agents call external tools. kolm exposes a recipe-mcp server so any MCP-aware agent (Claude Desktop, Cursor, Zed) can synthesize, run, and verify recipes as native tools.

/integrations →

Receipt chain

An HMAC-SHA256 chain over every output the artifact produces. Each receipt depends on the previous one; tampering with any output invalidates every later receipt. The chain is what makes a kolm output cryptographically auditable.

/threat-model →

Recall index

corpus index

A multimodal embedding index of approved context (text, image, audio, video, PDF). Bundled inside the artifact when you compile with --data. The runtime queries it locally during inference; no network call required.

Recipe pack

deterministic drafts

The set of deterministic token drafts a kolm artifact carries for fast inference. Indexed by prefix-shape and embedding-shape of the input. Used for speculative decoding at runtime - the runtime can predict whole spans of output without waking the model up. This is the compute moat.

Registry

atlas

The public index of compiled artifacts. Browseable at /atlas. Each entry has a K-score, manifest hash, tags, and (where public) downloadable artifact. Acts as the discovery surface and the recipe-pack flywheel - every public compile feeds the deterministic-pattern set.

RS-1 spec

recipe spec v1

The open spec for the .kolm file format and receipt chain. MIT-licensed. Defines manifest-v0.1.json, receipt-v0.1.json, the verifier contract, and the wire formats. Read it at /docs/rs-1.md.

Serve

kolm serve --mcp

The action that exposes a .kolm artifact as a runnable surface. kolm serve binds a local HTTP endpoint; kolm serve --mcp exposes the artifact as an MCP tool that any agent can call. Inference happens locally; receipts are emitted to disk.

/serve →

Verifier

test gate

A deterministic test function bundled with the artifact that decides whether an output is correct. Synthesized at compile time from your examples. Used (1) at compile to decide which frontier samples become training data, (2) at release to compute K-score on held-out tests, (3) at runtime to verify outputs before they ship.
