Glossary

The terms kolm uses.

Defined precisely. If a term in the docs is missing here, it is not load-bearing or it has been deprecated.

.kolm noun · the artifact

The signed zip produced by kolm compile. Seven entries: model weights, LoRA adapter, recipe pack, recall index, verifier, manifest, signature. Runs on any kolm runtime. /anatomy.

kolm noun · the binary

The CLI. Invoked as kolm in a shell. Subcommands: init, new, compile, run, eval, benchmark, score, inspect, serve, publish, capture, distill, install, tune, rag, doctor, logs, query, version. To Kolmogorov what git is to Git or gcc is to GCC.

Kolmogorov noun · the company

The team and product surface that ships kolm. Named for Andrey Kolmogorov, who in 1965 defined the size of the smallest program that produces a given output. That is the question the compiler answers.

K-score noun · the gate

A weighted scalar over five sub-scores measured at compile time: K = 0.40·A + 0.15·S + 0.15·L + 0.15·C + 0.15·V where A is accuracy on held-out evals, S is size penalty, L is latency, C is cost-per-call, V is recipe coverage. Default ship gate is 0.85; safety-critical 0.95; prototype 0.70. /k-score.

recipe noun · a specialist routine

A small reusable unit inside an artifact that handles a known sub-task without invoking the full model. The recipe pack is one of the seven layers. Recipes are extracted from observation traces during compile.

recall noun · the corpus index

The sqlite-vec index baked into the artifact. Each row is an example or fact the model can ground on at inference time. Built by deduplicating and embedding the input corpus.

verifier noun · the gate function

A small program embedded in the artifact that decides whether a candidate output for a given input is correct. May be a regex, a JSON-schema check, a deterministic test, or a tiny LM judge. Called by kolm eval and during k-sample voting.

k-sample noun · verified inference

The compile-time procedure: sample k candidates from the teacher, run each through the verifier, take the winning candidate as the label. Default k=8. Higher k buys label quality at the cost of teacher tokens.

distill verb · hand a corpus to the trainer

kolm distill (and the underlying POST /v1/specialists/auto-distill) hands a tenant's namespace corpus to an external trainer bridge once it crosses 1,000 captured pairs. The bridge owns the fit, scoring, and signing today; the route returns 503 without it. A closed-loop, server-owned distill that returns a signed .kolm directly through /v1/compile/:id/.kolm is target architecture.

manifest noun · the metadata

The JSON header inside the artifact. Lists each layer's hash, size, role, and lineage (teacher version, base model, compile parameters, K-score sub-scores). Signed as ring 4 of the receipt chain.

receipt noun · the per-call run metadata

Shipped today: each kolm run invocation emits a _kolm block carrying input hash, output hash, timestamp, artifact id, and K-score at ship time, alongside the artifact's HMAC-SHA256 signature over its archive. The issuer or any holder of the shared tenant secret can re-verify. Target architecture: per-call HMAC chaining across requests and Sigstore Rekor anchoring for tamper-evident timestamping.

receipt chain noun · four rings

Four nested HMAC stages that link a runtime receipt back to the original artifact: ring 1 (model weights), ring 2 (adapter), ring 3 (recipes + recall), ring 4 (manifest). kolm inspect re-derives all four in about 80 ms.

capture verb · proxy and record

Route frontier API calls through /v1/capture/anthropic or /v1/capture/openai so the (input, output) pair is recorded under a corpus_namespace. The recorded pairs become labels for a future distill. capture-and-distill.

RS-1 noun · the spec

The artifact-format specification for the .kolm file: layer schema, manifest fields, signature scheme. MIT-licensed and versioned independently of the compiler.

frontier model noun · the teacher

A large hosted model used as the teacher during compile (Anthropic Claude, OpenAI GPT, Google Gemini, your own). You bring the API key. Tokens are spent during k-sample, not at runtime.

base model noun · the student

The open-weight model the LoRA is trained on top of. Typical choices: Phi-3-mini, Qwen2.5-3B, Llama-3.2-3B for size; Qwen2.5-7B, Hermes-3-8B for capability. The base hash is recorded in the manifest.

LoRA noun · the adapter

A small low-rank weight delta applied on top of the base model at runtime. Typical size 50 to 200 MB. Stored as adapter.bin in the artifact.

MCP noun · Model Context Protocol

The protocol an artifact speaks when started with kolm serve --mcp. Clients (claude code, cursor, continue, cline) connect over stdio by default; pass --http to bind 127.0.0.1:8765. SSE transport and zeroconf/mDNS discovery are target architecture, not shipped today.

compile flywheel noun · the loop

The capture-and-distill loop. Shipped today: every proxied call lands in the tenant corpus, and a tenant can hand a 1000-pair namespace to an external trainer bridge. Target architecture: closed-loop in-app distill, triage-bound exports (kept-only), and revocation. The thing you buy when you pay kolm is the rails, not the compute.

airgap mode noun · hard offline

Set KOLM_AIRGAP=1 plus the Hugging Face offline switches (TRANSFORMERS_OFFLINE=1, HF_DATASETS_OFFLINE=1, HF_HUB_OFFLINE=1) and every kolm verb refuses network egress. Manage it with kolm airgap status|enable|disable|verify. Enable writes ~/.kolm/airgap.env; verify inspects and signature-checks an artifact entirely offline. /airgap.

auto-pick verb · backend selection

kolm compute pick scores every detected backend and picks the highest. Score is S = 0.35·avail + 0.20·cost_inv + 0.15·latency_inv + 0.15·repro + 0.15·perf. The perf term breaks ties between local-cpu and a present GPU; cost_inv favors free local hardware over paid cloud when capability is close.

BYOC noun · bring your own cloud

Deploy a compiled .kolm artifact onto infrastructure you own. Targets: fly, aws-nitro, gcp-cvm, azure-cvm, docker. Driven by kolm cloud deploy --target <t> --artifact <id>. TEE attestation is captured per deploy mode where the target supports it. /byoc.

canonical JSON noun · deterministic serialization

JSON written with sorted keys and recursive into arrays so two implementations produce byte-identical output. Used as input to CID hashing and to HMAC signing. Without it, a re-serialization in a different language would invalidate the chain.

CID noun · content identifier

A self-describing hash over an artifact's manifest hashes. Form: cidv1:sha256:<64-hex> computed from canonical-JSON of manifest.hashes. Embedded in the manifest, the receipt, and the audit log. Look up an artifact by CID at /v1/cid/:cid.

compute backend noun · where training runs

A named target for training or inference. Fourteen are registered: local accelerators (local-cpu, local-cuda, local-mps, local-mlx, local-rocm, local-directml), serverless cloud (modal, runpod, replicate, fal), managed (together), marketplace (vast, lambda), and self-hosted (remote-ssh). List them with kolm compute list; probe with kolm compute detect.

compute provenance noun · receipt subfield

The block every backend writes into receipt.compute: { backend, device, cost_usd, started_at, finished_at, duration_seconds, provenance{...} }. Third parties can read it and confirm which backend produced an artifact and at what reported cost.

HMAC chain noun · receipt integrity

The integrity scheme on a receipt. Four ordered rings; each ring's HMAC includes the prior ring's signature as input. Tampering with any earlier field breaks every later ring. The Rust verifier re-derives the chain on load.

MLX noun · Apple array framework

Apple's array framework for Silicon. The local-mlx backend uses mlx-lm directly. Lower memory footprint than torch-MPS on many shapes, native to M-series.

MPS noun · Metal Performance Shaders

Apple Silicon GPU compute path used by the local-mps backend. PyTorch backend (torch.backends.mps). Default on every Mac dev box that has it.

Rust verifier noun · runtime crate

The reference verifier at packages/runtime-rs/. Artifact::load plus verify over canonical-JSON, receipt-chain, and body signature. forbid(unsafe). Built as rlib, cdylib, and staticlib so other runtimes can link against it.

TEE noun · trusted execution environment

Hardware-isolated compute. Supported attestation paths: AWS Nitro (COSE_Sign1 + PCR0), AMD SEV-SNP (1184-byte binary), Intel TDX (quote), GCP CVM, Azure CVM. When a .kolm runs in a cloud TEE, kolm captures the attestation quote and writes it into the receipt. See also TOFU.

team workspace noun · multi-user namespace

A shared tenant scope with roles (owner, admin, member) and seat limits. Artifacts written under a team namespace are readable by team members per canRead(c, tenant, tenantId). Managed with kolm team create|list|show|invite|accept|members|role|remove. /teams.

TOFU noun · trust on first use

The attestation trust model. The first quote seen for an artifact is recorded and trusted; later quotes must match it or the verifier refuses. Standard pattern for unattended verification when no central PKI is available.

tunnel noun · remote access broker

An SSE-brokered path from kolm.ai to a local kolm agent serving a .kolm artifact. Seven-day TTL; last-write-wins across multiple agents holding the same token. Lifecycle: kolm tunnel new|list|start|close. /tunnels.
