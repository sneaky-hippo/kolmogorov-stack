Quickstart · v0.7 · 60 seconds

Build your first AI.

Install kolm, describe what you want it to do, train it on your examples, and run it on your laptop. Five minutes, no servers to spin up, your data never leaves the box.

Get API key → full docs

60-second path

  1. npm i -g github:sneaky-hippo/kolmogorov-stack
  2. kolm signup --email you@example.com  (or kolm login --key ks_... if you already have a key)
  3. kolm compile "your task" --examples ./examples.jsonl
  4. kolm run your-artifact.kolm "new input"
01

Install the CLI.

One npm install from the GitHub source. Works everywhere Node 20+ runs.

npm i -g github:sneaky-hippo/kolmogorov-stack
kolm version
02

Configure cloud auth.

The CLI stores your kolm API key in ~/.kolm/config.json. Environment variables still override local config. Two paths in: provision a tenant from the CLI, or paste an existing key with one flag.

kolm config base https://kolm.ai
kolm signup --email you@example.com    # provisions a tenant + API key in one call
# or, if you already have a key from /signup:
kolm login --key ks_...                 # paste-free; saves to ~/.kolm/config.json
03

Create examples.

Use newline-delimited JSON. Keep examples small, explicit, and representative of the task you want to ship.

{"input":"customer cannot log in after update","output":"bug"}
{"input":"please add invoice export","output":"feature"}
{"input":"where do I reset MFA-","output":"support"}
04compile

Turn the task into a .kolm.

The v0 compiler posts the task and examples to the configured cloud, downloads the completed artifact, and stores it in ~/.kolm/artifacts unless you pass --out.

# Minimal example - replace with your task and files
$ kolm compile "classify support tickets" \
  --examples ./support.examples.jsonl \
  --data ./support-policy \
  --base-model qwen2.5-coder-7b-instruct-q4_0 \
  --out ./artifacts

POST /v1/compile
ok wrote ./artifacts/support-tickets.kolm
ok K-score printed by the CLI
05run it

Run it on your machine.

The CLI runs your model locally and returns the answer. A receipt comes with each call if you need to prove what ran.

$ kolm run ./artifacts/support-tickets.kolm '{"input":"checkout fails on Android"}'
{
  "output": "bug",
  "latency_us": 82000,
  "k_score": { "composite": 0.92 },
  "receipt": {
    "spec":            "rs-1",
    "artifact_job_id": "cmp_8f2a...",
    "recipe_id":       "rcp_classify_v1",
    "version_id":      "ver_...",
    "ran_at":          "2026-05-08T17:30:00Z"
  }
}

$ kolm inspect ./artifacts/support-tickets.kolm
$ kolm eval ./artifacts/support-tickets.kolm
$ kolm score ./artifacts/support-tickets.kolm
06wire it into your agent

One command per harness. No JSON hand-editing.

Every compiled .kolm becomes an MCP tool the agent can call. kolm install writes the harness config in one shot; kolm doctor verifies the wiring.

$ kolm install claude-code --apply
wrote ~/.claude/settings.json

$ kolm install cursor --apply
wrote .cursor/mcp.json

$ kolm doctor
ok node ≥18 · kolm.yaml · 3 artifacts · ~/.kolm/config.json · git

# prefer the long-running daemon? same artifacts, stdio or HTTP transport.
$ kolm serve --mcp --http --port 11455
ok serving ~/.kolm/artifacts as local tools
