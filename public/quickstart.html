quickstart

Compile your first artifact.

Install kolm from GitHub, point it at https://kolm.ai, log in with your key, compile a task, run it on your laptop, and inspect the receipt. Five minutes, no servers to spin up, your data stays put.

Get API key → full docs

60-second path

  1. npm i -g @kolmogorov/kolm
  2. kolm login
  3. kolm compile "your task" --examples ./examples.jsonl
  4. kolm run your-artifact.kolm "new input"
01

Install the CLI.

One npm install from the GitHub source. Works everywhere Node 20+ runs.

npm i -g github:sneaky-hippo/kolmogorov-stack
kolm version
02

Configure cloud auth.

The CLI stores your kolm API key in ~/.kolm/config.json. Environment variables still override local config.

kolm config base https://kolm.ai
kolm login
# paste your ks_... API key
03

Create examples.

Use newline-delimited JSON. Keep examples small, explicit, and representative of the task you want to ship.

{"input":"customer cannot log in after update","output":"bug"}
{"input":"please add invoice export","output":"feature"}
{"input":"where do I reset MFA-","output":"support"}
04compile

Turn the task into a .kolm.

The v0 compiler posts the task and examples to the configured cloud, downloads the completed artifact, and stores it in ~/.kolm/artifacts unless you pass --out.

# Minimal example - replace with your task and files
$ kolm compile "classify support tickets" \
  --examples ./support.examples.jsonl \
  --data ./support-policy \
  --base-model qwen2.5-coder-7b-instruct-q4_0 \
  --out ./artifacts

POST /v1/compile
ok wrote ./artifacts/support-tickets.kolm
ok K-score printed by the CLI
05run and verify

Run locally. Keep the receipt.

A signed artifact can be run by the CLI and inspected later. The receipt proves which artifact, input, output, and verifier result were used.

$ kolm run ./artifacts/support-tickets.kolm '{"input":"checkout fails on Android"}'
{
  "output": "bug",
  "latency_us": 82000,
  "k_score": { "composite": 0.92 },
  "receipt": {
    "spec":            "rs-1",
    "artifact_job_id": "cmp_8f2a...",
    "recipe_id":       "rcp_classify_v1",
    "version_id":      "ver_...",
    "ran_at":          "2026-05-08T17:30:00Z"
  }
}

$ kolm inspect ./artifacts/support-tickets.kolm
$ kolm eval ./artifacts/support-tickets.kolm
$ kolm score ./artifacts/support-tickets.kolm
06serve

Expose artifacts to agent tools.

Use kolm serve --mcp when your local agent stack should discover the artifact as a callable tool.

$ kolm serve --mcp --http --port 11455
ok serving ~/.kolm/artifacts as local tools

Next: build your own from scratch.

The cloud compile path takes a task plus examples. The local spec path takes a JSON program any human or AI can write, compiles it offline, signs it under your local secret. Pick whichever fits the work.
