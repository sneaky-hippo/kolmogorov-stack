now

Shipped, in production

today · kolm.ai

Everything below is live on kolm.ai, runs in the public CLI, or sits inside any .kolm you compile this minute. Inspect the runtime source on GitHub.

shipped

MIT-licensed runtime

The kolm CLI and the runtime that executes a .kolm artifact ship under MIT. Source on github.com/sneaky-hippo/kolmogorov-stack. Run it on your laptop without us.

shipped

.kolm spec, RS-1

RS-1 is the wire format for a compiled artifact: manifest, spec, policy, optional sidecar, verifier, receipt. Open and versioned. Read RS-1.

shipped

Stripe billing, live

Self-serve checkout on Free, Starter, Pro, Teams, Business, Enterprise. POST /v1/account/change-plan returns a Stripe payment link bound to your tenant.

shipped

HMAC-SHA-256 signing

Every .kolm carries a single HMAC over its manifest + layer hashes. Every /v1/run emits a receipt that the issuer or any holder of the shared tenant secret can re-verify offline.

shipped

Registry catalog, public

A public read-only catalog of recipes, base-model fingerprints, and RS-1 entries at /v1/registry/public. Bulk export at /v1/registry/export.

shipped

MCP harness wire-up

kolm install claude-code, cursor, continue, cline. Writes mcpServers.kolm into the right config; stdio default, optional HTTP JSON-RPC on 127.0.0.1:8765.

next

In flight, v7.X target

v7.X · target 2026-Q2/Q3

Work that is actively underway. Code in the repo, work tracked in the changelog. Targets slip publicly.

v7.X target

SLSA Level 3 supply chain

Hermetic builds with provenance attestations published per release. CycloneDX SBOM emitted alongside the artifact. Closes the verifiable-build-of-the-runtime gap.

v7.X target

SOC 2 Type I

Auditor engaged for a point-in-time SOC 2 Type I. Scope: production environment, change management, access control, monitoring. Report shareable under NDA.

v7.X target

Sigstore artifact anchoring

Cosign-signed release artifacts and Rekor transparency-log entries for every .kolm binary we publish. Independent public-key verification, no shared secret required.

v7.X target

Ed25519 receipts

Per-call receipts signed with a tenant Ed25519 key. Anyone holding the public key can verify a run happened, without our service. Roadmap-tier today.

later

RS-2, mobile runtime

target 2026-Q3

A second runtime profile for iOS and Android. RS-2 carries a quantized adapter slot for 3B-class base models alongside the existing recipe slot, with the same signed receipt envelope.

2026-Q3 target

iOS runtime, RS-2

Swift Package Manager distribution. Runs a .kolm with adapter-tier payload against an on-device 3B-class base. Receipt format matches RS-1.

2026-Q3 target

Android runtime, RS-2

AAR distribution. Same RS-2 profile, same receipt envelope, NNAPI-backed inference. Designed for sovereign-data apps where the phone is the boundary.

later

2026-Q4 target

Type II audit, complete
