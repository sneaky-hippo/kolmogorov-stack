kolm.ai for VS Code

One command wires kolm into VS Code as an MCP server. Your compiled .kolm artifacts appear as tools the agent can call. The installer merges into your existing ~/.vscode/mcp.json without overwriting other servers.

Install command

copy · paste · run
node scripts/install-mcp.cjs vscode

Already have kolm installed globally? Run kolm install vscode instead. Pass --dry-run to preview the JSON without touching disk.

target config: ~/.vscode/mcp.json

What this does

The installer reads ~/.vscode/mcp.json (if present), parses the JSON, merges in a single mcpServers.kolm entry that points to kolm serve --mcp, and writes atomically using a tmp-file + rename. Other servers in the same file are preserved byte-for-byte. Re-running is a no-op when the kolm block is already correct.

The exact block injected:

{
  "mcpServers": {
    "kolm": {
      "command": "kolm",
      "args": ["serve", "--mcp"],
      "env": {}
    }
  }
}

If an existing mcpServers.kolm block differs from the canonical one, the installer prints a diff and refuses to overwrite. Pass --force to replace it.

After install

step 1

Reload the window

Cmd+Shift+P, then "Developer: Reload Window". VS Code picks up the new MCP server.

step 2

See kolm tools appear

Open the chat tool picker. Each .kolm artifact under your kolm.yaml is listed and callable.

step 3

Call from chat

Ask the agent to run a compiled task. It routes to the local kolm server, signs the call, writes a receipt.

Cross-platform paths

  • macOS & Linux: ~/.vscode/mcp.json
  • Windows: %USERPROFILE%\.vscode\mcp.json

Troubleshooting

If the installer prints "refusing to overwrite", you already have a different mcpServers.kolm entry. Inspect the diff, then either edit by hand or rerun with --force. The installer never crashes on an unknown platform: it prints the JSON block and the best-guess path so you can paste manually.
