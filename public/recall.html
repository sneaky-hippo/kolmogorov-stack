04 - The data moat

Approved context. Compiled into the artifact.

Recall is the grounding layer: approved context travels with the artifact so runtime behavior can be reviewed where the data already lives. kolm compile --data ingests a corpus into the artifact; kolm inspect shows exactly what was bound; kolm recall queries the embedded index without leaving the boundary.

# compile a task with approved local context
$ kolm compile "answer questions about our infra" \
  --data ./my-data \
  --examples ./infra.examples.jsonl \
  --out ./artifacts

# inspect what the artifact declares
$ kolm inspect ./artifacts/infra.kolm

# standalone recall query against the embedded index
$ kolm recall "how do we rotate the prod KMS key?" \
  --artifact ./artifacts/infra.kolm

01 - Ingest anything

Multimodal sidecars.

Every non-text file gets a Markdown sidecar, a structured tokenization that turns pixels, frames, and waveforms into something both humans and models can read. The sidecar is what gets embedded; the original is what gets returned.

.txt .md .ts .py

Text and code

Chunked by structure, embedded with bge-m3. Symbols, comments, headings preserved.

.png .jpg .heic

Images

CLIP embed plus caption sidecar via vision model. Both vector and lexical recall.

.mp3 .wav .m4a

Audio

Whisper transcript with CLAP timbre embed. Speakers and music both addressable.

.mp4 .mov

Video

Scene-detected, then per-shot caption + ASR + CLIP. Jump straight to the right second.

.pdf

PDF

Layout parse via unstructured. Per-page sidecar; tables, figures, footnotes addressable.

auto

Everything else

magika sniffs the bytes. Falls back to bytes-as-text for any file the kernel can read.

02 - Hybrid retrieval

BM25 plus vector. RRF, then rerank.

Lexical recall catches names and IDs. Vector recall catches paraphrase. RRF fuses them without tuning a weight. A small cross-encoder reorders the top of the list. The whole pipeline is what grounds every Distill k-sample and every kolm.wrap call when a corpus namespace is set.

query → BM25 top-50 + vector top-50 → RRF fusion → cross-encoder rerank → top-k

Every retrieved chunk carries its source path, byte range, modality, and score. The compiler quotes the source verbatim and the receipt anchors each citation to a content hash. Hallucinated citations are detectable end-to-end.

03 - Three back-ends, one interface

Same SDK. Same wire format.

Pick the back-end at the boundary; the rest of the stack does not notice. On-device is the default inside artifacts: every .kolm ships index.sqlite-vec so recall keeps working when the network is gone.

Managed

Pinecone

Default for Cloud-compiled artifacts. Per-tenant namespaces, regional residency, no infra.

KOLMOGOROV_INDEX_BACKEND=pinecone
Self-host

Trieve

Single Docker image. BYO embedding service or use the bundled bge-m3. VPC-friendly.

KOLMOGOROV_INDEX_BACKEND=trieve KOLM_TRIEVE_URL=https://trieve.your-vpc.local
On-device

sqlite-vec

What ships inside every .kolm at runtime. Single file. No network.

KOLMOGOROV_INDEX_BACKEND=sqlite-vec
