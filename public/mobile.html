mobile runtime

The same signed artifact, on every phone.

A .kolm compiled on a workstation runs unchanged on iOS and Android. Native adapters verify the manifest, the recipe pack, and the receipt locally before a single token is generated. No upload. No hosted call. No round trip.

01desktop

What ships today.

The CLI and MCP runtime cover macOS, Linux, and Windows. They inspect, verify, evaluate, run, and serve signed artifacts: the exact bytes that mobile adapters consume.

Node CLI runtime

kolm run, kolm eval, and kolm inspect operate on local .kolm artifacts.

MCP serving

kolm serve --mcp exposes local artifacts as tools for agents and desktop workflows.

Receipts

Every artifact and every run is HMAC-bound; readiness checks fail closed when production secrets are missing.
