tmux recipe

A serious compile takes hours. SSH sessions die. Laptops close. Wi-Fi roams. tmux solves all of this by keeping a shell alive on the server, independent of who is connected. Pair it with kolm and a four-hour compile is fire-and-forget; you reattach from anywhere and pick up exactly where you left off.

Why pair them

kolm compiles and distills are CPU + GPU intensive jobs that block a terminal. Without tmux, every time you close the laptop or the SSH socket times out, the job dies (or worse, half-dies and leaves a corrupted artifact). With tmux, the job runs in a named session on the remote box; the SSH connection is just a window into it.

The W229 kolm jobs registry and kolm watch verb make this even cleaner: the compile records itself to ~/.kolm/jobs.jsonl with its log path; if you reattach to the tmux pane, the log is still scrolling; if you connect from a totally new box, kolm watch <job-id> tails the same log file from outside the tmux session.

One-time setup

  1. Install tmux on the compile box: sudo apt install tmux or brew install tmux. Confirm with tmux -V.
  2. (Optional) Drop a minimal ~/.tmux.conf: set -g mouse on and set -g history-limit 50000. Mouse on makes scrollback usable; the larger history captures a full long log.
  3. Pick a session name convention that matches your team. We use kolm-<tenant>-<date> so multiple compiles never collide.

The daily flow

# 1. start the session and the compile
tmux new -s kolm-prod-20260518
kolm compile spec.json --out artifact.kolm

# 2. detach: ctrl-b, then d
# the compile keeps running. close your laptop. take a meeting.

# 3. on any device, find what is running
ssh box "kolm jobs list"
# id                    kind    status    updated_at           pid
# job-9f3a1c4b8e72      compile running   2026-05-18 14:22:01  18432

# 4. reattach to the tmux pane to see live output
ssh box -t "tmux attach -t kolm-prod-20260518"

# OR follow the log without entering tmux at all
ssh box "kolm watch job-9f3a1c4b8e72"

What this gets you

  • Compiles survive every disconnect. Laptop sleep, SSH timeout, Wi-Fi roam, server reboot, network partition all leave the job running.
  • Multi-window: one pane runs the compile, another tails captures, a third pulls the artifact when done. ctrl-b " splits horizontal, ctrl-b % splits vertical.
  • Multi-device handoff: start the compile from your laptop, check on it from your phone via Termius, finish triaging from your home desktop. Same session, three keyboards.
  • Disaster recovery: if the kolm process crashes, the tmux pane shows the stack trace until you explicitly close it. No log loss.
tmux is not a kolm dependency. Anything that gives you a detachable session works: screen, dtach, byobu, zellij. We pick tmux because it is the one in every distro, on every Mac via Homebrew, and the keybindings are muscle memory for most fleet operators.

Failure modes worth knowing

Server reboot. tmux sessions do not survive a host reboot. If you want the job to restart, write a systemd unit or a cron @reboot hook that calls tmux new -d -s kolm "<command>". The W229 jobs registry remembers the previous run's record either way.

Lost session name. tmux ls shows every active session. If you forgot what you called it, tmux ls from the host gives the answer in one line.

tmux pane size mismatch. When two clients with different window sizes attach to the same session, tmux shrinks to the smaller. ctrl-b D (capital D) lets you detach all other clients before you attach.
