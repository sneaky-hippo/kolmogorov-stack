Enterprise · self-hosted

Run kolm compile inside your VPC.

Same compile pipeline as kolm.ai, packaged as a standalone Node server. Runs behind your firewall, on your hardware, or fully air-gapped. The .kolm artifact this server produces is byte-identical to the cloud version because both call the same orchestrator with the same signing chain.

Three install paths.

Pick the one that matches your deployment posture. All three run the same workers/compile-server/server.mjs entrypoint and produce identical artifacts.

Container

Docker

Single image, single port. Good for evaluation and isolated single-server tenants.

docker build -f workers/compile-server/Dockerfile \ -t kolm-compile:w264 . docker run --rm -p 8080:8080 \ -e KOLM_SHARED_SECRET=$(openssl rand -hex 32) \ -e KOLM_OFFLINE=1 \ -v $(pwd)/data/artifacts:/data/artifacts \ kolm-compile:w264
Compose

docker-compose

Adds a named volume so artifacts survive restarts. One command up, one command down.

export KOLM_SHARED_SECRET=$(openssl rand -hex 32) docker compose -f workers/compile-server/docker-compose.yml up -d curl http://localhost:8080/v1/health
Kubernetes

Helm

Minimal chart with a Deployment, Service, and PVC. Pin image.repository at your internal registry for air-gapped clusters.

kubectl create namespace kolm kubectl -n kolm create secret generic kolm-compile-secret \ --from-literal=shared-secret=$(openssl rand -hex 32) helm -n kolm install kolm-compile \ workers/compile-server/helm

Shared-secret auth.

No API keys, no OAuth, no tenant database. The server boots refusing every request until KOLM_SHARED_SECRET is set so you cannot accidentally leave it open. Every endpoint except /v1/health requires the header below. Comparison is constant time.

Header

x-kolm-shared-secret: $KOLM_SHARED_SECRET

Rotate by changing the env var and rolling the pod. The server reads it once at boot.

Smoke test

curl -X POST http://localhost:8080/v1/compile?sync=1 \ -H "x-kolm-shared-secret: $KOLM_SHARED_SECRET" \ -H "content-type: application/json" \ -d '{"task":"redact PII from text"}'

Returns {job_id, status:"completed", poll} on success. Drop ?sync=1 for fire and forget.

No-egress verification.

With KOLM_OFFLINE=1 the server refuses any request whose handler path would touch the network (e.g. the optional deploy_hook fan-out). To prove this end to end during an audit, run a packet capture across one compile job and grep for any non-loopback traffic.

Linux

tcpdump

sudo tcpdump -i any -nn -w /tmp/kolm.pcap \ not host 127.0.0.1 and not host ::1 & TCPDUMP_PID=$! curl -X POST http://localhost:8080/v1/compile?sync=1 \ -H "x-kolm-shared-secret: $KOLM_SHARED_SECRET" \ -H "content-type: application/json" \ -d '{"task":"redact PII from text"}' sudo kill $TCPDUMP_PID sudo tcpdump -nn -r /tmp/kolm.pcap | wc -l # Expected: 0
macOS

pfctl + tcpdump

sudo tcpdump -i any0 -nn -w /tmp/kolm.pcap \ not host 127.0.0.1 and not host ::1 & TCPDUMP_PID=$! curl -X POST http://localhost:8080/v1/compile?sync=1 \ -H "x-kolm-shared-secret: $KOLM_SHARED_SECRET" \ -H "content-type: application/json" \ -d '{"task":"summarize email"}' sudo kill $TCPDUMP_PID sudo tcpdump -nn -r /tmp/kolm.pcap | wc -l
Belt and suspenders. For a regulated deployment, pair KOLM_OFFLINE=1 with a Kubernetes NetworkPolicy that denies egress, or run the container with --network none after the first compile.

Byte-identical to cloud.

This server calls the same createJob and runJob exports from src/compile.js that kolm.ai calls. Same signing chain, same manifest fields, same K-score formula, same .kolm zip layout. To verify, compile the same input pair against both endpoints and compare the hash.

sha256sum cloud.kolm self-hosted.kolm # both hashes match when KOLM_RECEIPT_SECRET and base_model are the same.

HTTP surface.

Mirrors the cloud /v1/compile endpoints. Anything that integrates with the cloud also integrates with the self-hosted server.

VerbPathDescription
GET/v1/healthLiveness probe. No auth. Returns mode, version, offline flag.
POST/v1/compileStart a compile job. Returns 202 with job_id + poll URL.
GET/v1/compile/:idJob status. Includes manifest, k_score, artifact_url.
GET/v1/compile/:id/.kolmStream the signed artifact bytes.
