01 · kolm-bench v1

The K-score benchmark you can run yourself.

30 frozen cases. 10 axes. One number per artifact. Every .kolm file gets graded against the exact same inputs anyone else's does. The spec is signed, the cases never change inside a version, and the leaderboard is read-only after submission. spec frozen 2026-05-18

See the leaderboard → Download bench spec (JSON) → What is K-score? →

Why a public bench? Because every model card today cherry-picks the metric. kolm-bench gives the buyer one signed receipt per artifact across the same 30 cases. Compare apples to apples. No marketing knobs.

02 · The 30 cases

Four task classes. Eight, seven, eight, seven cases each.

Frozen forever inside kolm-bench-v1. Replaced wholesale if and when the spec rolls to v2. Old artifacts keep their v1 receipts.

8 cases
Classification
Sentiment, intent, support routing. Single-label, deterministic. Tests scorer honesty on the easiest axis.
7 cases
Extraction
PII, healthcare fields, invoices, addresses. Structured output, exact-match scoring with field-level coverage.
8 cases
Generation
Summarization, translation, rewriting, PHI redaction. Contains-set scoring so the artifact can phrase freely.
7 cases
Code
Regex, function bodies, SQL, fix-the-bug. Multi-language. Contains-set scoring on key tokens.
03 · The 10 axes

Each case scores all axes K-score knows about.

K-score-v1 is five axes (A/S/L/C/V). K-score-v2 adds five more (R/F/T/E/Z) that surface when the artifact supplies the inputs. Missing axes redistribute weight to supplied ones — the composite stays comparable.

AAccuracyPass-rate on the case's reference output. Dominant term.w 0.30 / 0.40
SSizeInverse log of artifact bytes. Smaller is closer to 1.w 0.10 / 0.15
LLatencyp50 microseconds per call. 80 ms hits ~0.94.w 0.10 / 0.15
CCostUSD per call. Local-only artifacts hit 1.00.w 0.10 / 0.15
VCoverageCases covered / cases declared.w 0.10 / 0.15
RRobustnessHeld-out accuracy clamped to [0,1]. v2 only.w 0.05
FFairnessLowest subgroup accuracy. v2 only.w 0.10
TTeacher fidelitystudent_holdout / teacher_holdout. Distillation honesty. v2 only.w 0.05
EEnergy1 / (1 + joules_per_call / 100). v2 only.w 0.05
ZDrift1 - eval-set drift vs baseline. v2 only.w 0.05
04 · Run it

One command. Local. Reproducible.

# run the public bench against your .kolm artifact
$ kolm bench phi-redactor.kolm
spec:        kolm-bench-v1
frozen_at:   2026-05-18T00:00:00Z
cases:       30  (8 cls + 7 ext + 8 gen + 7 code)

cls-01  pass  K=0.95  (A=0.98 S=0.95 L=0.94 C=1.00 V=0.92)
cls-02  pass  K=0.95  (A=0.98 ...)
...
code-07 pass  K=0.91

bench composite: 0.94  pass_rate: 28/30  ships: yes
# axis breakdown for one artifact
$ kolm score phi-redactor.kolm --axis A,T,F,R
A (accuracy):           0.962
T (teacher fidelity):   0.970
F (fairness):           0.920
R (robustness):         0.930
# submit your signed receipt to the public leaderboard
$ kolm bench phi-redactor.kolm --submit
posted receipt to https://kolm.ai/v1/bench/submit
rank: pending review (signed-only submissions accepted)

The bench runs entirely on your machine. The artifact never leaves your box; only the signed score receipt does, and only when you pass --submit.

05 · The contract

What the bench promises · and what it does not.

Promises

Same cases on every machine in every shell. Spec frozen at kolm-bench-v1. Per-case axis breakdown reproducible from the score receipt. Score receipts can be verified at /verify-prod.

Does not

Rank base models. Decide if your artifact is good for your task. Replace your own eval set — bench is the floor, not the ceiling. Hand-tuned cases (your tests.jsonl) always beat the public set for your specific use.
