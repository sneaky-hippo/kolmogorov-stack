Why these five

The agentic stack that actually ships has been around for a decade. Tailscale (2019), Termius (2013), tmux (2007), git (2005), shell scripts (1971). They are unglamorous and load-bearing. What changed in 2026 is that agents now do real work on the same infrastructure, and the seam between "agent" and "tools" is where most of the friction lives.

kolm fits in the same row. A compiled AI is a file. The file moves over Tailscale. It runs in a tmux session. You version it in git. You invoke it from a script. You tail its captures from Termius. None of that needs a new abstraction layer.

What kolm does not do

kolm is not a replacement for any of these tools. We do not bundle a VPN, we do not fork a terminal multiplexer, we do not embed git, we do not ship a phone client, we do not abstract the shell. If you already use these tools, kolm is one more file on disk and four more CLI verbs.

The 5 verbs that close this seam are kolm jobs, kolm watch, kolm tail captures, kolm sync, and kolm profile. Each one is a thin wrapper over the corresponding foundation tool's native interface, not a re-implementation. Read the CLI reference for the full surface.

If you do not use any of these tools yet, kolm still works. Defaults run locally with no remote, sessions stay in your shell, profiles are optional, captures stream to stdout. The 5-foundations setup is for teams running multi-box AI infrastructure, not a prerequisite.

Five recipes

One page per tool, each one with the install snippet for the tool, the exact kolm verbs that pair with it, and a copy-paste boilerplate. All five recipes share the same shape so you can mix and match.

  • Tailscale recipe: tailnet-scoped git mirror, one-command artifact push to every device.
  • Termius recipe: mobile-runnable kolm tail captures + push-button distill from your phone.
  • tmux recipe: four-hour detached compile, reattach across reboots, live log follow.
  • Private git recipe: per-tenant artifact mirror, branch-per-tenant memory layer, receipt-chain in version control.
  • Script-everything recipe: kolmrc + per-machine profiles + a doctor that catches drift before it matters.
