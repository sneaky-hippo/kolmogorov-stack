EDI 834 enrollment transformer

A kolm template that ingests EDI 834 enrollment/maintenance transactions (5010A1) from sponsors and emits normalized member-roster JSON. Distinguishes add/change/term per the BGN02 purpose code and INS01 member-indicator. Pure rule-class - runs inside your BAA bridge with full receipt coverage.

Input & output spec

Input is raw EDI 834 X12 text from a plan sponsor (employer, exchange, government program). Output is a normalized member roster with one entry per INS loop, classified as one of five member-action values - add, change, term, cancel, reinstate, plus audit for INS03 030 audit-or-compare payloads - so downstream enrollment systems can apply the delta without re-parsing X12.

4-ring receipt schema

Every transform ships a receipt your auditor can re-verify offline. Because this is a rule-class recipe the execution ring carries a deterministic hash of the parsed INS loops plus the per-member action vector rather than a model judge id; the seal matches the model-class templates so downstream verifier code does not branch on recipe class.

ring 01 · pre-image
isa_control_num + bgn_reference + sponsor_tax_id_hash
tenant-rotated HMAC
ring 02 · derived
purpose_code + plan_dict_hash + loop_topology_hash
bound to recipe_version
ring 03 · execution
member_count + action_vector_hash + summary_totals
deterministic, no judge
ring 04 · seal
SHA-256(ring1..3) · verify endpoint
offline-verifiable

Verify command: kolm verify --hash <receipt_hash>. Runs locally against the seal, no network required.

BAA fast-track

Health plans on a BAA see kolm production-ready in two business days. The customer-hosted bridge runs in the tenant network boundary; PHI in the 834 envelope (subscriber name, SSN qualifier 34, member id, address) is hashed inside the bridge and never crosses our infrastructure; the audit log is yours to keep.

how it works

# 1. Sign the BAA (legal-team turnaround)
[ DocuSign · 48 hours typical · mutual NDA in same envelope ]

# 2. Provision the bridge in your VPC
$ kolm bridge install \
    --provider aws \
    --region us-east-1 \
    --baa <baa-id>

# 3. Compile this template (rule-class, no seed examples required)
$ kolm compile --spec edi-834.spec.json \
    --gate 0.93 \
    --out edi-834.kolm

# 4. Stream sponsor 834 files through it
$ kolm run edi-834.kolm \
    --input sponsor.834 \
    --emit roster.jsonl
