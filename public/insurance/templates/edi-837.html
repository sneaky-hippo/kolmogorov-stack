EDI 837 claims transformer

A kolm template that ingests raw EDI 837P/I claims (5010A1/A2), validates the loops + segments + qualifiers, and emits normalized claim JSON ready for adjudication, fraud-detection, or analytics. The transform is pure rule-class - no model inference required - so it runs deterministically inside the BAA bridge with full receipt coverage.

Input & output spec

Input is raw EDI 837P (professional) or 837I (institutional) X12 text. Output is a normalized JSON object with fixed field names so downstream adjudication, fraud, and analytics systems can rely on the shape without re-parsing X12.

4-ring receipt schema

Every transform ships a receipt your auditor can re-verify offline. Because this is a rule-class recipe the execution ring carries a deterministic hash of the parsed segments rather than a model judge id; the seal is identical to the model-class templates so downstream verifier code does not branch on recipe class.

ring 01 · pre-image
isa_control_num + claim_id + member_hash
tenant-rotated HMAC
ring 02 · derived
qualifier_dict_hash + loop_topology_hash
bound to recipe_version
ring 03 · execution
parsed_segments_hash + validation_flags
deterministic, no judge
ring 04 · seal
SHA-256(ring1..3) · verify endpoint
offline-verifiable

Verify command: kolm verify --hash <receipt_hash>. Runs locally against the seal, no network required.

BAA fast-track

Health plans on a BAA see kolm production-ready in two business days. The customer-hosted bridge runs in the tenant network boundary; PHI in the 837 envelope (subscriber name, member id, patient identifiers) is hashed inside the bridge and never crosses our infrastructure; the audit log is yours to keep.

how it works

# 1. Sign the BAA (legal-team turnaround)
[ DocuSign · 48 hours typical · mutual NDA in same envelope ]

# 2. Provision the bridge in your VPC
$ kolm bridge install \
    --provider aws \
    --region us-east-1 \
    --baa <baa-id>

# 3. Compile this template (rule-class, no seed examples required)
$ kolm compile --spec edi-837.spec.json \
    --gate 0.92 \
    --out edi-837.kolm

# 4. Stream production 837 traffic through it
$ kolm run edi-837.kolm \
    --input claims.837 \
    --emit normalized.jsonl
