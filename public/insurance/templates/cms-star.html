CMS Star contributor extractor (Medicare Advantage / Part D)

A kolm template that extracts CMS Star Ratings measure contributors per member or per contract. Identifies which Part C and Part D measures a member's claims + utilization + survey responses contributed to, the direction (positive / negative), and the per-measure weight (CMS 2025+ weighting: outcome measures 3x, intermediate-outcome 3x, patient-experience 4x, process 1x, improvement 5x). Class: synthesized-rule (class 1).

recipe-class: synthesized-rule (class 1) spec: CMS Star Ratings 2025+ Technical Notes programs: Part C + Part D

cms-star

Input & output spec

One member's claims + utilization + survey responses go in; the output is a list of measures the member's data contributed to, the direction of the contribution, and the CMS weight in effect for that measure under the 2025+ Technical Notes.

input

{
  "member": {
    "member_id":    "<redacted>",
    "contract_id":  "H1234",
    "part":         ["C", "D"],
    "age":          68
  },
  "claims": [
    { "date": "2024-08-12", "cpt": "77067" }
  ],
  "utilization": [
    { "measure_hint": "bp_controlled", "systolic": 132, "diastolic": 78 }
  ],
  "survey_responses": []
}

output

{
  "member_id_redacted": "<hash>",
  "contract_id":        "H1234",
  "measures": [
    {
      "measure_id":  "C09",
      "name":        "Controlling Blood Pressure",
      "part":        "C",
      "category":    "intermediate_outcome",
      "weight":      3,
      "direction":   "positive",
      "contribution": "bp_controlled"
    }
  ]
}

Measure category weighting

CMS pins per-category weights in the annual Technical Notes. The rule library encodes the 2025+ values so QA can re-derive any aggregate star calculation without re-reading the notes PDF.

outcome measures
weight 3
intermediate outcome
weight 3
patient experience
weight 4
access measures
weight 4
process measures
weight 1
improvement measures
weight 5

Measure identifiers (C01, C09, C13, D08, D09, D10, D12) are stable across years for the listed measures but the exact set on the Star scorecard changes annually; re-pin part_c_top_measures and part_d_top_measures against the current Technical Notes at start-of-year and bump the rule library version.

4-ring receipt schema

Every contributor extraction ships a receipt your auditor can re-verify offline. The chain proves which member event-stream was ingested, which rule library version was applied, the measures attributed, and the seal.

ring 01 · pre-image
member_hash + contract_id + claim_count
tenant-rotated HMAC
ring 02 · derived
rule_library_version + measure_table_hash
bound to recipe_version
ring 03 · execution
measures[] + direction + weight_sum
judge-attested for synthesized fields
ring 04 · seal
SHA-256(ring1..3) · verify endpoint
offline-verifiable

Verify command: kolm verify --hash <receipt_hash>. Runs locally against the seal, no network required.

BAA fast-track

Health plans on a BAA see kolm production-ready in two business days. The customer-hosted bridge runs in the tenant network boundary; PHI in the claims and survey streams is processed there and never crosses our infrastructure; the audit log is yours to keep.

how it works

# 1. Sign the BAA (legal-team turnaround)
[ DocuSign · 48 hours typical · mutual NDA in same envelope ]

# 2. Provision the bridge in your VPC
$ kolm bridge install \
    --provider aws \
    --region us-east-1 \
    --baa <baa-id>

# 3. Compile this template against the rule library
$ kolm compile --spec cms-star.spec.json \
    --gate 0.93 \
    --out cms-star.kolm

# 4. Run against your contract roster
$ kolm run cms-star.kolm \
    --input H1234-members.json \
    --emit star-contributors.json
