EDI 270/271 eligibility pair

A kolm template that handles both sides of the eligibility loop: the 270 inquiry from provider to payer asking "is this member covered for these services?" and the 271 response back with active/inactive benefits, service-type-codes, and copay/deductible. One symmetric transformer with --direction inquiry|response. Pure rule-class; deterministic; no model inference.

Input & output spec

Input is raw X12 5010A1 text for either a 270 inquiry or a 271 response. Direction is auto-detected from the ST01 functional identifier (270 vs 271). The output schema is symmetric: the same envelope wraps both sides of the loop so downstream code does not branch on direction.

4-ring receipt schema

Every transform ships a receipt your auditor can re-verify offline. Because this is a rule-class recipe the execution ring carries a deterministic hash of the parsed BHT, HL, NM1, EQ, EB, and AAA segments rather than a model judge id; the seal is identical to the model-class templates so downstream verifier code does not branch on recipe class.

ring 01 · pre-image
isa_control_num + trn_trace_id + member_hash
tenant-rotated HMAC
ring 02 · derived
eq_dict_hash + eb_dict_hash + aaa_dict_hash
bound to recipe_version
ring 03 · execution
direction + parsed_segments_hash + benefit_count
deterministic, no judge
ring 04 · seal
SHA-256(ring1..3) · verify endpoint
offline-verifiable

Verify command: kolm verify --hash <receipt_hash>. Runs locally against the seal, no network required.

BAA fast-track

Health plans on a BAA see kolm production-ready in two business days. The customer-hosted bridge runs in the tenant network boundary; PHI in the 270/271 envelope (subscriber name, member id, DOB, group number) is hashed inside the bridge and never crosses our infrastructure; the audit log is yours to keep.

how it works

# 1. Sign the BAA (legal-team turnaround)
[ DocuSign · 48 hours typical · mutual NDA in same envelope ]

# 2. Provision the bridge in your VPC
$ kolm bridge install \
    --provider aws \
    --region us-east-1 \
    --baa <baa-id>

# 3. Compile this template (rule-class, no seed examples required)
$ kolm compile --spec edi-270-271.spec.json \
    --gate 0.94 \
    --out edi-270-271.kolm

# 4. Stream production 270/271 traffic through it
$ kolm run edi-270-271.kolm \
    --input eligibility.x12 \
    --direction inquiry \
    --emit normalized.jsonl

# 5. On the payer side, run the same artifact in response mode
$ kolm run edi-270-271.kolm \
    --input responses.x12 \
    --direction response \
    --emit normalized.jsonl
