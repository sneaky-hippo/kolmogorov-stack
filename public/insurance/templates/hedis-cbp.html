HEDIS CBP — Controlling High Blood Pressure

A kolm template that computes the HEDIS Controlling High Blood Pressure (CBP) measure: the percentage of plan members 18-85 with hypertension whose most recent BP reading in the measurement year was adequately controlled (< 140/90 mmHg if age 18-59 or with diabetes; < 150/90 mmHg if age 60-85 without diabetes). Class: synthesized-rule (class 1). Input: FHIR R4 bundle of qualifying members + their Observation BP readings; output: { measure: "CBP", numerator, denominator, rate, members: [{ member_id_redacted, age, age_band, has_diabetes, qualifying_bp_reading, controlled }] }.

Input & output spec

Input is a FHIR R4 Bundle containing the qualifying member roster and their BP Observation resources for the measurement year. Output is a single measure record with numerator, denominator, rate, and a per-member array so QA can trace any rate calc back to the member that drove it.

Age-band thresholds

The CBP measure uses three thresholds depending on age and diabetes status. The rule library pins the exact mmHg cut points so QA can re-derive any per-member controlled flag without re-reading the technical specification.

age 18-59 (any diabetes status)
< 140 / 90 mmHg
age 60-85 with diabetes
< 140 / 90 mmHg
age 60-85 without diabetes
< 150 / 90 mmHg

Hypertension is determined by ICD-10-CM code I10 (essential primary hypertension) and the I11/I12/I13/I15 family. Continuous enrollment is required with no gap exceeding 45 days during the measurement year.

4-ring receipt schema

Every measure computation ships a receipt your auditor can re-verify offline. The chain proves which bundle was ingested, which version of the rule library was applied, the numerator and denominator that resulted, and the seal.

ring 01 · pre-image
bundle_hash + member_count + measurement_period
tenant-rotated HMAC
ring 02 · derived
rule_library_version + age_band_table_hash
bound to recipe_version
ring 03 · execution
numerator + denominator + per_member_digest
deterministic, no judge
ring 04 · seal
SHA-256(ring1..3) · verify endpoint
offline-verifiable

Verify command: kolm verify --hash <receipt_hash>. Runs locally against the seal, no network required.
