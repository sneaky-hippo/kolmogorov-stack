Open research questions
  • K-score component weights across verticals.
    Are 0.40 / 0.15 / 0.15 / 0.15 / 0.15 still optimal for healthcare vs finance vs legal vs edge? Cross-vertical hold-out tests on 5 verticals in progress; preliminary evidence suggests latency weight should rise for edge.
  • Receipt chain compression.
    The current HMAC chain is O(N) to verify N receipts. Investigating a Merkle tree variant for O(log N) inclusion proofs without losing per-run signatures.
  • PHI redactor false-negative rate.
    The 18-identifier regex redactor passes 99.6% on our test corpus. The remaining 0.4% is clinical-note edge cases (handwritten dates rendered as text, MRNs split across line breaks). Pursuing a learned redactor specialised to the long tail.
  • K-score temporal stability.
    Does an artifact that scored K = 0.95 today still score K = 0.95 in six months on the same hold-out set? Empirical longitudinal study in progress across the first 30 published recipes.

01frontier stack

What ships in production today.

The compiler is not a thin wrapper around a fine-tuning API. Each technique below is in the trainer or the runtime, gated on the hardware capability that justifies it, and recorded in the receipt of every run that uses it.

decode
Speculative decoding
Draft-model verification. Pairings: Qwen 7B→1.5B, Llama 3B→1B, Gemma 12B→1B. Wired in apps/runtime/serve.py.
memory
FP8 KV cache
Auto-on for compute capability sm_90+ (Hopper, Blackwell). vLLM kv_cache_dtype="fp8". 2x context per byte.
quant
AWQ · GPTQ · int4
Native vLLM weight-only quant. The 14B distill that runs on an A100-40GB box without paging to host RAM.
training
NVFP4 detect
Auto-enable on sm_120 (Blackwell, RTX 50-series) when torch ≥ 2.8 and cuBLASLt 12.9 are present. Falls back to bf16 otherwise.
attn
FlashAttention 3
Selected by _attn_impl_for() on Hopper / Blackwell; fa2 on Ampere; sdpa elsewhere. No user knob.
kernels
Liger fused kernels
Patches RMSNorm, SwiGLU, RoPE, and the LM head for Qwen / Llama / Gemma / Phi. ~1.6–2.0x training-time on Ampere+.
trainer
Unsloth fast-LoRA
Optional path for ≤14B SFT. Smaller VRAM envelope, identical adapter shape on disk.
optimizer
paged AdamW 8-bit
bitsandbytes paged states. Lets a 14B fine-tune fit a single A100-80GB or RTX 5090. Auto-on when CUDA is detected.
objective
UL2 span corruption
Optional pre-SFT mixed-denoising pass. Implementation at apps/trainer/span_objective.py.
align
DPO post-SFT
Optional trl.DPOTrainer stage on the SFT checkpoint when a preference pack is supplied. K-score gate still binding.
proof
HMAC-SHA256 chain
Five-step receipt over canonical-JSON manifest. CID (cidv1:sha256) pins the exact bytes. Verifier in pure Rust at packages/runtime-rs/.
isolation
TEE attestation
Nitro COSE_Sign1, SEV-SNP, TDX quote, GCP/Azure CVM, docker. Quote travels with the artifact. TOFU model.
align
KTO · ORPO · SimPO · IPO
Five-method preference router. Picks ORPO when the buyer has SFT data and tight VRAM, SimPO when only pairs and no ref model. Implementation at apps/trainer/preference.py.
reasoning
GRPO with verifiable rewards
Online RL for chain-of-thought (DeepSeek R1 pattern). Group-relative advantages, no value head. Reward functions: code-exec, math-checker, schema-validator. Wired to the K-score evaluator suite.
adapter
DoRA · rsLoRA · LoRA+ · NEFTune
Composable variants. --quality preset enables all four; --fast stays on plain LoRA. Optimizer auto-splits A and B param groups when LoRA+ is on.
decode
MEDUSA · EAGLE-2
Multi-head and tree-verified spec decoding beyond the draft-model baseline. ~2.3-2.8x (MEDUSA) and ~3-4x (EAGLE-2) on H100. Wired via vLLM 0.7+ speculative_model.
grammar
Constrained generation
Outlines + lm-format-enforcer logits processors. JSON-schema, regex, CFG, and choice modes. JSON-mode without retries, tool calls that always parse.
tools
OpenAI-compatible function calling
Native render for Qwen / Llama / Hermes; union schema fallback feeds the constrained-decode path. Tool emissions backed by JSON-schema validation, every call parses.
serving
Disaggregated prefill / decode
DistServe / Mooncake pattern. Single-node 2x-GPU INPROC mode and cross-host CROSSHOST mode with NVLink, RDMA, or TCP KV transport. Receipt records the transport.
eval
LLM-as-judge
Pointwise (1-10 rubric) and pairwise (A vs B, bias-corrected) modes. Reported alongside K-score in the receipt, never replaces the gate. Budget-capped per run.
safety
Hallucination detection
Vectara HHEM-2.1 grounding score plus a claim-decomposition mode for RAG outputs. Per-claim min as the aggregate (conservative). Lands in the receipt.
benchmarks
Public eval packs
MedQA, FinQA, LegalBench, GPQA, HumanEval, MT-Bench, AlpacaEval, Arena-Hard. Run alongside K-score; comparison numbers next to the buyer's own distribution.
data
Magpie · Evol-Instruct · Self-Instruct · RAFT
Cold-start synthesis when the capture pack is thin. The receipt carries a synth_provenance block: by-method counts, model, seed.
observability
OpenTelemetry export
OTLP/HTTP exporter for spans and metrics, no SDK pull. Set KOLM_OTEL=1 and point OTEL_EXPORTER_OTLP_ENDPOINT; lands in Honeycomb, Tempo, Datadog, Jaeger.
compute
14 compute backends
Local-first dispatch across CPU, CUDA, MPS, MLX, ROCm, DirectML; cloud add-ons on Modal, RunPod, Together, Vast, Lambda, Replicate, fal, and remote SSH. The picker scoring formula is documented at /compute.
02frontier reference

The forty-two, with the gating condition for each.

The evidenced answer to "are you state of the art?" is a list with the gating hardware or data condition for each entry. Below: every technique in the trainer, the runtime, or the eval pipeline, with the paper, the file path, and what must be true on the host machine or in the data for it to take effect. Recorded in the receipt of every run that uses it.

Training
01

LoRA variants: DoRA, rsLoRA, LoRA+, NEFTune

file apps/trainer/lora_variants.py papers Hu 2021, Liu 2024 DoRA, Kalajdzievski 2023 rsLoRA, Hayou 2024 LoRA+, Jain 2023 NEFTune gate PEFT ≥ 0.10

Plain LoRA is the floor; four variants compose on top. --quality turns on rsLoRA + DoRA + LoRA+ + NEFTune. --fast stays on plain LoRA for iteration. The optimizer auto-splits A and B parameters into separate LR groups when LoRA+ is on, with feature-detect fallback when PEFT is too old to recognise use_dora or use_rslora.

02

Preference optimization: DPO, KTO, ORPO, SimPO, IPO

file apps/trainer/preference.py papers Rafailov 2023 DPO, Ethayarajh 2024 KTO, Hong 2024 ORPO, Meng 2024 SimPO, Azar 2023 IPO gate trl ≥ 0.11.0

DPO was the first single-stage preference algorithm. Every newer method beats it on at least one axis: KTO eats binary labels (no pairs), ORPO folds SFT and preference into one pass (no reference model), SimPO drops the reference model entirely, IPO survives deterministic preferences. The router recommend_method() picks the right one from data shape and VRAM. See the report for the full decision tree.

03

GRPO with verifiable rewards (DeepSeek R1 pattern)

file apps/trainer/grpo.py papers Shao 2024 GRPO arXiv:2402.03300, DeepSeek-AI 2025 R1 arXiv:2501.12948 gate trl ≥ 0.12 + verifiable reward function

Group Relative Policy Optimization is online RL without a value head: sample G completions per prompt, score each with a programmatic reward, advantage is the group-mean-normalised score. Three built-in reward families: code_exec (subprocess sandbox + unit tests), math_checker (numeric or symbolic equivalence), schema_validator (JSON-schema or regex match). Reward functions plug into the K-score evaluator surface. See the report.

04

Paged AdamW 8-bit

file apps/trainer/trainer_real.py lib bitsandbytes ≥ 0.43 gate CUDA available

Paged optimizer states keep the 14B SFT envelope inside a single A100-80GB or RTX 5090. Auto-on when CUDA is detected; falls back to torch.optim.AdamW on Apple silicon, ROCm, and CPU.

05

Liger fused kernels

file apps/trainer/trainer_real.py · _maybe_apply_liger() lib liger-kernel ≥ 0.4.0 gate Ampere (sm_80) and newer

Patches RMSNorm, SwiGLU, RoPE, and the LM head for Qwen, Llama, Gemma, Phi. Roughly 1.6 to 2.0x training-time on Ampere and Hopper. No user knob; auto-applied when the architecture matches.

06

Unsloth fast-LoRA

file apps/trainer/backends/unsloth_path.py lib unsloth ≥ 2024.10 gate ≤ 14B params, CUDA, Triton 2.3+

Optional path for SFT up to 14B. Smaller VRAM envelope at the same step count; the adapter shape on disk is identical so the artifact stays portable across the standard PEFT path.

07

UL2 span corruption objective

file apps/trainer/span_objective.py paper Tay et al, 2022, arXiv:2205.05131 gate opt-in via spec

Optional pre-SFT mixed-denoising pass. Helps when the capture pack is short on contiguous prose. Implemented as a tokenizer post-processor over the SFT dataset; never replaces SFT, only precedes it.

08

NVFP4 detect

file apps/trainer/trainer_real.py gate sm_120 (Blackwell, RTX 50-series) + torch ≥ 2.8 + cuBLASLt 12.9 fallback bf16

Auto-detect new 4-bit floating-point training when all three preconditions are met. Falls back to bf16 silently; the receipt records which precision actually ran. Detection is conservative: when the cuBLASLt version cannot be probed cleanly, we choose bf16, not crash.

09

Vision-language fine-tuning (Qwen2.5-VL, LLaVA)

file apps/trainer/vlm.py papers Wang 2025 Qwen2.5-VL arXiv:2502.13923, Liu 2023 LLaVA arXiv:2304.08485, Tong 2024 Cambrian arXiv:2406.16860 gate transformers ≥ 4.49 + a VLM checkpoint

LoRA-only over the language tower; the vision encoder stays frozen by default so a single A100 can drive Qwen2.5-VL-7B with chart/document/screenshot mixes. The processor builds the chat template with image placeholders, the collator pads vision tokens correctly, and the receipt records which projector and which vision modules were touched. See the report.

10

Long-context training: YaRN, NTK-aware, Linear PI

file apps/trainer/long_context.py papers Peng 2023 YaRN arXiv:2309.00071, Chen 2023 Linear PI arXiv:2306.15595, Liu 2023 Ring Attention arXiv:2310.01889 gate model exposes rope_scaling in config

Three methods, one knob. YaRN is the default for 4-32x extension at low perplexity; NTK-aware for cheap 2-4x; Linear PI when the base model's rope_scaling field demands it. The trainer patches both model.config.rope_scaling and the nested text_config so VLMs extend cleanly. See the report.

11

Adapter merging: TIES, DARE, SLERP, linear

file apps/trainer/merge.py papers Yadav 2023 TIES arXiv:2306.01708, Yu 2023 DARE arXiv:2311.03099, Goddard 2024 Arcee arXiv:2403.13257 gate adapters share base model + shape

Four pure-tensor merge ops over PEFT LoRA adapters. Linear: weighted sum. SLERP: spherical interpolation along the unit sphere. DARE: Bernoulli drop + rescale by 1/density. TIES: topk-by-magnitude trim, sign-elect by weighted sum, merge only agreeing entries. Deterministic with a fixed seed; the receipt records method, weights, density, and per-adapter source CIDs. See the report.

12

Embedding and reranker fine-tuning

files apps/trainer/embedding.py, apps/trainer/reranker.py papers Oord 2018 InfoNCE arXiv:1807.03748, Kusupati 2022 Matryoshka arXiv:2205.13147, Chen 2024 BGE Reranker arXiv:2402.03216 gate triplets (query, pos, neg) or pointwise pairs (query, doc, label)

Two heads, one recall stack. The embedding trainer does InfoNCE with optional Matryoshka loss across nested dims (64/128/256/512/768) so one model serves four different latency tiers. The reranker trainer drops a cross-encoder head on top of any encoder, BCE for binary relevance or MSE for pointwise score. Both feed the registry's recall index. See the report.

13

Function-calling SFT (Hermes-FC)

file apps/trainer/function_calling.py papers Liu 2024 ToolACE arXiv:2409.00920, Patil 2023 Gorilla arXiv:2305.15334 gate trl + a teacher tool-call corpus

SFT over the open Hermes-FC chat template: tool schemas in the system prompt, tool calls as <tool_call> JSON blocks, tool responses as <tool_response>. The dataset validator rejects bad shapes before training; the parser at inference time recovers cleanly from minor format drift. Composes with the constrained-decode tool-call union schema at serve time. See the report.

14

Bradley-Terry reward modeling

file apps/trainer/reward.py papers Bradley-Terry 1952, Stiennon 2020 arXiv:2009.01325, Ouyang 2022 arXiv:2203.02155, Bai 2022 arXiv:2204.05862 gate chosen-vs-rejected preference pairs

The pairwise log-odds loss that powers RLHF, DPO sanity-checks, and best-of-N selection. A LoRA-adapted regression head on the SFT base; trained on (prompt, chosen, rejected) triples; shipped as its own *.rm.kolm artifact so the buyer's auditor can verify pair-accuracy independently from the policy. See the report.

15

Federated compile (FedAvg + secure aggregation + DP)

file apps/trainer/federated.py papers McMahan 2017 FedAvg arXiv:1602.05629, Bonawitz 2017 CCS, Beutel 2020 Flower arXiv:2007.14390, Abadi 2016 CCS DP gate clients reachable + pre-distributed PRG seeds

For hospitals, banks, and law firms that cannot pool rows. FedAvg over LoRA adapters (50 MB up/down per round, not 7 GB), Bonawitz pairwise HMAC-derived masks for secure aggregation, t-of-n Shamir reconstruction on client dropout, clip-and-noise differential privacy with a moments accountant. The receipt records cumulative (eps, delta) and the per-round dropout-recovery count. See the report.

16

Voice adapters (Whisper LoRA cross-attention)

file apps/trainer/audio.py papers Radford 2022 Whisper arXiv:2212.04356, Hu 2021 LoRA arXiv:2106.09685, Gandhi 2023 Distil-Whisper arXiv:2311.00430 gate 5-50 hours paired domain audio at 16 kHz

Whisper-large-v3 is the right base; the narrow change is LoRA on the decoder's cross-attention projections only, encoder frozen. ~40 MB adapter on disk; 30-second mel-spectrogram windows; the collator replaces pad-token ids with -100 so loss ignores padding; reports overall WER, focus-term WER, and per-speaker variance. See the report.

17

Response distillation (token-level KL, MiniLLM, on-policy)

file apps/trainer/distill.py papers Hinton 2015 arXiv:1503.02531, Kim & Rush 2016 arXiv:1606.07947, Gu 2024 MiniLLM arXiv:2306.08543, Agarwal 2024 arXiv:2306.13649 gate teacher and student share a vocabulary

The compression that gives Kolmogorov its name. Token-level KL between teacher and student softmaxes at a shared temperature, plus a small cross-entropy term against the teacher's sampled token. Three objectives ship (forward-KL, reverse-KL, JSD), top-k logit pruning collapses the activation memory by ~2000x on Qwen-class vocabularies, and on-policy sampling beats off-policy when the gap is large. See the report.

18

Mixture of LoRA experts (learned router across signed adapters)

file apps/trainer/moe.py papers Shazeer 2017 arXiv:1701.06538, Fedus 2022 Switch arXiv:2101.03961, Jiang 2024 Mixtral arXiv:2401.04088, Liu 2023 MoLE arXiv:2310.18339 gate ≥ 2 experts on the same base + labeled routing examples

One small MLP router that picks among N domain-specific LoRA adapters on the same base. Top-1 Switch-style or top-k Mixtral-style, with auxiliary z-loss and load-balance terms to keep the softmax stable and the experts evenly used. The router is the only joint parameter; every expert keeps its own signed artifact and its own audit trail. See the report.

19

Online iterative DPO (capture, judge, train, swap, repeat)

file apps/trainer/online_dpo.py papers Xu 2023 Cringe arXiv:2312.16682, Yuan 2024 Self-Rewarding arXiv:2401.10020, Calandriello 2024 arXiv:2403.08635, Rosset 2024 DNO arXiv:2404.03715, Dong 2024 RLHF Workflow arXiv:2405.07863 gate prior adapter + a judge (RM, LLM, self, or verifiable)

Sample fresh candidates from the current policy each round, judge them, form best-vs-worst pairs, train one DPO step against the previous round's policy, swap the new adapter, repeat. Four judge kinds (learned RM, LLM judge with order-swap correction, self-rewarding rubric, verifiable check shared with K-score). Each round emits its own signed artifact; the cross-round chain is the audit story. See the report.

Decoding
20

Speculative decoding (draft-model)

file apps/runtime/serve.py pairings Qwen 7B→1.5B, Llama 3B→1B, Gemma 12B→1B gate vLLM ≥ 0.6

Two-pass decode: small draft proposes k tokens; target verifies in parallel. ~2x throughput on most chat workloads, no quality loss. Wired via vLLM speculative_model=<draft>. See the report for the throughput math.

21

MEDUSA and EAGLE-2 spec heads

file apps/runtime/medusa.py papers Cai 2024 MEDUSA arXiv:2401.10774, Li 2024 EAGLE-2 arXiv:2406.16858 gate trained head artifact in the registry

Beyond the draft-model baseline: MEDUSA trains extra prediction heads on the base body (~2.3 to 2.8x throughput on H100, ~10% extra params). EAGLE-2 trains a single head that predicts hidden states and verifies a dynamic tree of continuations (~3 to 4x, ~5% extra params). Both wire through vLLM 0.7+ speculative_model with a prefix tag (medusa:, eagle2:).

22

EAGLE-3 and Lookahead decoding

files apps/runtime/eagle3.py, apps/runtime/lookahead.py papers Li 2025 EAGLE-3 arXiv:2503.01840, Fu 2024 Lookahead arXiv:2402.02057, He 2023 REST arXiv:2311.08252 gate EAGLE-3 needs a trained head; Lookahead needs none

The 2025 spec-decode upgrades. EAGLE-3 drops the feature-prediction auxiliary loss and trains on multi-layer hidden states; ~30% throughput over EAGLE-2 at the same target quality. Lookahead decoding skips draft training entirely: a session-local n-gram cache proposes continuations from the target's own decoding history; ~1.5-2x speedup at zero training cost. Both wire through the same speculative_config surface. See the report.

23

Constrained generation

file apps/runtime/constrained.py libs outlines ≥ 0.0.45, lm-format-enforcer ≥ 0.10 gate logits-processor-compatible runtime

JSON schema, regex, EBNF/CFG, and choice modes layer on top of any vLLM or transformers generation. JSON-mode without retries, tool calls that always parse, classifier outputs that always validate. Three-tier backend: Outlines first, lm-format-enforcer second, post-hoc parse-and-retry third. See the report.

24

OpenAI-compatible tool calling

file apps/runtime/tools.py formats Qwen tool_call, Llama function tags, Hermes-2-Pro, generic JSON envelope gate none

The tools field on a request renders into the prompt in the format the target model expects, and the constrained-decode union schema guarantees the emission parses. tool_choice translates to a tightened schema (none / auto / required / specific). Tool calls land in the receipt with a deterministic id, the parsed arguments, and the source bytes.

25

Test-time compute: best-of-N, self-consistency, reflexion

file apps/runtime/ttc.py papers Cobbe 2021 verifiers arXiv:2110.14168, Wang 2022 self-consistency arXiv:2203.11171, Shinn 2023 Reflexion arXiv:2303.11366, Snell 2024 TTC scaling arXiv:2408.03314 gate optional reward function or verifier

Three orthogonal compute-spend strategies at inference. Best-of-N draws N samples and picks the highest-scoring with a reward function or PRM. Self-consistency draws N and majority-votes the extracted answer. Reflexion runs a critic loop until the critic accepts or budget runs out. All three compose with constrained decoding so every sample parses. See the report.

Serving
26

FP8 KV cache

flag kv_cache_dtype="fp8" gate compute capability sm_90+ (Hopper, Blackwell) gain 2x context per byte

Auto-on when the host GPU exposes sm_90 or higher. Doubles the on-device KV budget at the same VRAM, which is the difference between 32k context and 64k for a 14B-param model on a single H100.

27

AWQ / GPTQ int4 weight quant

libs autoawq, auto-gptq gate sm_80+ for AWQ kernels VRAM ~4 GB for a 14B int4 model

Native vLLM weight-only quant. The 14B distill that runs on an A100-40GB without paging to host RAM, or on an RTX 5090 next to the buyer's training job. Selected per artifact in the manifest.

28

FlashAttention 3

file apps/trainer/trainer_real.py · _attn_impl_for() gate Hopper / Blackwell (sm_90+) fallback fa2 on Ampere, sdpa elsewhere

Auto-selected by the trainer per detected device. No user knob; the receipt records which attention implementation ran, so a buyer can re-run on the same SKU and reproduce.

29

Disaggregated prefill / decode

file apps/runtime/disagg.py papers Zhong 2024 DistServe arXiv:2401.09670, Qin 2024 Mooncake arXiv:2407.00079 gate ≥ 2 GPUs (INPROC) or RDMA/IB fabric (CROSSHOST)

Prefill (compute-bound) and decode (memory-bound) run on separate GPU pools so each gets the right parallelism. Single-node INPROC mode for ≥ 2 GPUs on the same box, CROSSHOST mode with NVLink, RDMA, or TCP KV transport for fleet deployment. Receipt records the mode and transport.

30

Multi-LoRA serving (S-LoRA pattern)

file apps/runtime/multi_lora.py papers Sheng 2023 S-LoRA arXiv:2311.03285, Chen 2024 Punica arXiv:2310.18547 gate peft ≥ 0.10, one shared base model

One base model, dozens of adapters, microsecond swaps. The router holds an LRU of warm adapters, drops the cold ones on demand, and logs swap latency above a threshold. PeftModel.load_adapter on first encounter, set_adapter on every subsequent. The receipt pins which adapter ran each request. See the report.

Evaluation
31

K-score release gate

file apps/eval/kscore.py formula K = 0.40·A + 0.15·S + 0.15·L + 0.15·C + 0.15·V gate K ≥ 0.85 to ship

The kolm flagship metric: a weighted aggregate of accuracy on the buyer's own captures, size, latency, cost, and coverage. The 0.85 cutoff is not negotiable; the entire other thirty-eight techniques on this page can win zero rows if the buyer's distribution is the one that matters.

32

LLM-as-judge evaluation

file apps/eval/judge.py papers Zheng 2023 MT-Bench arXiv:2306.05685, Liu 2023 G-Eval arXiv:2303.16634 gate optional, reported next to K-score

Pointwise (1-10 rubric) and pairwise (A vs B, bias-corrected by averaging both orderings) modes. Reported alongside K-score in the receipt; never replaces it. Budget-capped per run with a per-model token cost table.

33

Mixture of judges (panel scoring)

file apps/eval/judges_mix.py papers Verga 2024 PoLL arXiv:2404.18796, Wang 2024 MoA arXiv:2406.04692 gate ≥ 2 judge endpoints

A single judge is biased; a panel of three judges from different families is bias-attenuated. Aggregation modes: mean, median (default), majority, quorum-k, trimmed-mean. The receipt records each judge's score and the panel spread so a buyer can audit disagreement. See the report.

34

Hallucination detection (HHEM)

file apps/eval/hhem.py model vectara/hallucination_evaluation_model (HHEM-2.1) gate needs transformers, fits CPU

NLI-style grounding score in [0,1] between context and generated output. Two granularities: document-level and claim-level (split into sentences, score each, aggregate as the conservative minimum). Lands in the receipt for every RAG-distilled artifact.

35

Public eval packs

file apps/eval/packs.py packs MedQA, FinQA, LegalBench, GPQA, HumanEval, MT-Bench, AlpacaEval, Arena-Hard gate datasets ≥ 2.18

K-score is the right gate for the buyer's distribution, but the regulator and the engineering review board also want standard public-pack numbers. The packs run with three grader types: MCQ exact-match, sandboxed exec, and judge-graded. Numbers ship next to K-score, not on top of it.

36

Process reward model (step-level scoring)

file apps/eval/prm.py papers Wang 2023 Math-Shepherd arXiv:2312.08935, Lightman 2023 PRM-800K arXiv:2305.20050, Lai 2024 Step-DPO arXiv:2406.18629 gate a fine-tuned PRM checkpoint or generic LM

Step-level rewards for reasoning chains. An ORM scores the final answer; a PRM scores each intermediate step. Math-Shepherd's product aggregation penalizes any single bad step without being as harsh as min. Plugs into best-of-N at the TTC layer for richer reward than answer-only scoring. See the report.

Data
37

Synthetic data: Magpie, Evol-Instruct, Self-Instruct, RAFT

file apps/data/synth.py papers Xu 2024 Magpie arXiv:2406.08464, Xu 2023 Evol arXiv:2304.12244, Wang 2022 Self-Instruct arXiv:2212.10560, Zhang 2024 RAFT arXiv:2403.10131 gate needs a teacher endpoint (OpenAI-shaped)

Cold-start synthesis when the buyer has fewer than ~200 captures. Magpie exploits the chat-template prefix to autoregress user turns from a base model. Evol-Instruct evolves a seed pack along depth and breadth axes. Self-Instruct bootstraps from 8-shot seed samples. RAFT mixes gold-only, gold+distractor, and pure-distractor for RAG distillation. The receipt carries a synth_provenance block: by-method counts, model, seed.

38

Document ingestion (PDF, EML, HTML, CSV)

file apps/data/ingest.py papers Wang 2022 Self-Instruct arXiv:2212.10560, Hendrycks 2020 MMLU arXiv:2009.03300, Lewis 2020 RAG arXiv:2005.11401 gate documents on disk (no vector DB needed)

The RAG-replacement path. kolm: a compiler for task-specific AI artifacts.

Frontier inference is rented; the cost compounds. Hand-glued local pipelines are cheap to run but expensive to ship, audit, and revoke. We describe a build step that takes a task description plus a small corpus and produces a single signed file containing a base model pointer, a LoRA adapter, a recall index, evaluation packs, and a receipt chain. 04technical reports

Engineering notes from the lab.

Each report is a real engineering finding from shipping the compiler. Five-to-twelve-minute reads. Reverse chronological.

trainingMay 15, 2026

Response distillation · KL the teacher into the student.

The compression that gives Kolmogorov its name. Token-level KL between teacher and student softmaxes at a shared temperature, plus a small cross-entropy term against the teacher's sampled token. Three objectives (forward-KL, reverse-KL, JSD), top-k logit pruning that drops activation memory ~2000x on Qwen-class vocabularies, and on-policy sampling that beats off-policy when the capacity gap is large.

10 min →
trainingMay 15, 2026

Mixture of LoRA experts · learned routing across signed adapters.

One small MLP router that picks among N domain-specific LoRA adapters on the same base. Top-1 Switch-style or top-k Mixtral-style. Auxiliary z-loss and load-balance terms to keep the softmax stable and the experts evenly used. The router is the only joint parameter; every expert keeps its own signed artifact and its own audit trail.

9 min →
trainingMay 15, 2026

Online iterative DPO · close the preference loop with production receipts.

Sample fresh candidates from the current policy each round, judge them, form best-vs-worst pairs, train one DPO step against the previous round's policy, swap the new adapter, repeat. Four judge kinds (learned RM, LLM judge with order-swap correction, self-rewarding rubric, verifiable check). Each round emits its own signed artifact; the cross-round chain is the audit story.

11 min →
trainingMay 14, 2026

Bradley-Terry reward models · the scoring head behind preference optimization.

The pairwise log-odds loss that powers RLHF, DPO ablations, and best-of-N selection. A LoRA-adapted regression head on the SFT base; trained on chosen-vs-rejected pairs; shipped as its own signed artifact so the buyer's auditor can verify pair-accuracy independently from the policy. Why the receipt records the pair-accuracy and reward-gap mean, not the loss.

trainingMay 14, 2026

Federated compile · LoRA under secure aggregation and DP.

For hospitals, banks, and law firms that cannot pool rows. FedAvg over LoRA adapters (50 MB up/down per round, not 7 GB), Bonawitz HMAC-derived pairwise masks for secure aggregation, t-of-n Shamir reconstruction on client dropout, clip-and-noise differential privacy with a moments accountant. The receipt carries cumulative (eps, delta) and the per-round dropout-recovery count.

Three RoPE extension methods, one config knob. YaRN is the default for 4-32x at low perplexity; NTK-aware is cheap 2-4x; Linear PI when the base model's config demands it. methodsMay 14, 2026

Adapter merging · TIES, DARE, SLERP, linear.

Four pure-tensor ops over PEFT LoRA adapters. Linear is the floor; SLERP traces the unit sphere; DARE drops 50% with Bernoulli and rescales by 1/density; TIES trims by magnitude, elects sign by weighted vote, merges agreeing entries. Deterministic with a fixed seed. The receipt pins method, weights, density, and per-adapter source CIDs.

SFT over the open Hermes-FC chat template: tool schemas in system, <tool_call> JSON blocks in assistant, <tool_response> in user. The dataset validator rejects bad shapes before training. runtimeMay 14, 2026

Test-time compute · best-of-N, self-consistency, reflexion.

Three orthogonal compute-spend strategies at inference. Best-of-N picks the highest-scoring of N samples with a reward function or PRM. Self-consistency majority-votes the extracted answer. Reflexion runs a critic loop. All three compose with constrained decoding so every sample parses, and with the PRM at the step level for richer reward than answer-only scoring.

runtimeMay 14, 2026

Multi-LoRA serving · one base, dozens of adapters.

S-LoRA's pattern, distilled to its load-bearing core. The router holds an LRU of warm adapters, drops cold ones on demand, and logs swap latency above threshold. PeftModel.load_adapter on first encounter, set_adapter after. Receipt pins which adapter ran each request. The single-base, many-tenants serving primitive.

evaluationMay 14, 2026

Mixture of judges and process reward models.

A single judge is biased; a panel of three from different families is bias-attenuated. Aggregation: mean, median (default), majority, quorum-k, trimmed-mean. PRM scores each step in a reasoning chain, not just the final answer, so an unlucky-but-correct trajectory gets fair credit. Both compose with best-of-N at the TTC layer.

Both wire through the same speculative_config surface; the right choice depends on training budget vs. peak throughput.
