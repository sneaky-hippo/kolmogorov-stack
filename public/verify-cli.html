kolm verify: a free standalone CLI for any .kolm artifact.

Verifies signature chain, manifest schema, frozen-eval hash, and K-score recomputation. Apache-2.0. No login, no telemetry, no network call required.

v1.0.0 . Apache-2.0 . zero runtime deps . conforms to RS-1 v2.1

Apache-2.0 no login no telemetry offline by default verifies any .kolm

Install

One install command per platform. The verifier is a single static binary; you do not need node, python, or docker at runtime. Network is required only to download the binary itself; the actual verify run is offline.

curl install (Linux, macOS)
curl -fsSL https://kolm.ai/install/verify | sh
docker (any host, no install)
docker run --rm -v "$PWD:/work" -w /work kolm/verify:latest verify ./artifact.kolm

Both surfaces resolve to the same Apache-2.0 binary built reproducibly from the kolm-stack repo. The container image is the binary plus a 5 MB Alpine base; no third-party runtime libraries are linked in.

What it verifies

Four checks, run in order. Any failure stops the chain and surfaces the failing slot with a non-zero exit code. The verifier never phones home; it reads the artifact and the optional public-key file you point it at.

1. Signature chainHMAC-SHA256 (integrity) + Ed25519 (authenticity) + optional sigstore bundle with Rekor anchor. Cascade encoding read from manifest.signature_alg; checks run innermost to outermost per RS-1 §5.1.
2. Manifest schemaValidates the canonical-JSON manifest against the RS-1 v2.1 shape: required fields, four-class taxonomy match, artifact_class versus contents, training block coherence for distilled_model.
3. Frozen-eval hashRe-derives the deterministic train/holdout split from seeds.jsonl + split_seed and confirms train_hash ∩ holdout_hash = ∅. Rejects tautological-eval artifacts per RS-1 §4.
4. K-score recomputationRe-runs the comparator over the holdout, recomputes A / S / L / C / V (and v2 axes when present), and confirms the manifest k_score matches within tolerance. Reports per-axis values on success.

Usage

kolm verify ./artifact.kolm
kolm verify ./artifact.kolm --pubkey ./signer.pub
kolm verify ./artifact.kolm --json
kolm verify ./artifact.kolm --strict
kolm verify --help

Exit codes follow POSIX convention: 0 all checks passed, 1 signature failure, 2 manifest schema failure, 3 frozen-eval disjointness failure, 4 K-score recomputation mismatch, 64 usage error. The --json flag emits a structured report suitable for CI pipes; the --strict flag escalates warnings (missing sigstore anchor when the manifest claims sigstore signing) to hard failures.

What it does not do

No login. The verifier never asks for credentials, never sends a request to kolm.ai, never reads a cookie or token. The four checks run against the local artifact bytes and the optional public-key file.
No telemetry. Zero outbound network calls during verify. The binary is open source (Apache-2.0) and reproducibly built; you can audit the call graph and confirm there is no telemetry path. The single optional network call is the Rekor transparency-log lookup when --require-rekor is set and the artifact carries a sigstore bundle.
No compile, no inference. The verifier reads and checks; it does not train, distill, or run a recipe. For build-side workflows use the full kolm CLI at /quickstart. kolm verify is intentionally narrow.

Why it is free

An audit primitive whose verifier costs money is not an audit primitive. RS-1 is an open standard; the free verifier is the contract that the standard is actually open. The full kolm CLI (build, distill, compile, deploy) is also Apache-2.0 but ships under a different repository with backend integrations; the standalone verifier is the read-side only, packaged for the case where a regulator, an auditor, or a downstream consumer wants to confirm "this artifact matches its manifest" without installing the full stack.

Conformance

The standalone verifier passes the same interop test suite (tests/interop/) as the JavaScript / Python / Rust reference verifiers documented in RS-1 §9. Byte-identical canonical-JSON output, byte-identical CID computation, byte-identical receipt verification results. A second-source verifier can be re-implemented in under two engineer-weeks against the spec; kolm verify is one such source.

Reproducible build

The release binary is built with SOURCE_DATE_EPOCH=0, deterministic linker flags, and the toolchain pin documented at RS-1 §7.5.1. The build produces a .kolm manifest of its own (verifying-the-verifier); the hash is published in RELEASES.md in the source repo.
