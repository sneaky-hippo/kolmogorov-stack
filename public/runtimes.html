Runtimes — source-build llama.cpp for your hardware

kolm runtime build-from-source --target=<arch> downloads a pinned llama.cpp commit, emits the exact cmake flags for your target, builds in workers/runtime-build/, and drops a build-receipt.json next to the binary. The heavy SDKs (cuda-toolkit, rocm-hip-sdk, vulkan-sdk, xcode-cli-tools) live in the isolated worker — the root kolm install stays light.

Honest framing — what kolm does, what it doesn't

kolm runs the build for you, then steps out of the way.

You get pinned source, pinned cmake flags, a deterministic out-dir, and a build-receipt.json that extends the W212 capture receipt chain to runtime binaries. We do not fork upstream llama.cpp, we do not patch its kernels, and we do not ship a vendored copy. If you want a different commit, pass --commit=<sha>. If you want a different repo (e.g., a fork with FP8 patches), pass --repo-url=<url>.

This answers the @sudoingX "if you're serious about local inference, start at source level" critique directly: one command, one pinned commit, no abstractions over the build.

Use it

# list every target the worker knows about
kolm runtime targets

# show the cmake flags + sdk requirement for one target
kolm runtime info cuda-89

# check whether the toolchain is installed on this box
kolm runtime doctor --target=cuda-89

# do the build (clones llama.cpp, checks out the pinned commit, runs cmake, installs)
kolm runtime build-from-source --target=cuda-89 --out=/opt/kolm/runtime/cuda-89

# preview without running anything
kolm runtime build-from-source --target=cuda-90 --dry-run --json

Receipt

Every successful build writes build-receipt.json in the out-dir:

{
  "ok": true,
  "target": "cuda-89",
  "repo_url": "https://github.com/ggerganov/llama.cpp.git",
  "commit": "2026-05-17-frontier-pin",
  "cmake_args": ["-G","Ninja","-DGGML_CUDA=ON","-DCMAKE_CUDA_ARCHITECTURES=89", ...],
  "elapsed_ms": 318411,
  "built_at": "2026-05-18T13:14:22.501Z",
  "binary_dir": "/opt/kolm/runtime/cuda-89/bin",
  "receipt_kind": "kolm-runtime-build/1"
}

This receipt is the runtime sibling of the x-kolm-capture-durable header and the .kolm manifest signatures — every artifact in the kolm stack ships with proof of what produced it.

Where the heavy deps live

The cuda-toolkit, rocm-hip-sdk, vulkan-sdk, and xcode-cli-tools are declared in workers/runtime-build/package.json under system_requires, NOT in the root kolm package.json. The root install stays light. The worker is only invoked when you opt in to kolm runtime build-from-source.
